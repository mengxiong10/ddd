# CONTEXT（术语表）

- **君主（Lord）**：一方势力的统领，**本身也是一名武将**。城与武将通过 `lordId` 归属于某位君主；不设独立的「势力」实体。第一版为玩家 1 君主（刘备）+ AI 1 君主（曹操）。玩家方由 `playerLordId` 标记。
- **城 / 城池（City）**：地图上的据点，通过 `lordId` 归属某位君主。持有属性 `农业`、`商业` 及各自的**城级上限**，城级资源 `金`、`粮`，以及驻城武将。
- **武将（Officer）**：执行经营指令的人，通过 `lordId` 归属某位君主（君主本人 `lordId` 指向自身）。带静态属性 `智力`、`体力`；本游戏不做武将成长/养成。
- **智力（Intelligence）**：武将静态属性，影响开垦/招商的属性增量。**自 `05-items` 起公式改吃「有效智力」（基础 + 道具加成）**。
- **体力（Stamina）**：武将可消耗资源，执行经营指令时扣减；达不到阈值不能下令。
- **在任武将**：当前位于该城、未被指令占用、可被下令的武将。
- **农业（Agriculture）**：城属性，决定收粮量；通过「开垦」提升，有上限。
- **商业（Commerce）**：城属性，决定收税量；通过「招商」提升，有上限。
- **金（Gold）**：城级资源，来自收税，并用于支付经营指令开销。
- **粮（Food）**：城级资源，来自收粮。
- **经营指令**：玩家/AI 在下令阶段对城中武将下达的行为，本切片为 `开垦`、`招商`。
- **开垦**：经营指令，提升本城 `农业`，占用一名武将。
- **招商**：经营指令，提升本城 `商业`，占用一名武将。
- **占人**：一条指令占用武将——下令后该武将离城（busy），月末统一回出发城、期间不可再被下令。**例外：出征**占人后月末不回出发城（胜进驻新城 / 败成俘虏停留敌城）。
- **月度推进**：游戏以「月」为最小时间单位推进，每月经历下令与月末结算。
- **收粮（Harvest）**：每年 6 月、10 月各结算一次，每城 `本次收粮 = floor(农业 / 4)` 计入该城 `粮`。
- **收税（Tax）**：每年 3、6、9、12 月（每季度）各结算一次，每城 `本次收税 = floor(商业 / 2)` 计入该城 `金`。
- **兵 / 兵力（Troops）**：可作战的人力，分两处存放——城级的「后备兵」与武将级的「武将兵」，二者是各自独立的真实状态（非冗余）。
- **后备兵（Reserve Troops）**：城级未编队兵力。征兵注入到此；分配在后备兵与武将兵之间调度。初始 0。
- **武将兵（Officer Troops）**：某武将当前带的兵，受其「带兵量上限」约束。初始 100。
- **民忠（Loyalty）**：城属性，范围 `[0, 100]`，反映民心。决定征兵上限（民忠×20）。**自 `03-plunder-scout` 起可被掠夺减半**（不再纯静态）；**自 `06-city-commands` 起可由「出巡」小幅回升**（`+RandInt(1,4)` 封顶 100），仍无自然涨跌机制。
- **人口（Population）**：城属性，由「出巡」+100 提升（fixture 播种初值）。**`06-city-commands` 切片仅为展示状态、无任何下游规则**（不影响征兵/收粮/收税），其作用留待后续切片。
- **武力（Force）**：武将静态属性，参与带兵量公式。**自 `05-items` 起公式改吃「有效武力」（基础 + 道具加成）**。
- **等级（Level）**：武将静态属性，参与带兵量公式；本阶段不成长。
- **带兵量上限（Troop Capacity）**：某武将可带兵的上限，`等级×100 + 武力×10 + 智力×10`。
- **征兵（Recruit）**：经营指令，**占人**。把民忠与金钱转化为城后备兵；下令时后备兵 += N、执行人体力 -12、城金 -= `ceil(N/10)`，可征上限 `min(民忠×20, 金×10)`。
- **分配（Allocate）**：指令，**不占人**、立即生效。在城后备兵与某武将现有兵之间重分配，受该武将带兵量上限约束；不扣体力/金。
- **掠夺（Plunder）**：经营指令，**占人**，**效果时机=月末**。牺牲本城发展换粮草：下令时扣执行人体力 12、入队、离城；月末（先于收粮/收税）破坏本城（农业/商业/民忠各 `floor(/2)`），并给本城 `粮 +=(智力+武力)×5`、`金 +=(智力+武力)×2`。
- **侦察（Scout）**：指令，**占人**，**效果时机=下令瞬间**。花本城金 20、执行人体力 10，立即展示一座**非己方、非本城**目标城的详情面板；执行人离城，月末回城，core 月末无实质效果。
- **效果时机**：一条指令的实质效果在何时结算——既有指令多为「下令瞬间」，掠夺为「月末」。
- **月末待执行队列（pendingCommands）**：本月已下令、但效果延到月末才结算的指令集合，元素为带 `type` 的判别式项；由 `turn` 层按 `type` 分派执行（与 `apply` 同构），自 `04-campaign` 起含掠夺与出征。月末先于收粮/收税执行并清空，不跨月残留；出征排在普通待执行指令之后。占人仍用 `Officer.busy` 表达（与本队列是不同事实）。
- **出征（Campaign）**：军事指令，**占人 ✓**（一次最多 10 名武将），**效果时机=月末**。从本城派最多 10 名在任武将携随军粮草进攻一座**相邻敌城**；下令时扣本城随军粮草、占用武将、入待执行队列；月末结算战斗。出征是占人指令的**例外**：武将月末不回出发城——胜则进驻被占城、败则就地成俘虏（cityId 由战斗结算改写）。
- **随军粮草（Provisions）**：出征下令时从本城城粮扣除的随军携带粮（整数，1 ~ 本城城粮）。攻方胜则并入被占城城粮，败则损失。**本切片不参与胜负判定**（胜负只看兵力）。
- **相邻 / 邻接（Adjacency）**：城与城之间的**无向**地图邻接关系（`04-campaign` 起引入的地图拓扑）。出征的"路径可达"= 目标城与出发城相邻；本切片不做多跳路径搜索。
- **战斗（Battle）**：出征月末的胜负结算。**本切片从简**：按双方兵力总和比大小——攻方=出征武将兵之和，守方=目标城归属方武将兵之和 + 后备兵（排除城内俘虏）；攻方 **严格大于** 守方才胜，否则守方胜（含平局）。确定性、不耗 RNG、无兵力损耗。后续切片深化为战棋。
- **占领 / 城易主**：攻方胜后目标城 `lordId` 改为攻方，连同其农业/商业/资源/后备兵整体易主；原守军武将就地成俘虏。
- **俘虏（Captive）**：**派生状态**，非存储字段——当 `officer.lordId !== 所在城.lordId`（武将自身归属 ≠ 所在城归属）时即为俘虏。城被占领时原守军在原地即成俘虏。俘虏本切片"什么都不变"（lordId/cityId/兵/体力均不动），仅在派生意义上被俘；**不可被任何指令指派**，"在任武将"定义排除俘虏，守方兵力统计与重选君主候选池均排除俘虏。
- **道具（Item）**：带唯一 id 的实体，含 `武力加成`、`智力加成`（非负整数，可为 0）。归属为**单一真相源**——存于 `GameState.items`，每件记录其归属（属于某城 cityId **或** 某武将 officerId，二选一）；城/武将的道具列表均由 selector 派生，不存第二份。每名武将最多持 2 件（不变量）。
- **有效武力 / 有效智力（Effective Force / Intelligence）**：派生属性——基础武力/智力 + 所持道具对应加成之和。`05-items` 起带兵量上限、开垦/招商增量等所有武力/智力公式统一吃有效值。
- **武将忠诚（Loyalty, 武将级）**：武将级属性，范围 `[0, 100]`，普通武将初始 50；**与城的「民忠」是不同事实**。**君主忠诚派生恒为 100**（忠诚增减只作用于非君主）。`05-items` 起由赏赐(+8)/没收(−20)增减；本切片仅为状态，无下游规则（叛变/策反留待后续）。
- **赏赐（Reward）**：指令，**目标是武将（非执行人）、不占人、效果时机=下令瞬间**，无金/体力成本。把一件属于「目标武将所在城」的道具转给该城己方非俘虏武将（武将道具数 < 2）；成功后道具归属改为该武将、其忠诚 +8（封顶 100）。非法即 no-op。
- **没收（Confiscate）**：指令，**目标是武将（非执行人）、不占人、效果时机=下令瞬间**，无金/体力成本。把目标（己方非俘虏）武将所持的一件道具收回其所在城；成功后道具归属改为该城、其忠诚 −20（下限 0）。非法即 no-op。
- **出巡（Patrol）**：经营指令，**占人 ✓**，**效果时机=下令瞬间**（不入 pendingCommands）。花本城金 50、执行人体力 8，立即提升本城民忠（`+RandInt(1,4)`，封顶 100）与人口（+100）；执行人离城，月末回城。
- **宴请（Banquet）**：指令，**占人 ✗**、**效果时机=下令瞬间**。花本城金 100，给该城一名**在任**武将体力 +50（封顶 100），非君主额外忠诚 +1（封顶 100）；不占用其本月行动（被宴请者仍可被其他指令派遣）。
- **移动（Move）**：经营指令，**占人 ✓**，**效果时机=月末**（进 pendingCommands，`move` 分支）。把本城一名在任武将搬到己方另一座城（非本城、同势力，不考虑路径）；不扣体力/金。是占人**例外**（同出征）：月末武将落到目标城（`cityId` 改写）、不回出发城。
- **输送（Transport）**：经营指令，**占人 ✓**，**效果时机=月末**（进 pendingCommands，`transport` 分支）。把粮/金/**后备兵**送往己方另一座城（非本城、同势力，不考虑路径）；下令时即扣执行人体力 8 并从出发城扣掉所送资源；月末 **80% 送达**（目标城接收）、**20% 永久损失**（不退回）；执行人月末回出发城。
- **交易（Trade）**：经营指令，**占人 ✓**，**效果时机=下令瞬间**。在本城以固定单价买卖粮食兑换城金——买入 5 金/粮（上限 `floor(城金/5)`）、卖出 2 金/粮（上限本城所有粮，城金不设上限）；扣执行人体力 12；执行人离城，月末回城。
- **重选君主**：某势力君主被俘（君主所在城归属 ≠ 君主自身 lordId）且该势力仍有其他城时触发——从该势力剩余**未被俘**武将中取**智力最高**者为新君主，把该势力其余城与未被俘武将的 `lordId` 改归新君主；被俘君主与俘虏不改归属。
- **势力灭亡**：某势力君主被俘且已无任何城池时，该势力消亡（剩余武将均为俘虏）。本切片不据此做统一全国的全局胜负判定。
- **登场（Debut）**：未登场武将/道具到达其「登场年」后于月末出现到某城的运行时事件。`06-debut-search` 起，月末固定顺序扩展为 `pendingCommands → settle → 回城+体力恢复 → 月份+1 → 登场`；登场在「月份+1」之后、按新年份判定。
- **登场年（Debut Year）**：未登场实体现身的年份（只到年、不含月）；月末凡 `当前年 ≥ 登场年` 者登场。
- **未登场（Unappeared）**：实体尚未进入对局——暂不在 `state.officers`/`state.items`，存于 `GameState` 顶层的「待登场池」，记录其完整数据 + 登场年 + 目标城（可空=随机）。
- **待登场池（Debut Pool）**：`GameState` 顶层集合，持有未登场武将/道具数据；元素到达登场年后于月末登场并一次性移出。
- **无主 / 在野（Wandering）**：武将不归属任何君主——`Officer.lordId` 可空，`null` 即无主，统一覆盖「未登场」与「在野（已登场未被招募）」。在野武将对玩家隐藏：不计入「在任武将」、不参与守城兵力、不可被指令指派、不被判为俘虏，仅可经搜寻招募。归属武将 `lordId` 非空。
- **隐匿武将 / 隐藏道具**：在野武将（`lordId===null`）与未发现道具（`discovered===false`）的统称——已登场但需经搜寻方能被发现/招募。
- **已发现 / 未发现道具（Discovered）**：`Item.discovered` 字段。fixture 既有道具恒 `true`；登场道具落城时 `false`，对玩家隐藏且不可被赏赐，经搜寻发现后置 `true`。
- **伯乐（Recruiter）**：武将/道具的 `recruiterId: OfficerId | null` 字段（`null`=无指定）。决定搜寻时该目标能否被某执行人招募/发现。
- **搜寻（Search）**：经营指令，**占人 ✓**，**效果时机=月末**。前置：城有≥1 在任武将、执行人体力 ≥ 8、不需金钱；下令扣体力 8、占人、入队。月末四分支各 1/4：无事 / 尝试发现武将或道具 / 获得金钱 / 获得粮食（金钱、粮食上限均 `max(10, 有效智力×2)`，封顶 30000）。
- **发现判定**：搜寻「发现」分支的内部判定——先 `RandInt(0,149) < 有效智力` 过筛（否则无事），再 1/2 决定找武将或道具；候选为空当作无事。
- **招募（Recruit）**：搜寻发现并收编在野武将——伯乐=执行人必成、伯乐=null 看 `RandInt(0,109)<有效智力`、伯乐=他人必败；成功后该武将归执行人君主、忠诚 `RandInt(70,99)`、cityId 保持本城、troops 为 0。
- **性格（Personality）**：武将存储字段，单值 `0..4`。**两套解读同一值**：君主（`lordId===自身`）按**君主性格表**（0 和平 / 1 大义 / 2 奸诈 / 3 狂人 / 4 冒进），非君主按**普通武将性格表**（0 忠义 / 1 大志 / 2 贪财 / 3 怕死 / 4 卤莽）；随身份/重选君主自动切换解读，不另存。`08-personality-captive` 起普通武将性格驱动招降难度；**君主性格暂仅展示、无下游规则**。
- **招降（Suborn）**：指令，**占人 ✓**、**效果时机=月末**（入 `pendingCommands` `suborn` 分支）。让本城俘虏归顺本势力。前置：城有俘虏（目标）+ 在任武将（执行人）、执行人体力 ≥ 15、城金 ≥ 100；下令扣体力 15 / 城金 100、占人、入队。月末四关判定：①智力差关 `RandInt(0,99) > 执行人有效智力 − 目标有效智力 + 50` 失败；②按目标性格取招降系数；③读原始忠诚 `L0`、扣 `floor(L0/10)`（持久，含失败）、`L0>60` 直接失败；④终判 `RandInt(0,99) < floor(L/S)` 失败否则成功。成功：俘虏 `lordId` 改归执行人君主、忠诚 `RandInt(40,79)`。
- **招降性格系数（S）**：目标普通武将性格对应的招降难度系数——忠义 1 / 卤莽 2 / 大志 3 / 贪财 4 / 怕死 5（按 `personality` 取系数表 `[1,3,4,5,2]`）；系数越大终判失败阈值越小、越易招降。
- **处斩（Behead）**：指令，**占人 ✗**、**效果时机=下令瞬间**。永久消除本城俘虏：目标所持道具退回本城（`holder=城`）并标记 `discovered=true`，该武将从 `state.officers` 永久删除；无 RNG / 体力 / 金成本。
- **流放（Banish）**：指令，**占人 ✗**、**效果时机=下令瞬间**。把本城一人（在任武将或俘虏皆可，**非在任君主**；被俘君主可流放）变在野（`lordId=null`），**其道具随人带走**（`holder` 仍指向该武将、不退城、不改 `discovered`），并在**全部城**中等概率随机选一座（可能含原城）落城（`cityId` 改写）；消费 RNG。
- **城市状态 / 灾害状态（City Status）**：`City` 存储字段，单值 `正常 / 饥荒 / 旱灾 / 水灾 / 暴动`，初始正常（fixture 播种）。`09-city-disaster` 起每月可生成新灾、按状态破坏、并经自然或治理恢复。
- **防灾值（Disaster Prevention）**：`City` 存储字段，`[0, 100]` 整数（fixture 播种）。越高越不易发新灾、越快从旱灾/水灾恢复；由「治理」`+RandInt(1,4)` 封顶 100 提升。与民忠并列为「防灾 vs 防乱」两条投入线（民忠降暴动概率/加速暴动恢复）。
- **治理（Govern）**：经营指令，**占人 ✓**、**效果时机=下令瞬间**（不入 `pendingCommands`，执行人月末回城）。前置：城有≥1 在任武将、城金 ≥ 50、执行人体力 ≥ 8，且**非「已正常且防灾=100」**；下令瞬间把状态强制改正常（任何灾害立解）、防灾 `+RandInt(1,4)` 封顶 100、扣体力 8 / 城金 50、占人离城。
- **灾害破坏结算**：月末对异常城按状态破坏表扣减——所有项按当前值百分比扣，新值 `floor(当前×(1−百分比))`，「减半」即 50% 情形（与掠夺 ravage 同向取整）。饥荒/旱灾/水灾/暴动各有不同破坏组合（见 `09-city-disaster` PRD）。
- **灾害生成**：月末仅对正常且有归属城判定——掷 `R=RandInt(0,99)`，`R ≤ 防灾值` 无灾；否则掷灾种 `RandInt(0,4)`：0 旱灾 / 1 水灾 / 2 再掷 `RandInt(0,99)>民忠` 则暴动否则无事 / 3、4 无事。
- **灾害恢复**：月末对异常城判定——饥荒 `粮食>0` 即恢复（不耗 RNG）；旱灾/水灾 `RandInt(0,99)<防灾值` 恢复；暴动 `RandInt(0,99)<民忠` 恢复。月末固定顺序 `09-city-disaster` 起扩展为 `pendingCommands → settle → 回城+体力 → 月份+1 → 登场 → 灾害破坏 → 灾害生成/恢复`（纯追加，不动税粮日历）。
- **太守（Governor）**：**派生概念、零存储字段**（`world/queries.governorOf(state, cityId)`）。某城太守 = 若该城归属君主（`id===city.lordId`）正驻本城则为该君主，否则为本城在任武将（`lordId===city.lordId` 且非俘虏）中**有效智力最高者**（平局取 id 字典序最小）；空城/仅俘虏则无（`null`）。`10-diplomacy` 起仅服务于策反目标判定，无其它下游规则。
- **招揽（Entice）**：外交指令，**占人 ✓**、**效果时机=月末**（入 `pendingCommands` `entice` 分支）。把敌方一名在任非君主武将吸引归己。任意敌城、无需邻接。下令扣体力 20 / 城金 50、占人、入队。月末三关：①智力差关 `RandInt(0,99) > (执行人有效智力 − 目标有效智力)`（**无 +50 安全线**）失败；②忠诚关 `RandInt(0,99) < 目标忠诚` 失败（忠诚 100 必败）；③性格关 `RandInt(0,99) < S` 通过（`S=招揽系数[性格]`）。成功：目标迁入执行人所在城、`lordId=执行人君主`、忠诚 `=RandInt(40,79)`。
- **离间（Alienate）**：外交指令，**占人 ✓**、**效果时机=月末**（`alienate` 分支）。降低敌方一名在任非君主武将忠诚。前置/下令同招揽（体力 20 / 城金 50）。月末三关同招揽但智力差 `+50`、用离间系数；成功仅 `忠诚 −4`（下限 0）、无成功 RNG。低收益高消耗。
- **策反（Instigate）**：外交指令，**占人 ✓**、**效果时机=月末**（`instigate` 分支）。让敌方某城**太守且非君主**者自立为君、带走其城从原势力分裂（君主即太守时不可策反 → 永不触发重选君主）。前置：目标为其城 `governorOf` 且非君主、敌方、非俘虏；下令扣体力 20 / 城金 50、占人、入队。月末三关（智力差 `+50`、策反系数）。成功：目标 `lordId=自身`、其城 `lordId=自身`、该城其余原势力武将改归目标；第三方俘虏不动。
- **劝降（Induce）**：外交指令，**占人 ✓**、**效果时机=月末**（`induce` 分支）。劝敌方**君主**整体投降、吸收其全部城池与臣属。前置：目标为敌方君主、体力 ≥ 10、城金 ≥ 50、**城池压制**（执行人君主城数 ≥ 目标君主城数 ×2）；下令扣体力 10 / 城金 50、占人、入队。月末判定：①目标为玩家君主（`id===playerLordId`）直接失败（防 AI 劝降玩家君主，core 规则非归属校验）；②城池压制重校；③智力差关 `+50`；④君主性格关（君主性格表系数）。成功：目标君主全部城与城内臣属（含君主本人）归执行人君主、散落（不在其城内）武将转在野（`lordId=null`）。
- **外交性格系数**：四张性格难度系数表（`R=RandInt(0,99)`，`R<S` 通过该性格关）。招揽（普通武将表 0..4）`[5,20,30,40,15]`；离间 `[5,30,40,30,50]`；策反 `[5,60,20,10,30]`；劝降（**君主表** 0..4：和平/大义/奸诈/狂人/冒进）`[15,5,20,1,10]`。`08` 招降系数 `[1,3,4,5,2]` 为另一套（值越大越易）；外交系数则直接为通过率百分比。
- **兵种（Troop Type）**：武将的作战类型，六种——骑兵/步兵/弓兵/水军/极兵/玄兵，各有**基础移动力**（5/4/4/5/6/3）。`Officer.troopType` 为**基础兵种**存储字段（fixture 播种，可信、不跑 >105 门槛）。`11-troop-types` 起仅建模 + 装备门槛 + 展示，相克/水战/反骑兵等作战效果留待后续战棋切片。
- **有效兵种（Effective Troop Type）**：派生值（与有效武力/智力同构、不写回 Officer）——以基础兵种为起点，按所持道具「装备先后」依次应用其改兵种结果、合法的后者覆盖前者。战斗等下游临场读取。
- **移动力（Movement）**：派生属性 = 有效兵种基础移动力 + 所持道具移动力加成之和。`11-troop-types` 仅展示、无下游规则（不影响出征/移动/邻接）。
- **改兵种字段 / 移动力加成（Item）**：道具新增两字段。移动力加成（整数）佩戴即计入移动力。改兵种字段 `0..3`：`0` 不改 / `1` 水军 / `2` 玄兵（要求有效智力 > 105）/ `3` 极兵（要求有效武力 > 105）；只能改成水军/玄兵/极兵，骑/步/弓只来自基础兵种。门槛用**有效属性**（含该道具自身加成）严格 > 105，**派生模型下实时判定**：没收改兵种/加成道具后有效兵种随之回退。
- **装备（Equip）**：武将持有的道具是**有序**列表（最多 2 件，记录装备先后），改兵种按此顺序应用、后装备覆盖先装备。`11-troop-types` 起把道具归属从单一 `holder` 重构为武将有序装备列表（不新增「装备/卸下」指令，沿用赏赐/没收为装备/卸下事件）。
- **战斗 / 地图战（Battle）**：`12-battle` 起，出征/被攻打触发的**交互式逐日战棋**，替代 `04-campaign` 的月末速算。读取目标城对应战斗地图，按攻守模式摆双方单位（≤10/方），玩家逐日操作己方单位移动/普攻，最多 30 天，按粮草/主将/全灭/城池格判胜负；分出胜负后把剩余兵力/经验/等级写回 `Officer`，占城/俘虏/重选君主沿用 `04`。确定性、不耗对局 RNG。本切片**不做**天气/技能/人物状态/HP/MP/AI 战斗行动。
- **战斗子状态（BattleState）**：战斗是 `apply(state,action)` 之上的交互式子对局——出征/被攻打时**暂停经营月末**，进入独立 `BattleState` + 一组战斗专属 action（选单位/移动/攻击/休息/查看/撤退/天结束等），分胜负后写回 `GameState` 再续月末。形态留 `dev.md`。
- **攻守模式 / 主将（Battle Mode / Commander）**：战斗仅两模式——**玩家进攻**（玩家攻敌城）/ **玩家防守**（敌攻玩家城）；开战后双方武将分属「玩家方」「对手方」。主将 = 防守方第一名有效武将（进攻=对手方主将、防守=玩家方主将），被击溃立即触发胜负。
- **战斗派生属性**：单位由等级/有效武力/有效智力/体力/有效兵种/装备派生出**攻击力/防御力/移动力**。`基础攻击=floor(武力×(等级+10)×兵种攻击系数)`、`基础防御=floor(智力×(等级+10)×兵种防御系数)`；攻防再经地形折减档与地形防御系数修正。**HP/MP `12-battle` 不做**（仅供显示/技能，技能已砍）。扣的是**兵力**不是 HP，兵力归零即**击溃**。
- **战斗地图（Battle Map）**：领域数据，每座城对应一张固定地图（地形格/城池格/出生点）。棋盘 **32×32**；`12-battle` 先手写**少量模板地图**（fixture 播种）供所有可被攻城复用/指派，不逐城独立画。
- **地形（Terrain）**：8 种格——草地/平原/山地/森林/村庄/城池/营寨/河流；同时影响**移动消耗**（按兵种）、**攻防折减档**（无/中/重/极重 = `/1 /2 /4 /8`，按兵种）、**防御系数**（按地形）。
- **城池格（City Tile）**：战斗地图上的胜负点（也是地形、防御系数 1.5）。玩家进攻进入即玩家胜；玩家防守对手进入即玩家败。
- **接敌停步区（Zone of Control）**：敌方存活单位上下左右四格形成的移动压制区——进入后剩余移动力压到 1、不可继续穿越。属地图/移动规则，非人物状态。
- **战场粮草（Battle Provisions）**：双方进入战斗各自带入、独立参与战斗结算的粮（攻方默认=随军粮草、守方默认=目标城开战快照城粮）；每天行动后 `每日耗粮=floor(sqrt(本方未击溃单位总兵力)/3)`，归零即按胜负表判负。战后剩余处理沿用 `04`（胜并入/败损失）。
- **经验 / 升级（Experience / Level Up）**：`Officer.experience` 为 `12-battle` 新增存储字段。普攻造成兵力变化时行动者得经验（`伤害经验=floor(sqrt(实际兵力变化)/4)`，按等级差加成、击溃额外给经验）；经验满 100 扣 100、等级+1，一次只升一级；战后持久化到 `Officer`。
- **击溃（Routed）**：单位当前兵力归零的战斗内派生状态——退出后续行动、触发胜负检查（主将击溃/全灭/城池胜负点）。`13-battle-skills` 起等价于人物状态「死亡」。
- **技能 / 计谋（Skill）**：`13-battle-skills` 起的战斗内主动能力，施放扣 MP、过成功率判定后结算（扣兵/恢复/破粮/施加状态/改天气/情报/围攻）。共 30 条，本作纳入 27 条（去 `21 遁甲 / 26 潜踪 / 28 急行`）。
- **MP / 技能点**：单位施法资源，战斗内派生、**仅活在 `BattleUnit` 快照、战后不写回 Officer**。初始 `MP=floor((floor(智力×80/100)+floor(sqrt(武力)/2)+等级)×体力/100)`（吃有效武力/智力），即上限；休息 +1 封顶上限；援兵/援军/陷阱不回 MP。本作**不建 HP 模型**，扣的仍是兵力。
- **天气（Weather）**：战斗内全局状态，五种——晴/阴/风/雨/雹。开战=风，第 1 天开头先刷新一次、此后每天日循环开头刷新（均匀 `RandInt(0,4)`）；`天变(22)` 成功可改写；冰雹下多数技能不可用（由倍率列 `0` 表达）。
- **人物状态（Battle Status）**：`BattleUnit` 战斗内状态，`13-battle-skills` 建模——正常/混乱（不能行动）/禁咒（不能施法）/定身（移动力降为 1）/奇门（可穿越接敌停步区）/石阵（不能行动且每天损当前兵力 `floor(/8)`）/死亡（=击溃）。`遁甲/潜踪` 无技能可施加、不建模。每天开头对非死亡单位掷 `RandInt(0,59)<floor(有效智力/2)`：混乱/禁咒/定身/石阵成功复正常、奇门/潜踪失败复正常、正常不变、死亡跳过。
- **默认技能 / 个人技能 / 君主技能**：单位可用技能三来源。默认技能按兵种自带、随等级解锁 `已解锁数=floor(默认数×等级/21)+1`（按列表顺序、封顶默认数）；个人技能=`Officer` fixture 字段，恒可用、不受公式限制；君主技能=派生（`lordId===自身` 额外得 `谍报(30)`），不受公式限制。
- **施法能力 / 目标抗性 / 成功率**：`施法能力=施法者有效智力+等级+5`、`目标抗性=目标有效智力+等级+5`；`R=RandInt(0,目标抗性+19)`，`R≤floor(施法能力/2)` 成功，否则失败（失败仍扣 MP、无效果）。
- **技能范围掩码（9×9）**：每技能一张独立 9×9 掩码（中心=施法者），`1` 格才可能被选为目标；目标还需过阵营+目标地形+目标兵种（倍率≠0）校验；自身/无目标技能（天变/谍报）不走选格。掩码原文见根目录 `appendix-a-skill-ranges.md`。
- **倍率链**：技能兵力/破粮效果的固定结算序——`基础值 → ×天气% → ×目标兵种% → ×目标地形% → ×施法者地形%`，每步 `floor`；倍率 `0` 即不可用/无效，`50/60/80` 为减效。
- **破粮**：火箭(11) 的效果——按倍率链扣对手方战场粮草 `provisions`（`min(最终破粮值, 对手战场粮草)`），不立即判负，下一天开头查粮草时才可能触发失败。
- **围攻（27）**：成功后目标上下左右相邻的**施法者方**单位逐个对目标执行一次普攻（复用普攻结算、可致击溃）；非范围伤害。
- **谍报（30）**：君主技能，成功后揭示对手战场粮草；core 全可见，落地为 UI 揭示，core 仅判成功、不改状态。
- **战后处理（Campaign Aftermath）**：`14-campaign-aftermath` 起，`concludeBattle` 写回兵力/经验/等级后跑的完整流程——胜方回城 → 占城 → 败军逐人命运 → 君主遭劫（重选/灭亡）→ 城市战损 → 粮草合并。收敛在 `military/resolveCampaignOutcome`，**消耗 `GameState.rng`**（修订 `12` 确定性红线）。仅服务 battle 路径；`04` 的 `executeCampaign` 速算为死代码已删。
- **胜方回城**：战后把胜方所有存活单位（officer 未被战死即算活，含 0 兵被击溃者）落入本场目标城。
- **败军处理（Defeated Army）**：对**败方每名参战武将**逐一掷骰定命运——①`RandInt(0,99) > 有效智力` 直接被俘；②否则取其势力现存城随机落城逃跑成功，无城则逃跑失败；③逃跑失败者若为自家君主记为遭劫君主，再 `RandInt(0,99)===0` 战死、否则被俘。占城在败军处理之前，故逃跑只逃向势力其余存活城。
- **被俘 / 逃跑 / 战死**：败军三种归宿。被俘=lordId 不变、cityId 改目标城、兵清零（派生成俘虏）；逃跑=随机落该势力一座存活城、保留兵；战死=两槽道具并入目标城仓库且 `discovered=true`、officer 从 `officers` 永久删除。
- **占城（沿用 04、keep 模型）**：攻方胜后**仅翻** `city.lordId` 为攻方君主，不改城内武将 lordId/cityId/兵；原守军派生成俘虏。守军的差异化命运全部收敛到败军处理（不在占城里把守军转在野/清零）。
- **遭劫君主**：本场战斗被俘或战死的君主。识别后进入君主遭劫链：势力有城→立新君，无城→灭亡。
- **重选君主（深化）**：`14` 起新君忠诚强制 100；势力所有城与未被俘武将归属切新君。AI 势力自动取有效智力最高（平局 id 最小）；**玩家势力改为挂起手动选**（待选新君）。玩家君主死/俘不直接 Game Over。
- **待选新君（Pending Succession）**：玩家君主遭劫且势力仍有城时的**窄暂停态**（类比 `activeBattle`，存遭劫 lordId + 候选武将）——`GameState` 非空字段=月末挂起、等玩家从剩余武将手动选一位；core 纯同步、由新 action 兑现后换主（忠诚 100）并续跑月末。不泛化为通用 pendingChoice。
- **城市战损**：战后无条件对目标城扣发展——农业/商业/金各 `floor(×0.95)`、民忠 `floor(×0.90)`，胜负皆受。规则身份、内联收敛于 `world/city.ts`（与灾害破坏同位）。
- **粮草合并（覆盖式）**：战后目标城 `food` 被**覆盖**为双方剩余战场粮草之和（`playerProvisions + opponentProvisions`），非累加；取代 `04` 的"胜则 food += 随军粮草"。
