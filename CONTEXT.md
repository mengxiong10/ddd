# CONTEXT（术语表）

- **剧本（Scenario）**：一组具有确定历史时期、初始城池、武将与道具配置，可供玩家选择并创建新对局的开局数据。
- **存档（Save）**：一局已开始对局在某一时刻的完整状态快照，用于恢复并继续该局；不同于用于创建新对局的剧本。
- **君主（Lord）**：一方势力的统领，**本身也是一名武将**。城与武将通过 `lordId` 归属于某位君主；不设独立的「势力」实体。每局一位君主由玩家控制，其余现存君主均由 AI 控制；玩家方由 `playerLordId` 标记。
- **城 / 城池（City）**：地图上的据点；同一座城在不同剧本中保持同一身份与唯一 id，剧本只改变其时期状态。通过可空的 `lordId` 归属某位君主。持有属性 `农业`、`商业` 及各自的**城级上限**，城级资源 `金`、`粮`，以及驻城武将。
- **世界地图坐标（World Map Position）**：城池在 `12×9` 战略地图中的固定整数格位，不表示屏幕像素或归一化位置，也不决定城池邻接；开战时用于派生出发城相对目标城的八方向，以动态计算攻方出生阵形。
- **空城（Unowned City）**：`lordId=null`、尚无势力归属的城池；不是独立势力，不执行 AI 经营，可被其他势力占领。
- **武将（Officer）**：执行经营与战斗行为的人，通过 `lordId` 归属某位君主（君主本人 `lordId` 指向自身）；同一历史武将在不同剧本中保持同一身份与唯一 id。
- **智力（Intelligence）**：武将基础属性，参与经营、外交、带兵与战斗公式；所有规则读取「有效智力」（基础 + 道具加成）。
- **体力（Stamina）**：武将可消耗资源，执行经营指令时扣减；达不到阈值不能下令。
- **在任武将**：当前位于该城、归属该城势力、非俘虏且未被指令占用，可被下令的武将；在野与未登场武将不属于在任武将。
- **农业（Agriculture）**：城属性，决定收粮量；通过「开垦」提升，有上限。
- **商业（Commerce）**：城属性，决定收税量；通过「招商」提升，有上限。
- **金（Gold）**：城级资源，来自收税，并用于支付经营指令开销。
- **粮（Food）**：城级资源，来自收粮。
- **经营指令**：玩家或 AI 在下令阶段对城市、武将或目标势力发起的经营行为。
- **开垦**：经营指令，提升本城 `农业`，占用一名武将。
- **招商**：经营指令，提升本城 `商业`，占用一名武将。
- **占人 / 占用中（Busy）**：一条指令在 `pendingCommands` 中引用某武将，即该武将本月被占用、不可再次下令；占用由 `isBusy` 派生、不存字段，入队即占用、出队即释放，通常不改变 `cityId`，仅移动与出征在月末结算时改写位置。
- **月度推进**：游戏以「月」为最小时间单位推进，每月经历下令与月末结算。
- **收粮（Harvest）**：每年 6 月、10 月各结算一次，每城 `本次收粮 = floor(农业 / 4)` 计入该城 `粮`。
- **收税（Tax）**：每年 3、6、9、12 月（每季度）各结算一次，每城 `本次收税 = floor(商业 / 2)` 计入该城 `金`。
- **兵 / 兵力（Troops）**：可作战的人力，分两处存放——城级的「后备兵」与武将级的「武将兵」，二者是各自独立的真实状态（非冗余）。
- **后备兵（Reserve Troops）**：城级未编队兵力；征兵注入到此，分配在后备兵与武将兵之间调度。
- **武将兵（Officer Troops）**：某武将当前带领的兵力，受其「带兵量上限」约束。
- **民忠（Loyalty）**：城属性，范围 `[0, 100]`，反映民心。决定征兵上限（民忠×20）。**自 `03-plunder-scout` 起可被掠夺减半**（不再纯静态）；**自 `06-city-commands` 起可由「出巡」小幅回升**（`+RandInt(1,4)` 封顶 100），仍无自然涨跌机制。
- **人口（Population）**：城属性，由「出巡」+100 提升（fixture 播种初值）。**`06-city-commands` 切片仅为展示状态、无任何下游规则**（不影响征兵/收粮/收税），其作用留待后续切片。
- **武力（Force）**：武将静态属性，参与带兵量公式。**自 `05-items` 起公式改吃「有效武力」（基础 + 道具加成）**。
- **等级（Level）**：武将属性，参与带兵量与战斗公式；可由战斗经验升级，非玩家武将还可受 AI 月末升级规则影响。
- **带兵量上限（Troop Capacity）**：某武将可带兵的上限，`等级×100 + 武力×10 + 智力×10`。
- **征兵（Recruit）**：经营指令，**占人**。把民忠与金钱转化为城后备兵；下令时后备兵 += N、执行人体力 -12、城金 -= `ceil(N/10)`，可征上限 `min(民忠×20, 金×10)`。
- **分配（Allocate）**：指令，**不占人**、立即生效。在城后备兵与某武将现有兵之间重分配，受该武将带兵量上限约束；不扣体力/金。
- **掠夺（Plunder）**：经营指令，**占人**，**效果时机=月末**。牺牲本城发展换粮草：下令时扣执行人体力 12 并入队；月末（先于收粮/收税）破坏本城（农业/商业/民忠各 `floor(/2)`），并给本城 `粮 +=(智力+武力)×5`、`金 +=(智力+武力)×2`。
- **侦察（Scout）**：指令，**占人**，**效果时机=下令瞬间**。花本城金 20、执行人体力 10，立即取得一座**非己方、非本城**目标城的详情；指令入队作为占用标记，月末为空操作并出队。
- **效果时机**：一条指令的实质效果在何时结算，分为「下令瞬间」与「月末」；即时占人指令仍会入队保留占用标记。
- **月末待执行队列（pendingCommands）**：本月全部占人指令的判别式队列，也是武将占用状态的唯一真相源；月末先按入队序处理并移除所有非出征项（即时类为空操作），再按入队序处理出征，遇战斗或玩家选择可暂停并保留未结算出征。
- **出征（Campaign）**：军事指令，**占人 ✓**（一次最多 10 名武将），**效果时机=月末**。从本城派最多 10 名在任武将携随军粮草进攻一座相邻的敌城或空城；下令时扣本城随军粮草并入队，月末按出征分流结算，战后由胜方进驻与败军命运改写武将位置。
- **随军粮草（Provisions）**：出征下令时从本城城粮扣除的携带粮（整数，1 ~ 本城城粮），开战后成为攻方战场粮草；战后目标城粮被双方剩余战场粮草之和覆盖。
- **相邻 / 邻接（Adjacency）**：城与城之间的**无向**地图邻接关系（`04-campaign` 起引入的地图拓扑）。出征的"路径可达"= 目标城与出发城相邻；本切片不做多跳路径搜索。
- **占领 / 城易主**：攻方胜后目标城 `lordId` 改为攻方，连同其农业/商业/资源/后备兵整体易主；原守军武将就地成俘虏。
- **俘虏（Captive）**：**派生状态**，非存储字段——当 `officer.lordId !== 所在城.lordId`（武将自身归属 ≠ 所在城归属）时即为俘虏；俘虏不可被指令指派，不计入在任武将、守军或重选君主候选。
- **道具（Item）**：带唯一 id 的实体；同一道具在单个剧本中至多存在一件，原始时期数据出现重复引用时仅保留其中一处。含 `武力加成`、`智力加成`（非负整数，可为 0）。归属为**单一真相源**——已登场道具属于某城或某武将，独立未登场道具暂无归属；城/武将的道具列表均由归属派生，不存第二份。未登场武将可以持有已存在道具；该道具随持有者一起不可操作，武将登场后自然可用。每名武将最多持 2 件（不变量）。
- **有效武力 / 有效智力（Effective Force / Intelligence）**：派生属性——基础武力/智力 + 所持道具对应加成之和。`05-items` 起带兵量上限、开垦/招商增量等所有武力/智力公式统一吃有效值。
- **武将忠诚（Loyalty, 武将级）**：武将级属性，范围 `[0, 100]`，**与城的「民忠」是不同事实**；君主忠诚派生恒为 100，普通武将忠诚可被赏赐、没收、招降与外交规则读取或改写。
- **赏赐（Reward）**：指令，**目标是武将（非执行人）、不占人、效果时机=下令瞬间**，无金/体力成本。把一件属于「目标武将所在城」的道具转给该城己方非俘虏武将（武将道具数 < 2）；成功后道具归属改为该武将、其忠诚 +8（封顶 100）。非法即 no-op。
- **没收（Confiscate）**：指令，**目标是武将（非执行人）、不占人、效果时机=下令瞬间**，无金/体力成本。把目标（己方非俘虏）武将所持的一件道具收回其所在城；成功后道具归属改为该城、其忠诚 −20（下限 0）。非法即 no-op。
- **出巡（Patrol）**：经营指令，**占人 ✓**，**效果时机=下令瞬间**。花本城金 50、执行人体力 8，立即提升本城民忠（`+RandInt(1,4)`，封顶 100）与人口（+100）；指令入队作为占用标记，月末为空操作并出队。
- **宴请（Banquet）**：指令，**占人 ✗**、**效果时机=下令瞬间**。花本城金 100，给该城一名**在任**武将体力 +50（封顶 100），非君主额外忠诚 +1（封顶 100）；不占用其本月行动（被宴请者仍可被其他指令派遣）。
- **移动（Move）**：经营指令，**占人 ✓**，**效果时机=月末**（进 pendingCommands，`move` 分支）。把本城一名在任武将搬到己方另一座城（非本城、同势力，不考虑路径）；不扣体力/金。是占人**例外**（同出征）：月末武将落到目标城（`cityId` 改写）、不回出发城。
- **输送（Transport）**：经营指令，**占人 ✓**，**效果时机=月末**（进 pendingCommands，`transport` 分支）。把粮/金/**后备兵**送往己方另一座城（非本城、同势力，不考虑路径）；下令时扣执行人体力 8 并从出发城扣掉所送资源，月末 **80% 送达**、**20% 永久损失**，执行人位置不变。
- **交易（Trade）**：经营指令，**占人 ✓**，**效果时机=下令瞬间**。在本城以固定单价买卖粮食兑换城金——买入 5 金/粮（上限 `floor(城金/5)`）、卖出 2 金/粮（上限本城所有粮，城金不设上限）；扣执行人体力 12，指令入队作为占用标记，月末为空操作并出队。
- **重选君主**：势力君主遭劫且仍有城与合格候选时触发；AI 势力自动选择有效智力最高者（平局 id 最小），玩家势力暂停月末并由玩家选择，新君继承该势力城池与未被俘武将。
- **势力灭亡**：某势力君主被俘且已无任何城池时，该势力消亡（剩余武将均为俘虏）。本切片不据此做统一全国的全局胜负判定。
- **登场（Debut）**：未登场武将/道具满足年份条件后于月末进入某城的运行时事件；发生在月份推进之后、灾害之前，按新年月判定。
- **登场年（Debut Year）**：未登场实体现身的年份（只到年、不含月）；月末凡 `当前年 ≥ 登场年` 者登场。
- **未登场（Unappeared）**：已属于当前对局、但尚未出现到任何城市的武将或道具。未登场实体不能参与城市、指令、战斗或搜寻等活动，满足登场条件后才进入城市；未登场武将可预先持有道具，但两者均不可操作。
- **登场条件（Appearance Conditions）**：与实体进入世界并被玩家发现相关的一组条件，沿用原版的出生时间、伯乐与目标城三部分语义。出生时间与目标城决定何时、在何处进入城市，目标城未指定时随机选择；伯乐决定进入城市后由谁发现或招募。伯乐不是进入城市的前置条件。
- **无主 / 在野（Wandering）**：武将不归属任何君主——`Officer.lordId` 可空，`null` 即无主，统一覆盖「未登场」与「在野（已登场未被招募）」。在野武将对玩家隐藏：不计入「在任武将」、不参与守城兵力、不可被指令指派、不被判为俘虏，仅可经搜寻招募。归属武将 `lordId` 非空。
- **隐匿武将 / 隐藏道具**：在野武将（`lordId===null`）与未发现道具（`discovered===false`）的统称——已登场但需经搜寻方能被发现/招募。
- **已发现 / 未发现道具（Discovered）**：`Item.discovered` 字段。fixture 既有道具恒 `true`；登场道具落城时 `false`，对玩家隐藏且不可被赏赐，经搜寻发现后置 `true`。
- **伯乐（Recruiter）**：武将/道具登场条件中的指定招募者（无指定时为空）。决定搜寻时该目标能否被某执行人招募/发现。
- **搜寻（Search）**：经营指令，**占人 ✓**，**效果时机=月末**。前置：城有≥1 在任武将、执行人体力 ≥ 8、不需金钱；下令扣体力 8、占人、入队。月末四分支各 1/4：无事 / 尝试发现武将或道具 / 获得金钱 / 获得粮食（金钱、粮食上限均 `max(10, 有效智力×2)`，封顶 30000）。
- **发现判定**：搜寻「发现」分支的内部判定——先 `RandInt(0,149) < 有效智力` 过筛（否则无事），再 1/2 决定找武将或道具；候选为空当作无事。
- **招募（Recruit）**：搜寻发现并收编在野武将——伯乐=执行人必成、伯乐=null 看 `RandInt(0,109)<有效智力`、伯乐=他人必败；成功后该武将归执行人君主、忠诚 `RandInt(70,99)`、cityId 保持本城、troops 为 0。
- **性格（Personality）**：武将存储字段，单值 `0..4`。**两套解读同一值**：君主（`lordId===自身`）按**君主性格表**（0 和平 / 1 大义 / 2 奸诈 / 3 狂人 / 4 冒进），非君主按**普通武将性格表**（0 忠义 / 1 大志 / 2 贪财 / 3 怕死 / 4 卤莽）；随身份自动切换，普通武将性格参与招降/外交，君主性格参与外交与 AI 策略。
- **招降（Suborn）**：指令，**占人 ✓**、**效果时机=月末**（入 `pendingCommands` `suborn` 分支）。让本城俘虏归顺本势力。前置：城有俘虏（目标）+ 在任武将（执行人）、执行人体力 ≥ 15、城金 ≥ 100；下令扣体力 15 / 城金 100、占人、入队。月末四关判定：①智力差关 `RandInt(0,99) > 执行人有效智力 − 目标有效智力 + 50` 失败；②按目标性格取招降系数；③读原始忠诚 `L0`、扣 `floor(L0/10)`（持久，含失败）、`L0>60` 直接失败；④终判 `RandInt(0,99) < floor(L/S)` 失败否则成功。成功：俘虏 `lordId` 改归执行人君主、忠诚 `RandInt(40,79)`。
- **招降性格系数（S）**：目标普通武将性格对应的招降难度系数——忠义 1 / 卤莽 2 / 大志 3 / 贪财 4 / 怕死 5（按 `personality` 取系数表 `[1,3,4,5,2]`）；系数越大终判失败阈值越小、越易招降。
- **处斩（Behead）**：指令，**占人 ✗**、**效果时机=下令瞬间**。永久消除本城俘虏：目标所持道具退回本城（`holder=城`）并标记 `discovered=true`，该武将从 `state.officers` 永久删除；无 RNG / 体力 / 金成本。
- **流放（Banish）**：指令，**占人 ✗**、**效果时机=下令瞬间**。把本城一人（在任武将或俘虏皆可，**非在任君主**；被俘君主可流放）变在野（`lordId=null`），**其道具随人带走**（`holder` 仍指向该武将、不退城、不改 `discovered`），并在**全部城**中等概率随机选一座（可能含原城）落城（`cityId` 改写）；消费 RNG。
- **城市状态 / 灾害状态（City Status）**：`City` 存储字段，单值 `正常 / 饥荒 / 旱灾 / 水灾 / 暴动`，初始正常（fixture 播种）。`09-city-disaster` 起每月可生成新灾、按状态破坏、并经自然或治理恢复。
- **防灾值（Disaster Prevention）**：`City` 存储字段，`[0, 100]` 整数（fixture 播种）。越高越不易发新灾、越快从旱灾/水灾恢复；由「治理」`+RandInt(1,4)` 封顶 100 提升。与民忠并列为「防灾 vs 防乱」两条投入线（民忠降暴动概率/加速暴动恢复）。
- **治理（Govern）**：经营指令，**占人 ✓**、**效果时机=下令瞬间**。前置：城有≥1 在任武将、城金 ≥ 50、执行人体力 ≥ 8，且**非「已正常且防灾=100」**；下令瞬间把状态强制改正常、防灾 `+RandInt(1,4)` 封顶 100、扣体力 8 / 城金 50，指令入队作为占用标记，月末为空操作并出队。
- **灾害破坏结算**：月末对异常城按状态破坏表扣减——所有项按当前值百分比扣，新值 `floor(当前×(1−百分比))`，「减半」即 50% 情形（与掠夺 ravage 同向取整）。饥荒/旱灾/水灾/暴动各有不同破坏组合（见 `09-city-disaster` PRD）。
- **灾害生成**：月末仅对正常且有归属城判定——掷 `R=RandInt(0,99)`，`R ≤ 防灾值` 无灾；否则掷灾种 `RandInt(0,4)`：0 旱灾 / 1 水灾 / 2 再掷 `RandInt(0,99)>民忠` 则暴动否则无事 / 3、4 无事。
- **灾害恢复**：月末尾段对异常城判定——饥荒 `粮食>0` 即恢复（不耗 RNG）；旱灾/水灾 `RandInt(0,99)<防灾值` 恢复；暴动 `RandInt(0,99)<民忠` 恢复；灾害在待执行指令、收粮/收税、体力恢复、月份推进与登场之后运行。
- **太守（Governor）**：**派生概念、零存储字段**（`world/queries.governorOf(state, cityId)`）。某城太守 = 若该城归属君主（`id===city.lordId`）正驻本城则为该君主，否则为本城在任武将（`lordId===city.lordId` 且非俘虏）中**有效智力最高者**（平局取数字 id 最小）；空城/仅俘虏则无（`null`）。`10-diplomacy` 起仅服务于策反目标判定，无其它下游规则。
- **招揽（Entice）**：外交指令，**占人 ✓**、**效果时机=月末**（入 `pendingCommands` `entice` 分支）。把敌方一名在任非君主武将吸引归己。任意敌城、无需邻接。下令扣体力 20 / 城金 50、占人、入队。月末三关：①智力差关 `RandInt(0,99) > (执行人有效智力 − 目标有效智力)`（**无 +50 安全线**）失败；②忠诚关 `RandInt(0,99) < 目标忠诚` 失败（忠诚 100 必败）；③性格关 `RandInt(0,99) < S` 通过（`S=招揽系数[性格]`）。成功：目标迁入执行人所在城、`lordId=执行人君主`、忠诚 `=RandInt(40,79)`。
- **离间（Alienate）**：外交指令，**占人 ✓**、**效果时机=月末**（`alienate` 分支）。降低敌方一名在任非君主武将忠诚。前置/下令同招揽（体力 20 / 城金 50）。月末三关同招揽但智力差 `+50`、用离间系数；成功仅 `忠诚 −4`（下限 0）、无成功 RNG。低收益高消耗。
- **策反（Instigate）**：外交指令，**占人 ✓**、**效果时机=月末**（`instigate` 分支）。让敌方某城**太守且非君主**者自立为君、带走其城从原势力分裂（君主即太守时不可策反 → 永不触发重选君主）。前置：目标为其城 `governorOf` 且非君主、敌方、非俘虏；下令扣体力 20 / 城金 50、占人、入队。月末三关（智力差 `+50`、策反系数）。成功：目标 `lordId=自身`、其城 `lordId=自身`、该城其余原势力武将改归目标；第三方俘虏不动。
- **劝降（Induce）**：外交指令，**占人 ✓**、**效果时机=月末**（`induce` 分支）。劝敌方**君主**整体投降、吸收其全部城池与臣属。前置：目标为敌方君主、体力 ≥ 10、城金 ≥ 50、**城池压制**（执行人君主城数 ≥ 目标君主城数 ×2）；下令扣体力 10 / 城金 50、占人、入队。月末判定：①目标为玩家君主（`id===playerLordId`）直接失败（防 AI 劝降玩家君主，core 规则非归属校验）；②城池压制重校；③智力差关 `+50`；④君主性格关（君主性格表系数）。成功：目标君主全部城与城内臣属（含君主本人）归执行人君主、散落（不在其城内）武将转在野（`lordId=null`）。
- **外交性格系数**：四张性格难度系数表（`R=RandInt(0,99)`，`R<S` 通过该性格关）。招揽（普通武将表 0..4）`[5,20,30,40,15]`；离间 `[5,30,40,30,50]`；策反 `[5,60,20,10,30]`；劝降（**君主表** 0..4：和平/大义/奸诈/狂人/冒进）`[15,5,20,1,10]`。`08` 招降系数 `[1,3,4,5,2]` 为另一套（值越大越易）；外交系数则直接为通过率百分比。
- **兵种（Troop Type）**：武将的作战类型，六种——骑兵/步兵/弓兵/水军/极兵/玄兵，各有基础移动力、战斗攻防系数、地形适性与默认技能；`Officer.troopType` 只存基础兵种。
- **有效兵种（Effective Troop Type）**：派生值（与有效武力/智力同构、不写回 Officer）——以基础兵种为起点，按所持道具「装备先后」依次应用其改兵种结果、合法的后者覆盖前者。战斗等下游临场读取。
- **移动力（Movement）**：派生属性 = 有效兵种基础移动力 + 所持道具移动力加成之和，用于战场单位移动；不影响战略层移动或城池邻接。
- **改兵种字段 / 移动力加成（Item）**：道具新增两字段。移动力加成（整数）佩戴即计入移动力。改兵种字段 `0..6`：`0` 不改 / `1` 水军 / `2` 玄兵（要求有效智力 > 105）/ `3` 极兵（要求有效武力 > 105）/ `4` 骑兵 / `5` 步兵 / `6` 弓兵；仅玄兵、极兵有属性门槛。门槛用**有效属性**（含该道具自身加成）严格 > 105，**派生模型下实时判定**：没收改兵种/加成道具后有效兵种随之回退。
- **装备（Equip）**：武将持有的道具通过 `Item.holder={kind:'officer',officerId,equipSeq}` 表达，最多 2 件并按 `equipSeq` 排序；改兵种依次应用、后装备覆盖先装备，赏赐/没收分别承担装备/卸下语义。
- **战斗 / 地图战（Battle）**：玩家参与的出征触发的**交互式逐日战棋**；读取目标城战斗地图，双方各最多 10 名武将，玩家操作己方、战斗 AI 操作对手方，最多 30 天，按粮草/主将/全灭/城池格判胜负；天气、技能与人物状态会消费 `GameState.rng`，同 seed 可复现。
- **战斗子状态（BattleState）**：战斗是 `apply(state,action)` 之上的交互式子对局；出征/被攻打时暂停经营月末，进入独立战斗状态，经战斗 action 推进，分胜负后写回 `GameState` 并继续同一月末。
- **攻守模式 / 主将（Battle Mode / Commander）**：战斗仅两模式——**玩家进攻**（玩家攻敌城）/ **玩家防守**（敌攻玩家城）；开战后双方武将分属「玩家方」「对手方」。防守方选取按「太守领衔 + 其余兵力降序」。**攻守各有主将**：攻方主将=出征名单首位、守方主将=防守方首位（即太守）；主将开战定格、不随减员漂移，**任一方主将被击溃立即触发该方失败**。
- **战斗派生属性**：单位由等级/有效武力/有效智力/体力/有效兵种/装备派生出**攻击力/防御力/移动力/MP**；本作不建 HP，伤害扣兵力，兵力归零即进入死亡状态。
- **战斗地图（Battle Map）**：具有稳定数字身份 `1..7` 的 **32×32** 地形与关键格布局，每座城跨所有剧本固定对应其中一张。
- **地形（Terrain）**：8 种格——草地/平原/山地/森林/村庄/城池/营寨/河流；同时影响**移动消耗**（按兵种）、**攻防折减档**（无/中/重/极重 = `/1 /2 /4 /8`，按兵种）、**防御系数**（按地形）。
- **城池格（City Tile）**：战斗地图上的胜负点（也是地形、防御系数 1.5）。玩家进攻进入即玩家胜；玩家防守对手进入即玩家败。
- **接敌停步区（Zone of Control）**：敌方存活单位上下左右四格形成的移动压制区——进入后剩余移动力压到 1、不可继续穿越。属地图/移动规则，非人物状态。
- **战场粮草（Battle Provisions）**：双方进入战斗各自带入、独立参与战斗结算的粮（攻方默认=随军粮草、守方默认=目标城开战快照城粮）；每天行动后 `每日耗粮=floor(sqrt(本方未击溃单位总兵力)/3)`，归零即按胜负表判负。战后剩余处理沿用 `04`（胜并入/败损失）。
- **经验 / 升级（Experience / Level Up）**：`Officer.experience` 为 `12-battle` 新增存储字段。普攻造成兵力变化时行动者得经验（`伤害经验=floor(sqrt(实际兵力变化)/4)`，按等级差加成、击溃额外给经验）；经验满 100 扣 100、等级+1，一次只升一级；战后持久化到 `Officer`。
- **击溃（Routed）**：单位兵力归零时进入人物状态「死亡」并退出后续行动；`BattleUnit.status='dead'` 是击溃的唯一真相，不另存 `routed` 字段。
- **技能 / 计谋（Skill）**：`13-battle-skills` 起的战斗内主动能力，施放扣 MP、过成功率判定后结算（扣兵/恢复/破粮/施加状态/改天气/情报/围攻）。共 30 条，本作纳入 27 条（去 `21 遁甲 / 26 潜踪 / 28 急行`）。
- **MP / 技能点**：单位施法资源，战斗内派生、**仅活在 `BattleUnit` 快照、战后不写回 Officer**。初始 `MP=floor((floor(智力×80/100)+floor(sqrt(武力)/2)+等级)×体力/100)`（吃有效武力/智力），即上限；休息 +1 封顶上限；援兵/援军/陷阱不回 MP。本作**不建 HP 模型**，扣的仍是兵力。
- **天气（Weather）**：战斗内全局状态，五种——晴/阴/风/雨/雹。开战=风，第 1 天开头先刷新一次、此后每天日循环开头刷新（均匀 `RandInt(0,4)`）；`天变(22)` 成功可改写；冰雹下多数技能不可用（由倍率列 `0` 表达）。
- **人物状态（Battle Status）**：`BattleUnit` 战斗内状态——正常/混乱（不能行动）/禁咒（不能施法）/定身（移动力降为 1）/奇门（可穿越接敌停步区）/石阵（不能行动且每天损当前兵力 `floor(/8)`）/死亡（=击溃）。每天开头对非死亡异常单位掷 `RandInt(0,59)<floor(有效智力/2)`：混乱/禁咒/定身/石阵成功复正常，奇门失败复正常。
- **默认技能 / 个人技能 / 君主技能**：单位可用技能三来源。默认技能按兵种自带、随等级解锁 `已解锁数=floor(默认数×等级/21)+1`（按列表顺序、封顶默认数）；个人技能=`Officer` fixture 字段，恒可用、不受公式限制；君主技能=派生（`lordId===自身` 额外得 `谍报(30)`），不受公式限制。
- **施法能力 / 目标抗性 / 成功率**：`施法能力=施法者有效智力+等级+5`、`目标抗性=目标有效智力+等级+5`；`R=RandInt(0,目标抗性+19)`，`R≤floor(施法能力/2)` 成功，否则失败（失败仍扣 MP、无效果）。
- **技能范围掩码（9×9）**：每技能一张独立 9×9 掩码（中心=施法者），`1` 格才可能被选为目标；目标还需过阵营+目标地形+目标兵种（倍率≠0）校验；自身/无目标技能（天变/谍报）不走选格。掩码原文见根目录 `appendix-a-skill-ranges.md`。
- **倍率链**：技能兵力/破粮效果的固定结算序——`基础值 → ×天气% → ×目标兵种% → ×目标地形% → ×施法者地形%`，每步 `floor`；倍率 `0` 即不可用/无效，`50/60/80` 为减效。
- **破粮**：火箭(11) 的效果——按倍率链扣对手方战场粮草 `provisions`（`min(最终破粮值, 对手战场粮草)`），不立即判负，下一天开头查粮草时才可能触发失败。
- **围攻（27）**：成功后目标上下左右相邻的**施法者方**单位逐个对目标执行一次普攻（复用普攻结算、可致击溃）；非范围伤害。
- **谍报（30）**：君主技能，成功后揭示对手战场粮草；core 全可见，落地为 UI 揭示，core 仅判成功、不改状态。
- **战后处理（Campaign Aftermath）**：地图战或 AI 速算分出胜负后的统一流程——胜方进驻 → 占城 → 败军逐人命运 → 君主遭劫（重选/灭亡）→ 城市战损 → 粮草合并；收敛在 `military/aftermath.resolveCampaignOutcome` 并消耗 `GameState.rng`。
- **胜方进驻**：战后把胜方所有未战死单位（含兵力归零的被击溃者）放入本场目标城。
- **败军处理（Defeated Army）**：对**败方每名参战武将**逐一掷骰定命运——①`RandInt(0,99) > 有效智力` 直接被俘；②否则取其势力现存城随机落城逃跑成功，无城则逃跑失败；③逃跑失败者若为自家君主记为遭劫君主，再 `RandInt(0,99)===0` 战死、否则被俘。占城在败军处理之前，故逃跑只逃向势力其余存活城。
- **被俘 / 逃跑 / 战死**：败军三种归宿。被俘=lordId 不变、cityId 改目标城、兵清零（派生成俘虏）；逃跑=随机落该势力一座存活城、保留兵；战死=两槽道具并入目标城仓库且 `discovered=true`、officer 从 `officers` 永久删除。
- **占城（沿用 04、keep 模型）**：攻方胜后**仅翻** `city.lordId` 为攻方君主，不改城内武将 lordId/cityId/兵；原守军派生成俘虏。守军的差异化命运全部收敛到败军处理（不在占城里把守军转在野/清零）。
- **遭劫君主**：本场战斗被俘或战死的君主。识别后进入君主遭劫链：势力有城→立新君，无城→灭亡。
- **待选新君（Pending Succession）**：玩家君主遭劫且势力仍有合格候选时的窄暂停态，只存遭劫君主 id，候选由当前状态动态派生；玩家选定后新君忠诚为 100，并继续同一月末流程。
- **城市战损**：战后无条件对目标城扣发展——农业/商业/金各 `floor(×0.95)`、民忠 `floor(×0.90)`，胜负皆受。规则身份、内联收敛于 `world/city.ts`（与灾害破坏同位）。
- **粮草合并（覆盖式）**：战后目标城 `food` 被**覆盖**为双方剩余战场粮草之和（`playerProvisions + opponentProvisions`），非累加；取代 `04` 的"胜则 food += 随军粮草"。
- **AI 经营（AI Economy）**：月末对每座非玩家城按君主性格选择内政/外交/军备路径并生成命令，全程消费 `GameState.rng`；AI 走固定成长、不扣金/体力、不走玩家校验的简化规则，占人同样通过入队形成派生占用，月末效果复用领域执行器。
- **AI 城兜底（AI City Bottom-line）**：每月在 AI 策略之前对每座非玩家城——状态强制"正常"（等于 AI 城免疫一切灾害）、防灾 +1 封顶 100、粮 <100 直接补到 500（无中生有）。
- **AI 策略路径（AI Strategy Path）**：每座 AI 城每月按君主性格的两个阈值（内政/外交）+ 一次 `RandInt(0,99)` 三选一——内政 / 外交 / 军备，再跑对应模块逐武将随机生成命令。
- **AI 自动升级速度（aiLevelUpRate）**：`GameConfig` 新增平衡旋钮（0..99，默认 0=关）。每月对所有非玩家、非俘虏武将各掷 `RandInt(0,99)`，小于该值则 `level += 1`（直接改等级、不走经验、无上限）；军备模块在 `month%3===0` 时另给强化对象 +1 级。玩家武将永不受影响。
- **AI 即时招降/处斩**：AI 外交模块对本城俘虏的特殊即时效果——招降直接把俘虏 `lordId` 改本势力（派生不再是俘虏）、处斩直接删除俘虏（道具退本城、`discovered=true`）；均**不入队、不走玩家招降/处斩的月末四关/命令模型**。
- **AI 出征（AI Campaign）**：军备模块仅由首位武将尝试（每城每月至多一次），过门槛后向守军合计最弱的相邻敌城出征；名单按兵力降序选取、至少留 1 人守城，随军粮草取本城全部粮，通过入队形成派生占用，不走 `canCampaign`、不扣金/体力。
- **出征四类分流（Campaign Dispatch）**：月末逐条 campaign 按「目标城是否有守军 + 攻方/守方是否玩家」分流——①无守军（攻方任意）→ 不进战斗直接占城；②玩家进攻 AI 城（有守军）→ 交互式战斗；③AI 进攻玩家城（有守军）→ 挂起待守军选择、玩家选守军后进入防守战；④AI vs AI（有守军）→ 速算。
- **速算（Quick Resolve）**：AI vs AI 出征（及守方兵力为 0 的直接占城）不进地图战的胜负结算——按进攻方视角，`A`=攻方总兵力、`D`=守方总兵力（目标城内该势力非俘虏武将兵 + 城后备兵）、`FA`/`FD`=随军粮草/目标城粮：`A==0` 直接败、`D==0` 直接胜；否则按胜率表（`A≥2D`→70% / `A>D` 且 `FA>FD`→60% / `A>D` 否则→40% / `2A<D`→2% / `A≤D` 且 `FA>FD`→30% / 否则→10%，**兵力相等并入「较少」档**）掷 `RandInt(0,99)<胜率` 判攻方胜。不看地形/技能/兵种/操作、不损耗兵力，胜负后复用 `resolveCampaignOutcome`。
- **待守军选择（Pending Defense）**：AI 进攻有守军的玩家城时形成的窄暂停态，只记录目标城；玩家选择最多 10 名守军后进入防守战，选择 0 人则弃守并直接结算占城，随后继续同一月末。
- **战斗 AI（Battle AI）**：`17-battle-ai` 起对手方（始终 `side==='opponent'`：`attack` 模式=AI 守军、`defend` 模式=AI 攻方）在地图战中的自主行动，替换 `endDay` 旧「对手方 no-op」。玩家点 `endDay` 时一次性跑完所有可动 AI 单位的「选将→选落点→攻击/技能/休息」，再扣粮进下一天；每步后即时胜负检查，已分胜负即停。AI **从不撤退**，刻意可预测可反制（只走最短路径、不为软柿子拐弯、选位只算普攻不评估技能收益）。全程消费 `GameState.rng`、确定性遍历、同 seed 可复现。仅对手方；玩家方仍手动、AI vs AI 仍走 `16` 速算。
- **战斗目标点（Battle Target Point）**：AI 选将/选落点的牵引点，由攻守模式二分（无籠城/特定配置分支）——`attack` 模式（AI 守城）= 玩家主将（`attackerCommanderId`）所在格（扑主将）；`defend` 模式（AI 推城）= 城池格中心（推城）。
- **AI 选将（Battle Unit Selection）**：每次 AI 行动前从对手方单位中选——跳过死亡/混乱/石阵/已行动者，在合法单位中取**离战斗目标点曼哈顿距离最小**者（平局 officerId 定序）；纯按距离，不看兵力/输出。
- **AI 选落点（Battle Tile Selection）**：在可达落点（`reachableTiles`）中按阶梯取——①已站城池格则不动 ②可进城（仅 AI 攻方进入即胜）即进 ③否则：预估普攻伤害更高 → 都打不到则更近目标点 → 同伤害选防御地形（山地/树林/村庄/城池/营寨）→ 否则走更远。精确比较器/坐标 tie-break 留 dev，须确定性。
- **AI 预估伤害（Estimated Damage）**：在候选落点上扫普攻范围内活着的玩家方单位——**主将在范围内无条件优先（视作无穷伤害**，击溃即 AI 胜），否则用普攻公式取最大扣兵。**只看普攻、不评估技能收益**。
- **AI 攻击方式（按兵种派生）**：玄兵 = 技能优先 + 最佳选技；其余兵种 = 攻击优先 + 简单选技；不给 `Officer` 新增字段。攻击优先=先普攻再技能、回退普攻/休息；技能优先=先技能、回退普攻/休息。
- **AI 放技能筛（Skill Gate）**：施法前过筛——禁咒不放；`RandInt(0,149) > 有效智力` 跳过（高智力更愿放）；非玄兵 `RandInt(0, floor(带兵量×1.5)) < 当前兵力` 跳过（兵多保留，玄兵免此筛）；再过 MP/天气/地形/范围合法；治疗类只给损失 ≥1/4 兵力的友军。**简单选技**=可用技能中随机抽一个、有目标即放；**最佳选技**=偏好高序号技能、取第一个有合法目标者。
- **命令反馈（Command Feedback）**：`18-command-feedback` 起 `core` 对外给出的**零中文运行时**反馈，两轴——失败走 `reason code`、成功/月末结算/系统事件走 `结果事件`。本地化与多变体台词由 UI 据 `docs/business-command-rules.md` 映射，core 不挑台词、不掷骰。
- **reason code（ReasonCode）**：`shared/command.ts` 的字符串字面量联合，替代 `CommandCheck.reason` 的中文串（如 `gold-insufficient`/`officer-not-found`/`is-captive`），跨命令复用、机器可读。
- **结果事件（OutcomeEvent）**：`shared/outcome.ts` 的判别式联合，描述命令成功/月末结算/系统事件并携必要数据（如 `develop-done` 带 attr/newValue/delta、`city-disaster` 带城+灾种、`lord-succeeded` 带新旧君主）。**只为"携数据/决结果分支"产**；纯确认与 UI 已知量不产。
- **事件并列通道（WithEvents）**：`WithEvents<S>={state,events}`——事件与 state **并列返回、绝不进 `GameState`**（瞬态反馈不污染持久聚合）。产事件的函数返回 `WithEvents<GameState>`，orchestrator 用 `step`/`lift` 组合子逐层 thread 拼接；事件**不消耗 `GameState.rng`**（确定性回归逐字节一致）。
- **下令结果（CommandResult / WithCheck）**：`CommandResult = CommandCheck & WithEvents<GameState> = {ok, reason?, state, events}`（`shared/outcome.ts` 的泛型 `WithCheck<S>`，`game.ts` 别名 `CommandResult`）。所有下令函数 X **自报告**——返回 `CommandResult`，把内部本就计算的 `canX` 结果一并带出（失败 `commandFail(check,state)`：`reason`+`state` 不变+空事件；成功 `commandOk(next,events?)`：`ok:true`）。**校验只在 X 内跑一次**、`reason` 自然冒泡，统一 `develop`/`search` 等所有下令签名。
- **applyWithEvents / apply**：`applyWithEvents(state,action,config)→CommandResult` 为富入口，一次调用返回 `ok/reason/state/events`；`apply=applyWithEvents(...).state` 是丢弃反馈的退化包装。store 派发使用 `applyWithEvents`，两者都不把事件写入 `GameState`。
- **store 层（Store）**：`src/store/` 的 Zustand store 持有当前 `GameState`，`dispatch(action)` 走 `applyWithEvents`，`canDispatch(action)` 走 `canApply`，`newGame({scenarioId,playerLordId,seed})` 从剧本创建对局；store 不写游戏规则或中文运行时文案。
- **通知队列 / 反馈队列（Feedback Queue）**：store 自身持有、非 `GameState` 且不持久化的瞬态结构化反馈集合；成功时加入下令确认与全部 `OutcomeEvent`，失败时加入 `ReasonCode`，并提供消费/出队接口。
- **UI 文案映射（Message Mapping）**：`src/ui/` 的本地化模块把下令确认、`OutcomeEvent` 与 `ReasonCode` 映射为中文 toast，并负责多变体选择与玩家可见性过滤；core/store 保持零中文。
- **主流程 UI（Main Flow UI）**：`21-main-flow-ui` 起的完整界面，移动端横屏优先，串起「选剧本 → 选君主 → 大地图经营 → 战斗地图」四段；用 Tailwind + shadcn/ui 重建，所有美术处先用语义色块/网格占位（图片后补），只做 UI 层、不改 store/core 规则。
- **大地图视图（World Map View）**：按 `City.x/y` 的 `12×9` 世界网格渲染的经营主视图——城池为色块节点（势力色），相邻城之间画邻接连线，点选城进入该城经营。
- **势力色（Faction Color）**：UI 给每位君主按其 id 自动分配的 HSL 色相，用于地图与战斗中区分归属；玩家势力统一高亮、空城为灰。纯展示概念，不入 core。
- **战斗地图视图（Battle Map View）**：玩家参战时进入的全屏交互式战棋界面——`32×32` 地形色块网格（缩放铺满屏高、可平移），双方单位为带标签色块，提供选将/移动/攻击/施法/休息/结束当日，接通 `activeBattle` 全流程。
- **旋转提示遮罩（Rotate Hint）**：移动端竖屏时覆盖屏幕的「请旋转至横屏」提示；本作横屏优先、不做竖屏适配。
