# 美术圣经（Art Bible）

本作美术方向与素材生成规范。**目标**：所有美术素材风格统一、与代码 UI 令牌同源，逐批补入时零代码改动即可生效（见 `src/ui/assets/registry.ts`）。

## 1. 风格基调

- 方向：**国风 / 水墨写意**。宣纸暖底、墨色线条、朱砂点缀、赭金高光、竹青辅色。
- 关键词：水墨、写意、留白、宣纸质感、淡雅、统一柔光、东方古典、三国题材。
- 禁忌：写实照片质感、强烈高饱和霓虹、现代元素、画面内嵌文字/水印、卡通 Q 版（除非另立分支）。

## 2. 调色板（与 `src/index.css` 的 `@theme` 令牌同源，唯一真相源）

| 语义   | HSL                | 用途                     |
| ------ | ------------------ | ------------------------ |
| 宣纸底 | `hsl(40 30% 93%)`  | 背景、留白               |
| 墨     | `hsl(28 28% 16%)`  | 线条、主文字             |
| 朱砂   | `hsl(8 62% 48%)`   | 主色、印章、强调         |
| 赭金   | `hsl(38 68% 50%)`  | 高光、君主/我方高亮      |
| 竹青   | `hsl(140 24% 36%)` | 辅助、成功反馈、我方单位 |

> 生成 prompt 中显式给出上述十六进制/HSL，并在后处理时量化到该板，确保整批同源。
> 地形/势力/战斗单位色见 `src/ui/battle/terrain-color.ts` 与 `src/ui/faction-color.ts`（亦派生于此板）。

## 3. 通用 prompt 前缀（每个素材 = 前缀 + 该素材描述）

```
中国水墨写意风格，宣纸质感暖底，墨色线条，朱砂与赭金点缀，淡雅留白，
统一柔和打光，东方古典，三国题材，无文字无水印，{素材描述}
```

锁风格手段：固定随机种子 + 复用本前缀 + 风格参考（Midjourney `--sref` / SDXL·Flux 的 IP-Adapter 或 LoRA），整批同源生成。

## 4. 素材类别与规格

| 类别         | 注册表 getter     | 尺寸/比例       | 规格要点                          | 落库路径                               |
| ------------ | ----------------- | --------------- | --------------------------------- | -------------------------------------- |
| 武将立绘     | `officerPortrait` | 1:1，512×512    | 半身、统一景别/视角、透明或宣纸底 | `public/art/officers/<officerId>.webp` |
| 城邑图标     | `cityIcon`        | 1:1，128×128    | 俯视小景、统一线宽、透明底        | `public/art/cities/<cityId>.webp`      |
| 地形贴图     | `terrainTile`     | 1:1，256×256    | 可平铺、限定到地形调色板          | `public/art/terrain/<terrain>.webp`    |
| 标题 key art | （封面屏背景）    | 横版，1920×1080 | 大场景、留白给标题、低对比不抢字  | `public/art/cover.webp`                |

id 一律用稳定数字目录 id（见 `20-original-scenarios`）。

## 5. 后处理 → 落库 → 校验

1. 抠图/裁剪到统一尺寸；导出 `webp`。
2. 量化到第 2 节调色板。
3. 放入对应 `public/art/<category>/`，在 `src/ui/assets/registry.ts` 对应表登记 `[id]: '/art/...'`。
4. **一致性评审**：逐张对照本文件风格前缀与调色板，离风格者废弃重抽。

## 6. 红线：图像模型只产「素材」，不产「整页」

UI 布局/组件/地图/面板一律由**代码渲染**（真实 DOM/SVG）。图像模型只用于生成上表的**单个素材**，再由代码合成进界面。不要把页面结构丢给图像模型生成「整页效果图」——那是不可交互、不可维护的 PNG。
