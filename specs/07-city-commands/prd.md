# city-commands（出巡 / 宴请 / 移动 / 输送 / 交易）PRD

## 问题
现有经营指令只覆盖「就地发展」（开垦/招商/征兵）与「应急/情报/军事」（掠夺/侦察/出征），玩家缺少几类常用经营手段：

- 没有提升**民忠**的正向手段（民忠目前只会被掠夺减半，无回升口）。
- 没有快速给某武将**回体力**、顺带笼络人心的手段。
- 己方多城之间无法**调度武将**，也无法**互通粮/金/兵**——资源被锁死在产出城。
- 城金与城粮之间无**兑换**渠道，缺粮时只能干等收粮。

## 目标
在既有月度循环与指令框架上新增 5 条经营指令，形态与既有 `canX`/`X` 两段、经 `game.apply` 分派一致，不引入新机制（除「移动」「输送」两条进 `pendingCommands` 外）：

- **出巡**（占人 ✓，效果即时）：花城金/体力提升本城民忠与人口。
- **宴请**（占人 ✗，效果即时）：花城金给一名在任武将回体力、涨忠诚，且不占用其本月行动。
- **移动**（占人 ✓，效果=月末）：把武将搬到己方另一座城（占人例外：月末落到目标城、不回出发城）。
- **输送**（占人 ✓，效果=月末）：把粮/金/后备兵送往己方另一座城，80% 送达、20% 永久损失。
- **交易**（占人 ✓，效果即时）：在本城买入/卖出粮食兑换城金。

## 关键行为 / 用户故事
1. 作为玩家，我可以对本城在任武将下「出巡」：立即提升本城民忠（随机小幅）与人口，扣其体力与城金；武将离城、月末回城。
2. 作为玩家，我可以「宴请」本城一名在任武将：立即花城金，给他回体力、（非君主时）涨一点忠诚；他本月仍可被派去做其他事（宴请不占人）。
3. 作为玩家，我可以把本城一名在任武将「移动」到己方另一座城：本月占用该武将，月末他落到目标城（不回出发城）。
4. 作为玩家，我可以让本城在任武将「输送」粮/金/后备兵到己方另一座城：下令时即从本城扣掉这些资源与执行人体力；月末 80% 送达目标城、20% 永久损失，执行人回原城。
5. 作为玩家，我可以让本城在任武将「交易」：按固定单价买入或卖出粮食兑换城金，立即结算。
6. 作为玩家，上述任一指令前置不满足时下令被拒绝、状态不变。

## 规则细节

### 出巡（占人 ✓ · 效果即时 · 不入 pendingCommands）
- 作用城 = 执行人所在城（`officer.cityId`）。
- 前置条件：本城至少 1 名在任武将（执行人）；本城金 ≥ 50；执行人体力 ≥ 8。
- 下令阶段（立即结算，消费 RNG）：
  - 本城民忠 `+= RandInt(1, 4)`，封顶 100
  - 本城人口 `+= 100`
  - 执行人体力 `-= 8`、本城金 `-= 50`
  - 执行人离城（占人，`busy`）
- 执行阶段：执行人月末回出发城（由 `Officer.busy` 月末翻回，不入 `pendingCommands`，core 月末无额外效果）。

### 宴请（占人 ✗ · 效果即时）
- 目标是一名武将（非执行人语义）；作用城 = 目标武将所在城。
- 前置条件：本城至少 1 名在任武将；本城金 ≥ 100；**目标武将为该城在任武将**（available 且非俘虏）。
- 效果（立即发生，无 RNG）：
  - 本城金 `-= 100`
  - 目标武将体力 `+= 50`，封顶 100
  - 目标武将**非君主**时：忠诚 `+= 1`，封顶 100（君主忠诚派生恒 100，跳过写入，沿用 `officerLoyalty` 约定）
- **不占人**：目标武将 `busy` 不变，本月仍可被其他指令派遣；不入 `pendingCommands`、不耗 RNG。

### 移动（占人 ✓ · 效果=月末 · 入 pendingCommands）
- 被移动的武将即被占用者本人，无独立执行人；不扣体力、不扣金。
- 前置条件：本城至少 1 名在任武将（被移动者）；目标城存在，且**不是本城**、**与本城同势力（己方城，`target.lordId === officer.lordId`）**。
- 下令阶段：被移动武将离城（占人）；入 `pendingCommands`（新增 `move` 分支，记 `officerId` + `targetCityId`）。
- 执行阶段（月末，非 campaign 趟、按入队序）：把该武将加入目标己方城——`officer.cityId = targetCityId`。
- **占人例外**（同出征）：月末武将不回出发城；`busy` 仍在月末统一翻回 false（此时已落到目标城），turn 层无特例。

### 输送（占人 ✓ · 效果=月末 · 入 pendingCommands）
- 作用城（出发城）= 执行人所在城。「兵」两端均指**后备兵**（`reserveTroops`）。
- 前置条件：本城至少 1 名在任武将（执行人）；执行人体力 ≥ 8；目标城存在，且**不是本城**、**与本城同势力**。
- 下令阶段（立即结算扣减，不消费 RNG）：选执行人 → 选输送粮（0 ~ 本城粮）、金（0 ~ 本城金）、兵（0 ~ 本城后备兵）→ 选目标友城：
  - 执行人体力 `-= 8`
  - **立即从出发城扣掉**：城粮 `-= 粮`、城金 `-= 金`、后备兵 `-= 兵`
  - 执行人离城（占人）；入 `pendingCommands`（新增 `transport` 分支，记 `officerId` + `targetCityId` + `food` + `gold` + `troops`）
- 执行阶段（月末，非 campaign 趟、按入队序，消费 RNG）：
  - **80% 概率送达**：目标城 城粮 `+= 粮`、城金 `+= 金`、后备兵 `+= 兵`
  - **20% 概率失败**：资源永久消失，不退回出发城
  - 无论成败：执行人月末回出发城（`busy` 月末翻回）

### 交易（占人 ✓ · 效果即时 · 不入 pendingCommands）
- 作用城 = 执行人所在城。一条指令含**买入**或**卖出**两种操作（择一 + 数量）。
- 前置条件：本城至少 1 名在任武将（执行人）；执行人体力 ≥ 12。
- 买入粮食（5 金 = 1 粮）：数量上限 `floor(城金 / 5)`；立即 城粮 `+= 购买量`、城金 `-= 购买量 × 5`。
- 卖出粮食（2 金 = 1 粮）：数量上限 = 本城所有城粮；立即 城粮 `-= 出售量`、城金 `+= 出售量 × 2`（城金不设上限）。
- 下令后：执行人体力 `-= 12`、离城（占人）。
- 执行阶段：执行人月末回出发城（不入 `pendingCommands`，core 月末无额外效果）。

### 月末执行顺序（沿用既有，不改）
`pendingCommands（先非 campaign 按入队序：掠夺/移动/输送，再 campaign：出征）→ settle（收粮/收税）→ 回城 + 体力恢复 → 月份 +1`。移动/输送在第一趟随其它非 campaign 指令按入队序执行。

### 数值 & 归类（依总纲取舍）
- **扁平成本/上限 → 平衡旋钮（config）**：出巡体力 8 / 城金 50、宴请城金 100 / 体力回 50、输送体力 8、交易体力 12。
- **公式系数 / 转化率 → 规则身份（内联领域常量）**：出巡民忠随机幅度 `RandInt(1, 4)`、人口增量 100；宴请忠诚 +1；输送送达概率 80%；交易买价 5 金/粮、卖价 2 金/粮。
- **量纲上限（内联）**：民忠/体力/忠诚封顶 100。
- 门槛字面量：在任武将 `≥1`、各资源门槛同上消耗值。

## 范围 & 不做
- 做：
  - 出巡、宴请、移动、输送、交易 5 条指令的 `canX` 校验 + 变更，经 `game.canApply/apply` 分派。
  - `City` 新增**人口**字段（fixture 播种、出巡 +100）；`PendingCommand` 新增 `move`、`transport` 两个判别式分支及其月末执行。
  - 民忠新增正向回升口（出巡）。
  - 对应指令在 UI 面板的下令入口（含输送的多步选择、交易的买/卖切换）。
- 不做（本切片）：
  - **人口暂无任何下游游戏规则**（仅展示状态）——经与用户确认、明知违反 YAGNI 仍保留，待后续切片再赋予作用（征兵上限、收粮收税等是否吃人口留待后续）。
  - 路径/距离/邻接对移动与输送的限制（只要己方城即可，不要求相邻、不做多跳寻路）。
  - 交易价格的浮动/市场机制（买卖单价固定）；城金上限。
  - AI 使用这 5 条指令（AI 切片再接）。
  - 武将兵参与输送（仅后备兵可输送）。

## 验收标准
- [ ] 出巡：本城有在任武将、城金 ≥ 50、执行人体力 ≥ 8 时可下令；立即民忠 `+=RandInt(1,4)`（封顶 100）、人口 +100、扣体力 8、扣城金 50、执行人 busy；不入 pendingCommands。前置任一不满足时 no-op。
- [ ] 宴请：本城金 ≥ 100、目标为该城在任武将时可下令；立即扣城金 100、目标体力 +50（封顶 100）、非君主忠诚 +1（封顶 100），君主忠诚仍 100；目标 busy 不变、可再被派遣；不入 pendingCommands、不耗 RNG。前置任一不满足时 no-op。
- [ ] 移动：本城有在任武将、目标城为己方城且非本城时可下令；被移动者 busy、入 pendingCommands（move）；不扣体力/金。月末把其 `cityId` 改为目标城、不回出发城，busy 月末翻回。前置任一不满足时 no-op。
- [ ] 输送：本城有在任武将、执行人体力 ≥ 8、目标城为己方城且非本城、且输送量在 0~本城对应资源内时可下令；立即扣执行人体力 8 并从出发城扣掉粮/金/后备兵、执行人 busy、入 pendingCommands（transport）。月末 80% 送达（目标城 +粮/金/后备兵）、20% 永久损失；执行人月末回出发城。前置任一不满足时 no-op。
- [ ] 交易：本城有在任武将、执行人体力 ≥ 12 时可下令。买入量 ≤ `floor(城金/5)`，城粮 += 量、城金 -= 量×5；卖出量 ≤ 本城粮，城粮 -= 量、城金 += 量×2；扣体力 12、执行人 busy；不入 pendingCommands。前置任一不满足或数量越界时 no-op。
- [ ] 月末顺序不变：移动/输送在第一趟（非 campaign）按入队序执行，仍先于 settle 与回城。
- [ ] 既有指令与循环（开垦/招商/征兵/分配/掠夺/侦察/出征/收粮收税/体力恢复/跨年）行为不受影响。
- [ ] 玩家入口经 store 注入 `playerLordId` 校验执行人/目标武将归属己方；`lordId` 不作为 Action 字段。
- [ ] 给定相同 RNG 种子，整段推进结果可复现（core 纯函数、确定性；出巡/输送消费 RNG）。
