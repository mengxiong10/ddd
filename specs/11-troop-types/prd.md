# troop-types 兵种系统 PRD

## 问题
武将目前没有「兵种」维度：所有武将同质，既无机动差异、也无后续战棋相克的承载点；道具只能加武力/智力，缺少改变机动与作战类型的策略表达。

## 目标
为武将引入「兵种」与派生「移动力」两个维度，并让道具新增「移动力加成」「修改兵种」两类功能。本切片只建数据模型 + 装备门槛校验 + 面板展示；相克战斗与移动力的实际作用留待后续战棋切片（沿用「人口」先建模后用的先例）。

## 关键行为 / 用户故事
1. 作为玩家，我能在面板上看到每名武将的兵种与当前移动力。
2. 作为玩家，我把带「移动力加成」的道具给武将后，他的移动力随之提高。
3. 作为玩家，我把带「改兵种」的道具给武力/智力够格的武将后，他的有效兵种变成对应兵种（水军/玄兵/极兵）。
4. 作为玩家，当武将武力/智力不够时给他玄兵/极兵道具，兵种不变（道具仍佩戴、加成照常）。
5. 作为玩家，没收改兵种道具后，该武将的有效兵种随之回退（与武力/智力加成一致，纯派生）。

## 关键规则（与用户对齐结论）

### 六种兵种（基础移动力）
| 兵种 | 基础移动力 | 设计定位 |
|------|:--:|------|
| 骑兵 | 5 | 攻强防弱、平地强 |
| 步兵 | 4 | 攻弱防厚 |
| 弓兵 | 4 | 远程、反骑兵 |
| 水军 | 5 | 水战万能 |
| 极兵 | 6 | 最高机动 + 高攻，需武力 > 105 才能装备 |
| 玄兵 | 3 | 法师型、最高 MP，需智力 > 105 才能装备 |

- 兵种基础移动力与「定位」文字属规则数据：移动力是规则身份（内联领域常量）；定位文字属 UI，不入 core。
- 兵种相克/水战/反骑兵等**作战效果本切片不实现**（无战棋格子可承载），仅记录定位备查。

### 武将兵种（派生 + 基础存储）
- 武将新增**基础兵种** `troopType`（存储字段，fixture 播种），可为六种中任意一种；fixture 是可信真相源，**基础兵种不跑 > 105 门槛校验**（门槛只约束道具改兵种）。
- **有效兵种 = 派生值**（与有效武力/有效智力同构、不写回 Officer）：以基础兵种为起点，按所持道具的「装备先后」依次应用其改兵种结果，合法的后者覆盖前者。
- **战斗等下游需要兵种时，临场读取有效兵种**（本切片无下游消费）。

### 道具新增两字段
- **移动力加成**（整数）：佩戴即生效。
- **改兵种字段**（`0..3`）：`0` 不改兵种 / `1` 改为水军 / `2` 改为玄兵（要求有效智力 > 105）/ `3` 改为极兵（要求有效武力 > 105）。
  - 改兵种字段只能把兵种改成水军/玄兵/极兵；骑兵/步兵/弓兵只能来自基础兵种。
- 一件道具可同时带武力/智力加成、移动力加成、改兵种字段。

### 装备门槛（派生模型下的判定）
- 玄兵/极兵门槛用**有效属性**（基础 + 该武将所持全部道具加成，**含正在被判定的这件道具自身**）**严格 > 105**。
- 派生模型：门槛在求有效兵种时实时判定。**没收改兵种道具、或没收使有效属性跌破 105 的加成道具后，有效兵种会随之回退**——即原始需求里「装备后不再重复检查」放宽为「随道具状态实时派生」（与武力/智力加成行为一致）。
- 不合法（属性不够）时该道具的改兵种不生效、兵种维持，但道具仍佩戴、其余加成照常。

### 移动力（派生，仅展示）
- **移动力 = 有效兵种的基础移动力 + 所持道具移动力加成之和**，纯派生（与有效武力/智力同构）。
- 本切片移动力**仅供展示**，无任何下游规则（不影响出征/移动/邻接判定）。

### 装备有序（≤2 件，后者覆盖）
- 武将持有的道具是**有序**的（最多 2 件，记录装备先后）；改兵种时按此顺序应用，**后装备的覆盖先装备的**。
- 这要求把道具归属从单一 `holder` 重构为「武将持有一个有序装备列表」（详见待定决策）。

## 范围 & 不做
- 做：
  - 武将 `troopType` 基础兵种字段 + fixture 播种；六种兵种基础移动力规则数据。
  - 有效兵种派生（基础 + 道具改兵种、有序覆盖、门槛校验）。
  - 道具「移动力加成」「改兵种」两字段 + fixture 播种；移动力派生。
  - 道具归属重构为「武将有序装备列表（≤2）」，并迁移既有 reward/confiscate/search/captive/queries/fixture 及其测试。
  - 武将兵种、移动力、道具新字段在 UI 面板展示。
- 不做（本切片）：
  - 兵种相克、水战、反骑兵等任何作战效果；移动力对出征/移动/路径的实际作用。
  - 新增「装备/卸下」独立指令（沿用既有赏赐/没收即装备/卸下事件）。
  - 兵种成长/转职；道具购买/打造/掉落/损坏。
  - 移动力或兵种进入战斗结算（战斗仍只比兵力总和）。

## 验收标准
- [ ] 武将有基础兵种 `troopType`（fixture 播种）；六种兵种各有正确基础移动力。
- [ ] 有效兵种 = 基础兵种被所持改兵种道具按装备先后覆盖后的派生值；无改兵种道具时即基础兵种。
- [ ] 改兵种字段 `1` 必生效改水军；`2`/`3` 仅在有效智力/武力（含该道具自身加成）严格 > 105 时生效，否则兵种不变、道具仍佩戴。
- [ ] 没收改兵种道具（或使属性跌破阈值的加成道具）后，有效兵种实时回退（纯派生、不写回 Officer）。
- [ ] 武将持 2 件改兵种道具时，后装备者的兵种结果覆盖先装备者。
- [ ] 移动力 = 有效兵种基础移动力 + 所持道具移动力加成之和；仅展示，不影响任何既有指令/月末逻辑。
- [ ] 道具归属重构后无双向冗余、既有道具相关行为（赏赐/没收/搜寻/处斩流放道具退城）全部回归通过。

## 待定决策
- **道具归属重构形态**（留给 dev）：武将存有序 `items: ItemId[]`（≤2）作为单一真相源、城内道具另存；还是保留 `holder` 判别式并在 officer 情形加「装备槽序号」以最小改动表达顺序。前者更贴近「装备槽」语义但改动放大较大（触及现有 holder 单一真相源红线与一批测试），后者改动小但语义略隐晦——dev 阶段按可维护性/改动放大权衡定稿。
- **fixture 基础兵种与道具新字段的具体播种值**：留待 dev/实现阶段定平衡。
