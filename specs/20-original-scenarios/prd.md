# original-scenarios PRD

## 问题

游戏需要从原版四个历史时期创建完整对局。现有转换虽已导入四剧本，但用姓名拼音充当字符串 ID、重复写入四份共享数据，并把未登场实体搬运于独立 `pendingDebuts` 池；这些选择会产生重名碰撞、重复状态与新增数据时的改动放大。原始时期数据还会重复引用同一道具，而本项目要求单个剧本内每件道具唯一。

## 目标

保留四个原版剧本及既有开局流程，将剧本数据规范化为稳定数字身份目录与时期状态：城市、武将、道具跨剧本使用稳定数字 ID，共享资料只保存一份；所有本剧本成员从开局起存在于对局聚合中，未登场由位置为空派生；生成期确定性消除重复道具引用。

## 关键行为 / 用户故事

1. 作为玩家，我要从四个原版时期和其中任一现存君主开局，以体验完整的 38 城局面。
2. 作为开发者，我要让城市、武将、道具使用跨剧本稳定的数字身份，以避免重名、拼音碰撞和引用漂移。
3. 作为开发者，我要把城市身份、武将身份、道具基础资料与邻接拓扑各保存一份，时期文件只保存成员与时期状态，以便新增和核查数据。
4. 作为系统，我要让未来武将从开局起属于 `officers`、以无城市表达未登场，并在到年后进入指定或随机城市。
5. 作为系统，我要让未登场武将预先持有其装备，武将登场后装备自然可用，不维护第二份待登场装备清单。
6. 作为系统，我要保证单个剧本内同一道具至多一件；原版重复引用按稳定扫描顺序只保留第一次。
7. 作为开发者，我要让相同剧本、君主和 seed 生成相同状态，并让所有基于 ID 的遍历和平局规则改用数字升序。

## 数据规则

### 身份与时期成员

- `CityId`、`OfficerId`、`ItemId` 均为从 1 开始的普通 `number`，三类各自独立编号空间。
- 城市沿用原版 1..38；道具沿用原版基础道具 1..37；武将按时期 1→4、时期内原始编号升序首次出现时分配稳定 ID，同一历史武将跨时期复用。
- 武将身份目录确定后只追加、不重排；真正同名不同人允许拥有不同 ID。
- 共享目录不自动成为剧本成员；只有时期文件列出的 ID 才进入该剧本。
- 生成 JSON 使用数组，创建对局后按 ID 索引为只读字典。

### 登场

- 删除独立待登场池；本剧本的现存与未来武将从开局起都在 `GameState.officers`。
- `Officer.cityId=null` 表示未登场；`Item.holder=null` 表示独立待登场道具，不另存 `appeared`。
- 武将与道具均持有必填 `appearanceConditions={birth,recruiterId,cityId}`；原版 0 引用转换为 `null`。
- 武将在 `year >= birth+16` 时登场；独立道具在 `year >= birth` 时登场。目标城为空则在登场时随机选择。
- 月末登场按武将数字 ID 升序，再按独立道具数字 ID 升序处理；未登场武将持有的装备不单独处理、不消费 RNG。
- 未登场武将可从开局起持有已发现装备；装备随持有者不可操作，武将登场后自然生效。
- 初始数据不允许已到年份却仍无位置；独立待登场道具必须未发现。

### 道具唯一性

- `goods.json` 的每条记录是唯一道具基础资料；单个剧本中每个 `ItemId` 至多出现一次。
- 生成器按原始 `goods_queue` 顺序，再按武将原始编号升序与装备槽 1→2 扫描引用；第一次派发，后续重复忽略。
- 第一时期六种兵符保留其首个城市引用；第四时期绝影由华佗保留、管辂的重复装备忽略。
- 武将基础武力/智力仍扣除原始装备槽的全部加成；被去重装备的武将失去该装备的有效加成。
- 城市道具的发现位来自原始队列高位；装备一律已发现，包括未登场武将装备。
- 基础目录中未被时期引用且没有有效未来登场安排的道具不进入该剧本。

### 共享数据

- 生成产物拆为 `cities.json`（id/name）、`officers.json`（id/name）、`items.json`（基础属性）、`adjacency.json`（共享边）和四份时期状态。
- 时期数据只保存剧本摘要、城市状态、武将状态、道具状态和各自 ID，不重复共享名称/属性/邻接。
- 原始快照保持原样；共享武将目录显式校订 `荀或→荀彧`、`李决→李傕`、`博士仁→傅士仁`、`张合→张郃`，不改变稳定数字 ID。
- `GameState.adjacency` 保留；创建剧本时从共享文件构造，继续可注入、可序列化。
- 所有非空 `lordId/cityId/holder/recruiterId` 必须引用同一时期成员或共享城市，悬空引用使生成失败。

### 确定性顺序

- 所有过去依赖字符串 ID 字典序的遍历、RNG 消费与平局规则改为数字 ID 升序。
- 不保留姓名/拼音排序键；本次允许相同 seed 相对旧版本产生不同结果，但新模型内必须可复现。

## 范围 & 不做

- 做：四剧本生成与选择、稳定身份目录、时期状态拆分、数字 ID、道具唯一化、统一登场模型、四个已确认姓名校订、空城和既有兵种/开局规则回归。
- 不做：旧存档迁移（当前未实现持久化）、动态下载剧本、通用 schema 框架、战略地图坐标、其余原版姓名校订、人口上限和年龄寿命规则。
