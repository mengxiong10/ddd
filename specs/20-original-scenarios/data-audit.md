# 原版剧本数据核查

本文件记录原版数据转换时发现的索引陷阱与完整性核查结果，供人工复核；它不是领域规则来源。

## 武将队列索引

`periods[].generals[].id` 是提取器补充的 **1-based id**，但 `general_queue` 保存的是原版 `g_Persons` 的 **0-based 索引**。原版源码从队列取值后直接访问 `g_Persons[p]`，因此转换关系是：

```text
generalId = generalQueueValue + 1
```

若漏掉 `+1`，整条城市人才队列都会错配，并会把实际在城武将误判为异常或待登场。

## 南方小鬼

“南方小鬼”在四个剧本中都已存在于开局城市队列，不经过 `PersonUpDatadate` 的年份登场：

| 剧本     | general id | general_queue 原值 | 开局城市   |
| -------- | ---------: | -----------------: | ---------- |
| 董卓弄权 |        187 |                186 | 零陵（36） |
| 曹操崛起 |        185 |                184 | 西凉（1）  |
| 赤壁之战 |        189 |                188 | 巴郡（32） |
| 三足鼎立 |        176 |                175 | 晋阳（5）  |

其 `birth=0` 不参与登场判断，因为队列已经把它物化在城中。

## 武将物化核查

采用以下口径重算四个剧本：

- 城市 `general_queue` 切片决定开局在城武将及其 `cityId`，读取时执行 `+1` 转换。
- 不在城市队列、姓名非空且 `birth + 16 > start_year` 的武将进入待登场池。
- 空姓名槽位不物化。

结果：

| 剧本     | 未来待登场 | 姓名非空但无法归类 | 队列引用空姓名 | 队列重复引用 |
| -------- | ---------: | -----------------: | -------------: | -----------: |
| 董卓弄权 |         30 |                  0 |              0 |            0 |
| 曹操崛起 |          9 |                  0 |              0 |            0 |
| 赤壁之战 |         10 |                  0 |              0 |            0 |
| 三足鼎立 |         10 |                  0 |              0 |            0 |

按正确索引转换后，当前没有需要人工裁决的异常武将记录。

## 道具出现条件

城市 `goods_queue` 保存 0-based 道具模板索引，高位 `0x8000` 表示已发现；四剧本开局队列均未设置该高位，因此队列中的道具实例开局均为未发现。

`goods.appearance_conditions` 只有三种道具带非零条件，且四个剧本完全相同：

| 道具        | birth | bole | city |
| ----------- | ----: | ---: | ---: |
| 倚天剑（8） |   144 |    2 |   17 |
| 青虹剑（9） |   144 |    2 |   17 |
| 的卢（25）  |   144 |  167 |   28 |

`birth=144` 与四剧本开局年份及现有年份登场模型不一致。本轮仅按城市队列和武将装备物化道具实例，不据此创建待登场道具；该字段语义保留待核查。

## 原版姓名

原始快照保留原版字形，生成器在共享武将目录输出时显式校订以下已确认误写：

- `荀或` → `荀彧`
- `李决` → `李傕`
- `博士仁` → `傅士仁`
- `张合` → `张郃`

`南方小鬼`、`通宵虫` 按原版角色保留，不视为空槽或自动清洗对象。

### 稳定身份

武将运行时身份使用跨时期稳定的数字 ID，不再从姓名拼音派生；同音姓名不会形成 ID 碰撞。姓名校订只改变共享目录的显示名称，不改变既有数字身份与时期引用。
