# troops（兵力系统）PRD

## 问题

经营循环已能产出金/粮，但还没有「兵」。玩家无法把城里的资源转化成可作战的兵力，也无法在城与武将之间调配兵力。没有这一层，后续的战斗/占领就没有输入。本切片提供两条指令：**征兵**（把民忠+金钱变成城的后备兵）和**分配**（在城后备兵与某武将兵力之间重分配）。

## 目标

在现有月度循环上加入「兵」资源与两条指令：

- **征兵**：占人指令，下令时把后备兵注入城、扣执行人体力与城金；月末执行人回城。
- **分配**：不占人、立即生效，在「城后备兵」与「某武将现有兵」之间重分配，受该武将带兵量上限约束。

并为此引入必要的新状态：城的 `民忠`、`后备兵`，武将的 `兵`、`等级`、`武力`。

## 关键行为 / 用户故事

1. 作为玩家，我可以对城里**在任**的武将下「征兵」指令，输入征兵数 N，使城后备兵 += N；该武将扣体力、城扣金后离城（占人），月末回城。
2. 作为玩家，征兵时系统按 `min(民忠×20, 金×10)` 给出可征上限，N 不得超过该上限。
3. 作为玩家，当城金不足、执行人体力 < 12、或城内无在任武将时，征兵被拒绝。
4. 作为玩家，我可以对城里的某武将下「分配」指令，输入目标兵力 N，在「城后备兵」与「该武将现有兵」之间重分配——既能从城调兵给武将，也能把武将的兵上交回城。
5. 作为玩家，分配立即生效、**不占人**（被分配的武将本月仍可继续做事，例如先分兵再派出征），不扣体力、不扣金。
6. 作为玩家，分配的 N 受 `min(武将带兵量上限, 后备兵 + 武将现有兵)` 约束，超出则拒绝。

## 规则细节

### 新增状态

- **城.民忠**：城属性，范围 `[0, 100]`（上限 100）。本切片为**静态值**：征兵不消耗民忠，也不随月度变化（涨/跌民忠留后续切片）。
- **城.后备兵**：城级未编队兵力，`≥ 0`，初始 0。
- **武将.兵**：该武将当前带兵数，`≥ 0`，初始 100。
- **武将.等级、武将.武力**：武将静态属性，用于带兵量公式。

### 武将带兵量上限

- `带兵量上限 = 等级×100 + 武力×10 + 智力×10`

### 征兵（占人 ✓）

- 目的：把民忠与金钱转化为城后备兵。
- 前置条件（下令时校验，全部满足才可下令）：
  - 城内至少 1 名在任武将（作为执行人）
  - 城金 ≥ 1（**仅最低门槛**，实际扣金见下）
  - 执行人体力 ≥ 12
- 可征上限：`min(民忠×20, 金×10)`
- 下令阶段（立即发生）：
  - 玩家输入征兵数 N（`1 ≤ N ≤ 可征上限`）
  - 城后备兵 += N
  - 执行人体力 -= 12
  - 城金 -= `ceil(N / 10)`（不足 10 也按 1 金起收，杜绝零成本征兵）
  - 执行人离城（占人）
- 执行阶段（月末）：执行人回到出发城。

### 分配（占人 ✗，立即生效）

- 目的：在城后备兵与某武将现有兵之间重新分配。
- 前置条件：城里至少 1 名在任武将（作为分配目标，须在该城且在任）。
- 可分配上限：`min(武将带兵量上限, 后备兵 + 武将现有兵)`
- 效果（立即）：
  - 玩家输入目标兵力 N（`0 ≤ N ≤ 可分配上限`）
  - 城后备兵 = 原后备兵 + 武将原兵 − N
  - 武将兵 = N
- 不扣体力、不扣金、不占人。

### 与月度循环的衔接

- 征兵是占人指令，沿用既有「下令离城 → 月末回城」机制（与开垦/招商一致）。
- 分配不进入占人机制，下令当下即结束，对月末结算无影响。
- 本切片不改动收粮/收税/体力恢复等既有结算。

## 范围 & 不做

- 做：`兵` 资源（城后备兵 + 武将兵）；城 `民忠`、武将 `等级/武力` 新字段；征兵（占人）与分配（不占人）两条指令及其前置/上限/扣减；并入既有月度循环；fixture 补充默认值。
- 不做（本切片）：民忠的涨跌/恢复机制；征兵对民忠的消耗；AI 征兵/分配行为（AI 城静止）；战斗/编队/出征；兵的伤亡/补员；UI（core 可推进即可）；武将成长。

## 验收标准

- [ ] 城/武将新增 民忠、后备兵、兵、等级、武力 字段，初始局面按 fixture 表加载。
- [ ] 征兵：N 在 `[1, min(民忠×20, 金×10)]` 内时，后备兵 += N、执行人体力 -12、城金 -= ceil(N/10)、执行人 busy；月末回城。
- [ ] 征兵前置任一不满足（无在任武将 / 城金 < 1 / 体力 < 12 / N 超上限）时拒绝，状态不变。
- [ ] 征兵 N < 10 仍至少扣 1 金（ceil 生效），无法零成本征兵。
- [ ] 分配：N 在 `[0, min(带兵量上限, 后备兵+武将原兵)]` 内时，后备兵 = 原后备兵 + 武将原兵 − N、武将兵 = N；不扣体力/金、不占人。
- [ ] 分配可双向：N 大于武将原兵=从城调兵给武将；N 小于=上交兵回城；N=0=全部上交。
- [ ] 分配 N 超过带兵量上限或超过「后备兵+武将原兵」时拒绝。
- [ ] 被分配武将同月仍可被下其它令（如随后征兵/开垦/出征）。
- [ ] 既有经营循环（收粮/收税/体力恢复/跨年）行为不受影响。
- [ ] 给定相同 RNG 种子，整段推进结果可复现。
