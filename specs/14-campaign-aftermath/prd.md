# campaign-aftermath（完整战后处理）PRD

> 把 `04-campaign` 留下的极简战后处理（`resolveCampaignOutcome`）深化为一套完整流程：
> 胜方回城 → 占城 → 败军逐人命运（被俘/逃跑/战死）→ 君主遭劫（重选 / 灭亡，玩家手动选新君）→ 城市战损 → 粮草合并。
> 前置：依赖 `12-battle`（`BattleState`/`concludeBattle`/月末可重入）已实现。

## 问题

`resolveCampaignOutcome` 现在只做三件事：把出征武将搬到目标城、胜则翻城归属并 `food += 随军粮草`、对两方君主跑一次自动重选。它**没有**败军的差异化命运——所有败方武将一律"就地冻结成俘虏"，没有逃跑、没有战死、没有掉落，被俘也不清兵；君主被俘只会自动立新君，玩家无从参与；城市挨打不掉发展、战场剩余粮草处理也粗糙。战斗打完缺少"代价感"与"戏剧性"，撑不起占城/灭国的分量。

## 目标

在 `concludeBattle` 写回兵力/经验/等级**之后**，跑一套完整战后处理：胜方存活单位入目标城；攻方胜则占城（仅翻城归属，沿用 04）；**败方每名参战武将逐一掷骰决定被俘 / 逃跑 / 战死**（消耗 RNG）；据此识别遭劫君主，AI 势力自动立新君、玩家势力**挂起月末交玩家手动选新君**，无人可立则灭亡；不论胜负目标城吃一次战损；双方剩余战场粮草覆盖式合并进目标城。本切片**只服务 battle 路径**（玩家进攻可达），并**删除已不可达的 `executeCampaign` 速算死代码**。

## 关键行为 / 用户故事

1. 作为系统，战斗分胜负、写回每单位兵力/经验/等级后，把**胜方所有存活单位**（officer 未被 8.5 战死即算活）放入本场**目标城**。
2. 作为系统，**攻方胜**时占城：目标城 `lordId` 改为攻方君主（沿用 04，**仅翻城归属**——不动城内武将的 lordId/cityId/兵；原守军因城易主派生成俘虏）。占城在败军处理**之前**发生，使败方"逃跑"只能逃向其势力**其余存活城**。
3. 作为系统，对**败方每名参战武将**逐一判定命运（按 officerId 定序、消耗 RNG）：
   - **第一关 直接被俘**：`RandInt(0,99) > 该武将有效智力` → 直接被俘；否则进逃跑。
   - **第二关 逃跑**：取该武将所属势力**所有现存城**，有则随机选一座落城、逃跑成功（保留兵力）；一座都没有则逃跑失败。
   - **第三关 逃跑失败的归宿**：若该武将是自家君主则记为"遭劫君主"；再掷骰——`RandInt(0,99) === 0` → **战死**，否则 → **被俘**。
4. 作为系统，**被俘**：lordId 不变、cityId 改为目标城、**兵力清零**（所在城已属胜方 → 派生成俘虏）。**战死**：该武将两个装备槽道具全部并入目标城仓库并标记 `discovered=true`，该武将从 `officers` **永久删除**。
5. 作为系统，识别到任一遭劫（被俘 / 战死）君主后进入君主遭劫链：该势力**仍有城**→ 立新君；**已无城**→ 灭亡（该势力所有城已无意义/不再属它，剩余武将均为俘虏）。
6. 作为系统，**AI 势力**立新君：在剩余**未被俘**武将中取**有效智力最高**者（平局取 id 字典序最小）自动立为新君；新君忠诚强制 100；该势力所有城与未被俘武将归属切到新君。
7. 作为玩家，**玩家君主**遭劫且本势力仍有城时：本场战后处理跑完全部状态转换后，**月末挂起**为"待选新君"态；我从本势力剩余武将中**手动选一位**；兑现后该武将成新君（忠诚 100、势力城与武将归其），月末**续跑**剩余 campaign 与尾段。玩家君主死/俘**不直接 Game Over**。
8. 作为系统，**不论胜负**目标城承受一次战损：农业 −5%、商业 −5%、金 −5%、民忠 −10%（各按当前值 `floor`）。
9. 作为系统，战后把**双方剩余战场粮草之和覆盖式写入**目标城城粮（`目标城粮 = 玩家方剩余战场粮 + 对手方剩余战场粮`，**替换**而非累加，取代现有 `food += 随军粮草`）。

## 规则细节

### 战后处理总顺序（在 `concludeBattle` 写回兵力/经验/等级之后）

1. 胜方存活单位 → 目标城（8.3）。
2. 攻方胜 → 占城：`city.lordId = 攻方君主`（8.4，仅翻归属）。
3. 败军处理：败方每名参战武将逐一掷骰（8.5），收集遭劫君主集合。
4. 君主遭劫：对每个遭劫君主 → 重选（AI 自动 / 玩家挂起）或灭亡（8.7 / 8.8）。
5. 城市战损：目标城 −5%/−5%/−5%/−10%（8.10）。
6. 粮草合并：目标城粮 = 双方剩余战场粮之和（8.11，覆盖）。

> 步骤 5、6 只动目标城、与新君是谁无关，故玩家"待选新君"挂起放在**全部状态转换跑完之后**（最小侵入）。挂起期间被劫君主的势力暂为"无新君"，玩家兑现选择后立即补齐。

### 败军 = 败方参战武将

- 胜方 = 攻方胜则攻方、否则守方；**败方 = 另一方**的参战武将（≤10）。
- "胜方存活单位"包含战中被击溃（0 兵、`status=dead`）但 officer 未被 8.5 战死者——他们仍回城（0 兵）。

### 占城（沿用 04、capture_model=keep）

- 占城**只翻 `city.lordId`**，不改城内武将归属/位置/兵。原守军因 `officer.lordId !== 所在城.lordId` 派生成俘虏（既有规则）。
- 连锁（推导自然后果、零特殊代码）：本势力旧将随城回归（不再满足俘虏条件）、敌方第三国俘虏关押者从原城势力变本势力但仍是俘虏。
- 8.4.2 草案里"非君主守军→在野 + 一律清零"**不采用**——守军处理收敛到 8.5（参战单位）。

### 重选君主（深化 `world/succession`）

- 触发：某势力君主被 8.5 判定被俘 / 战死，且该势力**仍有城**。
- 候选：该势力剩余**未被俘**武将；空则灭亡。
- AI 势力自动取有效智力最高（平局 id 最小）；**新君忠诚强制 100**；势力所有城与未被俘武将归属切新君。
- 玩家势力改为挂起手动选（见下）。

### 待选新君（玩家专用暂停态，类比 `activeBattle`）

- `GameState` 新增一个**窄判别式暂停字段**（非空 = 月末挂起、等玩家选新君），存 `lordId`（遭劫的玩家君主）+ 候选武将。**不**泛化为通用 pendingChoice（YAGNI）。
- core 纯同步：不用 Promise；由一个**新 action**（玩家选定某候选）兑现——把势力城与未被俘武将归属切到该候选、其忠诚 100，清空暂停字段，再**续跑月末**剩余 campaign 与尾段。

### 灭亡（8.8）

- 君主遭劫且**无未被俘武将可立**（或已无城）→ 该势力灭亡：剩余武将均为俘虏；本切片仍不做"统一全国"的全局胜负判定。

### 城市战损（8.10）

- 无条件：目标城 `agriculture/commerce/gold` 各 `floor(×0.95)`、`loyalty` `floor(×0.90)`。规则身份、内联领域常量（与灾害破坏同位、收敛在 `world/city.ts`）。

### 粮草合并（8.11）

- `concludeBattle` 把战斗结束时 `playerProvisions + opponentProvisions` 之和**覆盖写入**目标城 `food`（取代 04 的 `addFood(target, provisions)`）。即使败方带去的粮也归胜方所有。

### 确定性红线（修订）

- `resolveCampaignOutcome` 自此**消耗 `GameState.rng`**（被俘/逃跑/战死掷骰、逃跑随机选城）。`13-battle-skills` 已让战斗耗 RNG；本切片把"战后处理"也纳入耗 RNG 范围。整局同 seed 仍可复现。需同步修订 AGENTS 红线。

### 删除死代码

- 删除 `executeCampaign` 速算分支与其 `defenderTroops` 辅助，以及 `turn/end-month` 里"非玩家 campaign 速算 fallback"分支（`12-battle` 起 AI 不出征、仅玩家进攻可达，速算路径已不可达）。`resolveCampaignOutcome` 仅由 `concludeBattle` 调用。

## 范围 & 不做

- 做：胜方回城；占城（沿用 04 翻归属）；败军逐人命运（被俘/逃跑/战死，耗 RNG）；战死掉落+永久删除；君主遭劫链（AI 自动立新君 / 玩家手动选新君的暂停态 + 续跑月末 / 灭亡）；新君忠诚 100 + 势力换主；城市战损；覆盖式粮草合并；删除 `executeCampaign` 速算死代码。
- 不做（本切片）：8.9 结构化战后事件流（core 不产消息，留 UI/后续）；通用 pendingChoice 抽象（只做窄"待选新君"）；统一全国的全局胜负 / Game Over 流程；AI 主动出征与 AI 战斗行动（沿用 12/13，玩家防守仍不可达）；俘虏的赎回/释放等新处置（沿用既有招降/处斩/流放/夺回）；占城对 >10 守军中未参战者的特殊处理（沿用 keep 模型不变）。

## 验收标准

- [ ] 胜方所有存活单位（含 0 兵未战死者）战后位于目标城；攻方胜则目标城 `lordId` 改攻方（仅翻归属，守军 lordId/cityId/兵不被占城逻辑改动）。
- [ ] 占城发生在败军处理之前：败方逃跑只逃向其势力**其余存活城**；占下其最后一城则该势力无城可逃 → 逃跑必失败。
- [ ] 败军逐人：`RandInt(0,99) > 有效智力` 直接被俘；否则有城则随机选一座落城逃跑成功；无城则逃跑失败，`RandInt(0,99)===0` 战死、否则被俘。
- [ ] 被俘：lordId 不变、cityId=目标城、兵清零。战死：装备道具并入目标城仓库且 `discovered=true`、officer 从 `officers` 删除。
- [ ] 君主被俘/战死且势力有城：AI 自动立有效智力最高（平局 id 最小）新君、忠诚 100、势力城+未被俘武将归新君；玩家则挂起待选、手动兑现后同样换主并续跑月末，且不 Game Over。
- [ ] 君主被俘/战死且无可立武将（或无城）：势力灭亡（剩余皆俘虏）。
- [ ] 目标城无条件战损：农/商/金 `floor(×0.95)`、民忠 `floor(×0.90)`。
- [ ] 粮草合并：目标城粮被覆盖为双方剩余战场粮之和（非累加）。
- [ ] `resolveCampaignOutcome` 消耗 RNG，整局同 seed 可复现；AGENTS 红线已修订。
- [ ] `executeCampaign` 速算分支与相关死代码已删除；既有经营循环/战斗流程行为不受影响。
