# main-flow-ui PRD

## 问题

`19-store-ui` 接通了 `ui → store → core`，但 UI 只是**列表版占位**：选剧本/君主是两个下拉框，经营是「城池列表 + 武将列表 + 指令面板」的三栏堆叠，且**没有大地图、没有交互式战斗界面**（战斗只能靠暂停态最小占位推进）。这套界面无法体现「在地图上经营、统一全国」的玩法，也不适合移动端操作。需要做一套**完整的主流程界面**，覆盖从开局到战斗的全链路，移动端横屏优先、简洁好操作。

## 目标

用全量 Tailwind + shadcn/ui 重建一套**移动端横屏优先**的主流程界面，串起 `选剧本 → 选君主 → 大地图经营 → 战斗地图` 四段，让玩家能在网格大地图上选城、下令、出征，并在交互式战斗地图上逐日操作打完一场仗。**本切片只做 UI 层**（store/core/feedback 规则不动），所有需要美术的地方先用**带语义的色块/网格**代替，结构与配色先定下来、图片后补。

## 关键行为 / 用户故事

1. 作为玩家，开局走一个两步卡片流程：先在**剧本卡片页**点选一个剧本，再进**君主卡片页**点选一位君主，确认后开始对局。
2. 作为玩家，进入对局后我看到一张 **12×9 网格大地图**：每座城是一个色块（颜色=所属势力、空城为灰、我方势力高亮），城与城之间画**邻接连线**，点色块即选中该城。
3. 作为玩家，选中**我方城**后进入该城经营：先从**指令面板选一条命令**，再据该命令**选执行武将 / 选目标**（城→命令→选人），下令后看到中文 toast 反馈。
4. 作为玩家，选中**敌城 / 空城**时只看其可见信息，并可对其发起需要目标城的指令（如出征、侦察）。
5. 作为玩家，我能点「结束策略（月末）」推进一个月，月末多条结算事件以中文 toast 呈现。
6. 作为玩家，当我方出征/被攻打触发玩家参战时，进入**全屏战斗地图**：32×32 地形色块网格、双方单位为带标签色块，我能选将→移动→攻击/施法/休息→结束当日，逐日推进直到分出胜负，结束后回到大地图。
7. 作为玩家，进入暂停态（待选新君 / 待选守军）时，界面用对应的选择面板让我把流程推下去、不卡死。
8. 作为移动端玩家，竖屏时看到「请旋转至横屏」提示遮罩，横屏即正常游玩；触控目标足够大、布局在常见手机横屏尺寸下不溢出。

## 界面结构与配色（先定结构色，图片后补）

- **整体**：横屏优先，顶部信息条（纪年/君主/月末/新游戏）+ 主区域；竖屏遮罩提示旋转。
- **开局**：剧本卡片页 → 君主卡片页（卡片即色块 + 文字，可点选高亮）。
- **大地图**：`12×9` 网格画布，城=色块节点（势力色），邻接=连线，选中城高亮；底部/侧边为选中城的信息与指令区。
  - 势力色：按君主 id 自动分配 HSL 色相；**我方势力统一高亮**（金边/蓝底类），**空城灰色**。
- **战斗地图**：全屏，`32×32` 地形色块网格（缩放铺满屏高、可平移）；单位=带简短标签的色块（按攻/守两方区分色），选中单位与可达/可攻击格高亮；侧边为当前单位动作（移动/攻击/施法/休息/结束当日）与天气/日数/双方概览。
  - 地形语义色（占位）：草地浅绿 / 平原淡黄 / 山地棕褐 / 森林深绿 / 村庄土黄 / 城池灰石 / 营寨赭棕 / 河流蓝。
- 所有「以后会换图」处（城节点、武将头像、地形格、道具/兵种图标）本切片一律用上述语义色块 + 文字标签。

## 范围 & 不做

- 做：
  - 用 Tailwind + shadcn/ui 重建 UI 组件层，**保留 `store` 与 feedback/messages 文案映射不变**（替换 app-shell / new-game / pause-dialogs 等界面）。
  - 两步开局卡片流程、12×9 网格大地图（势力色块 + 邻接连线 + 选城）、城→命令→选人的经营交互。
  - **完整交互式战斗地图 UI**：32×32 地形网格、单位色块、选将/移动/攻击/施法/休息/结束当日，接通 `activeBattle` 全流程；接通待选守军（`pendingDefense`）与待选新君（`pendingSuccession`）暂停态。
  - 移动端横屏布局 + 竖屏旋转提示遮罩。
- 不做：
  - 任何美术资源（图片/精灵/动画/音效）——一律色块占位，留后续补图切片。
  - localStorage 存档/读档——沿用 `19` 决定，留后续切片；刷新即丢、`newGame` 重置。
  - 任何游戏规则 / 中文运行时串以外的文案逻辑写进 store/ui（红线，中文只在 UI 映射模块）。
  - 改动 `core` / `store` / `data` 规则与接口（如确需新增只读 selector，经 `store/selectors.ts` 暴露，不写规则）。
  - 桌面端精细适配、主题/换肤、多语言。

## 验收标准

- [ ] 引入 Tailwind + shadcn/ui 并跑通构建；旧占位界面（列表版 app-shell 等）被新流程替换。
- [ ] 开局能走「剧本卡片 → 君主卡片 → 开始」两步流程并成功创建对局。
- [ ] 大地图按 `12×9` 坐标渲染城池色块，势力色正确（我方高亮、空城灰），相邻城之间有连线，点选城可切换选中。
- [ ] 选中我方城后能按「先选命令、再选执行人/目标」完成一轮经营，成功/失败均有中文 toast；不可下的命令置灰并给原因。
- [ ] 点月末能推进一个月并把多条结算事件渲染为 toast。
- [ ] 玩家参战时进入全屏 32×32 战斗地图，能完成选将→移动→攻击/施法/休息→结束当日的逐日操作，直至分出胜负并返回大地图。
- [ ] 待选守军、待选新君暂停态有对应选择界面，能把流程推进、不卡死。
- [ ] 竖屏显示旋转提示遮罩；横屏在常见手机横屏分辨率下布局不溢出、触控目标可点。
- [ ] store/ui 不含游戏规则；中文文案仍只落在 UI 映射模块。

## 待定决策

- 32×32 战斗地图在小尺寸横屏上的具体呈现（铺满屏高的格子偏小）：暂定「缩放铺满屏高 + 可平移」，开发期按真机手感微调是否加缩放/小地图。
- 大地图邻接连线在城密集处可能视觉拥挤：暂定全画，开发期视效果决定是否仅高亮选中城的邻接。
