# city-disaster PRD

## 问题
城市目前只有「正常」一种隐式状态，缺少天灾人祸带来的经营波动与风险管理。玩家无法通过预防与治理在「防灾投入」与「灾后损失」之间做权衡，经营循环缺少负反馈与变数。

## 目标
为城市引入灾害状态（饥荒/旱灾/水灾/暴动）与「防灾值」，让灾害每月有概率发生、按状态造成持续破坏、并能自然或经「治理」指令恢复正常；玩家可用治理在灾害爆发时止损并逐步提高防灾值降低未来风险。

## 关键行为 / 用户故事

### 城市状态与防灾值（新增城字段）
1. 每座城新增**状态**：`正常 / 饥荒 / 旱灾 / 水灾 / 暴动`，单值存储，初始 `正常`（fixture 播种）。
2. 每座城新增**防灾值**：`[0, 100]` 整数，初始由 fixture 播种；越高越不易发生新灾、越快从旱灾/水灾恢复。

### 治理指令（占人 ✓ · 效果时机=下令瞬间，执行人月末回城）
3. 作为君主，我要对受灾或防灾不足的城下「治理」，以便立即解除灾害并提升防灾值。
4. **前置条件**（任一不满足则禁止下令）：
   - 城里至少 1 名在任武将（执行人）。
   - 城金 ≥ 50。
   - 执行人体力 ≥ 8。
   - **不可在「已正常且防灾值 = 100」时下令**（避免浪费）。
5. **下令瞬间生效**（不入月末待执行队列）：
   - 状态强制改为 `正常`（任何灾害立刻解除）。
   - 防灾值 `+= RandInt(1, 4)`，封顶 100。
   - 扣执行人体力 8、扣城金 50。
   - 执行人离城（busy），月末回出发城。

### 月末·灾害破坏结算（异常城，按状态破坏表）
6. 月末对**异常状态**的城，按其状态扣减对应数值；**所有项按当前值的百分比扣除**，新值 = `floor(当前值 × (1 − 百分比))`，「减半」即 50% 情形 `floor(当前值 / 2)`（统一向下取整，与既有掠夺 ravage 一致）。
7. 破坏表（含 AI 城）：

| 项目   |   饥荒   | 旱灾 | 水灾 |   暴动   |
| ------ | :------: | :--: | :--: | :------: |
| 粮食   |    —     | -5%  | -5%  |   -5%    |
| 商业   |   -5%    |  —   | -10% |   -5%    |
| 金钱   |    —     |  —   | -10% |   -5%    |
| 民忠   |   -5%    |  —   |  —   |   -10%   |
| 后备兵 | **减半** | -25% | -25% | **减半** |
| 人口   |   -25%   | -25% | -25% |    —     |
| 农业   |   -5%    | -5%  | -5%  |   -5%    |

### 月末·灾害生成（仅正常城）
8. 月末对每座**正常**且有归属的城判定一次（含 AI 城；无归属空城跳过）：
   - 掷 `R = RandInt(0, 99)`；若 `R ≤ 防灾值` → 本月无灾。
   - 否则进入灾种判定，掷 `RandInt(0, 4)`：

     | 灾种值 | 结果 |
     | ------ | ---- |
     | 0 | 旱灾 |
     | 1 | 水灾 |
     | 2 | 再掷 `RandInt(0, 99)`；若 > 民忠 → 暴动；否则无事 |
     | 3 | 无事 |
     | 4 | 无事 |

   - 直观：防灾越高越难进入灾种判定；旱灾/水灾纯看防灾，与民忠无关；暴动越低民忠越易触发。

### 月末·灾害恢复（仅异常城，每月判定一次）
9. **饥荒**：`粮食 > 0` 即刻恢复正常（不耗 RNG）。
10. **旱灾 / 水灾**：掷 `RandInt(0, 99)`，`< 防灾值` 则恢复（防灾越高越快）。
11. **暴动**：掷 `RandInt(0, 99)`，`< 民忠` 则恢复（民忠越高越快）。

### 月末顺序（纯追加，不动既有步骤）
12. 固定顺序扩展为：`待执行指令 → 收粮/收税(settle，仍按旧月) → 回城+体力恢复 → 月份+1 → 登场 → 灾害破坏结算 → 灾害生成/恢复`。
    - 灾害两步在最末：饥荒可借当月收粮翻身；灾害本身不看月份/年份。
    - 同月内对异常城**先破坏后判恢复**（先吃当月损失，再判是否回正常）。
    - 治理使城本月先变正常，但当月生成步仍可能再掷出新灾——可接受。

### 与既有机制的协同
13. 治理提升防灾值 → 降低未来发灾概率、加速旱灾/水灾恢复；出巡提升民忠 → 降低暴动概率、加速暴动恢复。二者构成「防灾 vs 防乱」两条投入线。

## 范围 & 不做
- 做：
  - `City` 新增 `状态` 与 `防灾值` 两个存储字段（fixture 播种）。
  - 治理指令（下令阶段，归 economy）；归属校验（只能治理己方城）留 store 派发口。
  - 月末灾害破坏结算、灾害生成、灾害恢复（对全部有归属城，含 AI）。
  - 破坏/生成/恢复均确定性、可注入 RNG、可单测。
- 不做：
  - AI 主动下「治理」（AI 下令本阶段仍空步；AI 城同样吃破坏/生成/恢复）。
  - 灾害的 UI 表现细节（状态文案、图标）超出本 PRD 数值规则之外的视觉打磨。
  - 灾害对战斗/胜负判定的额外耦合（仅通过既有派生量如后备兵/民忠间接影响）。
  - 新增除上述两字段外的其它城字段，或把破坏/恢复系数做成可配置旋钮（系数=规则身份，内联）。

## 验收标准
- [ ] `City` 含状态（5 取值）与防灾值（`[0,100]`）字段，fixture 初始为正常 + 播种防灾值。
- [ ] 治理：满足前置时下令瞬间把状态改正常、防灾 `+RandInt(1,4)` 封顶 100、扣体力 8/城金 50、执行人 busy 月末回城；任一前置不满足或「已正常且防灾=100」时 no-op。
- [ ] 治理不进 `pendingCommands`（效果即时）。
- [ ] 月末破坏：四种异常状态按破坏表 `floor(当前×(1−百分比))`（减半即 50%）扣减，正常城不受破坏。
- [ ] 月末生成：仅正常城按 `防灾值`→灾种→（暴动看民忠）判定，结果可复现。
- [ ] 月末恢复：饥荒看粮食>0、旱灾/水灾看防灾、暴动看民忠，按规则恢复。
- [ ] 月末顺序为「…→ 月份+1 → 登场 → 破坏 → 生成/恢复」，且收粮/收税日历（6/10、3/6/9/12）与既有测试不变。
- [ ] core 纯函数 + 可注入 RNG，新增逻辑有单测覆盖关键分支与边界。
