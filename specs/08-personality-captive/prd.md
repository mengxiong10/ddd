# personality-captive（性格系统 / 招降 / 处斩 / 流放）PRD

## 问题

- 武将与君主没有「性格」维度，既无法表达个体差异，也缺少"招降难易""外交倾向"这类玩法的数据基础。
- 俘虏目前是「死状态」：占领后守军派生成俘虏，但玩家对俘虏没有任何处置手段——既不能收编、也不能清除、也不能放走，俘虏只能干耗在城里。

## 目标

- 给每名武将加一个 `0-4` 的单值「性格」；普通武将与君主用**两套** 5 性格表解读**同一个值**。本切片：普通武将性格驱动招降难度；君主性格仅作展示占位。
- 新增三条俘虏处置指令——**招降**（收编俘虏归己）、**处斩**（永久删除俘虏并退还其道具）、**流放**（把人扔成在野、落到随机城），形态与既有 `canX`/`X` 两段一致，经 `game.apply` 分派。

## 关键行为 / 用户故事

1. 作为玩家，我能看到每名武将/君主的性格（按其身份显示对应那套 5 性格之一）。
2. 作为玩家，我能对本城俘虏下「招降」：派一名在任武将执行，月末按智力差 + 性格四关判定，成功则该俘虏归顺本势力、成为可用武将。
3. 作为玩家，我能对本城俘虏下「处斩」：立即永久删除该俘虏，其所装备道具退回本城仓库并标记已发现。
4. 作为玩家，我能对本城一人（在任武将或俘虏皆可）下「流放」：立即把他变成在野，从本城移除并随机丢到一座城。
5. 作为玩家，上述任一指令前置不满足时，下令被拒绝、状态不变。

## 规则细节

### 性格（Personality）

- 武将新增存储字段 `personality ∈ {0,1,2,3,4}`，fixture 播种。
- **同一值两套解读**：`officer` 为君主（`lordId===自身`）按**君主性格表**，否则按**普通武将性格表**。
- **重选君主**使某武将转为君主时，其同一 `personality` 值自动改用君主表解读（派生涌现，不另存）。
- 普通武将性格（**0 忠义 / 1 大志 / 2 贪财 / 3 怕死 / 4 卤莽**）——本切片用于招降难度。
- 君主性格（**0 和平 / 1 大义 / 2 奸诈 / 3 狂人 / 4 冒进**）——本切片**仅展示，无任何下游规则**（同「人口」占位，明知 YAGNI 仍保留字段，待 AI / 外交切片赋义）。

### 招降（Suborn，占人 ✓ · 效果=月末 · 入 pendingCommands［suborn 分支］）

**目的**：让本城的俘虏归顺本势力。

- 作用城 = 执行人所在城（`officer.cityId`）。指令额外带真输入 `captiveId`（被招降的本城俘虏，类比 scout 的 `targetCityId`）。
- 前置条件（`canSuborn`）：本城有俘虏（目标）；本城有在任武将（执行人）；执行人体力 ≥ 15；城金 ≥ 100；目标为该城俘虏、执行人为该城在任武将。
- **下令阶段**（立即扣减，不消费 RNG）：
  - 执行人体力 `-= 15`、城金 `-= 100`。
  - 执行人离城（`busy`）；入 `pendingCommands`（`suborn` 分支，记执行人 `officerId` + 目标 `captiveId`）。
- **执行阶段**（月末，非 campaign 趟、按入队序，消费 RNG）四关：
  1. **智力差关**：`阈值 = 执行人有效智力 − 目标有效智力 + 50`（双方均取 `effectiveOfficer` 有效智力，遵架构红线）。掷 `R1 = RandInt(0, 99)`，若 `R1 > 阈值` → **失败**（忠诚不变）。
  2. **读性格系数 S**（按目标普通武将性格）：忠义 = 1、卤莽 = 2、大志 = 3、贪财 = 4、怕死 = 5（即按 `personality` 取系数表 `[1, 3, 4, 5, 2]`）。
  3. **降忠诚**：读目标**原始忠诚** `L0`（直接取 `officer.loyalty` 存储值，**不走 `officerLoyalty` 派生**——故被俘君主按其 raw 忠诚处理）。智力关通过后**恒扣** `floor(L0/10)` 并**持久化**（即便后续判定失败也保留，使来月更易招）；降后忠诚 `L = L0 − floor(L0/10)`。**若 `L0 > 60` → 直接失败**（扣减仍保留）。
  4. **终判**：`失败阈值 = floor(L / S)`；掷 `R2 = RandInt(0, 99)`，若 `R2 < 失败阈值` → **失败**，否则 → **成功**。等价：`S` 越大（怕死/贪财）成功率越高，`L` 越高失败率越高。
- **成功效果**：
  - 目标 `lordId` 改为执行人君主（`执行人.lordId`）→ 与所在城同势力，派生上不再是俘虏，成为该城在任武将。
  - 目标忠诚改写为 `RandInt(40, 79)`（消费 RNG）。
- **执行结束**：执行人月末回出发城（`busy` 月末翻回）。

### 处斩（Behead，占人 ✗ · 效果=下令瞬间 · 不入队）

**目的**：永久消除本城俘虏。

- 作用城 = 目标俘虏所在城。前置条件：城里至少 1 名俘虏；目标为该城俘虏。
- 效果（立即，无 RNG、无体力/金成本、不占任何武将）：
  - 目标若装备道具：其所持**全部道具**归属改回本城（`holder = 城`），并标记 `discovered = true`。
  - 该武将从 `state.officers` **永久删除**。

### 流放（Banish，占人 ✗ · 效果=下令瞬间 · 不入队）

**目的**：把人扔出去，使其变成在野状态出现在随机城。

- 作用城 = 目标所在城。前置条件：城里有目标——**在任武将或俘虏皆可**；目标**不是「在任君主」**（非俘虏且 `lordId===自身`，即某活势力的现任君主，含玩家自己）；**被俘君主可被流放**。
- 效果（立即，消费 RNG 选城；不占人、无体力/金成本）：
  - 目标变在野：`lordId = null`（Wandering）。
  - **目标所持道具随人带走**：道具归属保持 `holder = officer`（指向该武将），不退回本城、不改 `discovered`；他带着装备成在野（来日经搜寻被招募时一并回归）。
  - 在**全部城**中等概率随机选一座（可能含原城），目标 `cityId` 改为该城。

### 归属（己方）校验口径（store 派发口，core 不校验归属）

- **招降**：按**执行人**归属（`执行人.lordId === playerLordId`）——沿用既有占人指令口径。
- **处斩 / 流放**：按**作用城**归属（`city.lordId === playerLordId`）——因目标可能是敌方俘虏、无己方执行人；流放己方在任武将时该城本就己方，口径一致。

### 月末执行顺序（沿用既有，不改）

`pendingCommands（非 campaign 趟新增 suborn，按入队序；再 campaign）→ settle（收粮/收税）→ 回城+体力恢复 → 月份+1`（→ 登场，若 06 已实现）。招降在第一趟随其它非 campaign 指令按入队序执行。

### 数值 & 归类（依总纲取舍）

- **扁平成本 → 平衡旋钮（config）**：招降体力 15 / 城金 100。
- **公式系数 / 阈值 / 概率 → 规则身份（内联领域常量）**：智力差安全线 `+50`；招降性格系数表 `[1,3,4,5,2]`；降忠诚比例 `1/10`；招降门槛忠诚 `60`；招降成功忠诚区间 `RandInt(40,79)`。
- **量纲上限（内联）**：忠诚 `[0,100]`。
- 门槛字面量：俘虏 `≥1`、在任武将 `≥1`。

## 范围 & 不做

- 做：
  - `Officer` 新增 `personality` 字段（fixture 播种，0-4）；普通武将/君主两套解读（派生/UI 展示）。
  - 招降、处斩、流放三条指令的 `canX` 校验 + 变更，经 `game.canApply/apply` 分派；招降入 `pendingCommands`（`suborn` 分支）月末四关判定。
  - 处斩永久删除武将 + 退还其道具（标记已发现）；流放置在野并随机落城（道具退回本城）。
  - store 派发口：招降按执行人归属、处斩/流放按作用城归属校验己方。
  - UI：性格展示；三条指令的下令入口（选俘虏 / 执行人 / 目标）。
- 不做（本切片）：
  - **君主性格的任何下游规则**（AI 倾向、外交、打仗偏好）——仅展示占位（用户已确认明知 YAGNI 仍保留）。
  - 离间 / 策反 / 在野招揽 等其它会用到性格的玩法（招降之外）。
  - AI 使用这三条指令（AI 切片再接）。
  - 性格的运行时变化（性格静态，不成长、不漂移）。

## 依赖（与 06-debut-search 的基建耦合）

- **流放置「在野」依赖 Wandering 概念**（`Officer.lordId` 可空 `null`）——目前 `lordId` 为非空 `OfficerId`。
- **处斩 / 流放「标记已发现」依赖 `Item.discovered` 字段**——目前 `Item` 无该字段。
- **流放随机落城**与 06 登场落城同型。
- → 若 `06-debut-search` 尚未实现，本切片需先引入 `lordId` 可空 + `Item.discovered`（或与 06 合并实现）。

## 验收标准

- [ ] 每名武将有 `personality ∈ {0..4}`；君主按君主表、非君主按普通表解读**同一值**；重选君主后新君主自动用君主表解读其原值。
- [ ] 招降：城有俘虏、有在任武将、执行人体力 ≥ 15、城金 ≥ 100 时可下令；下令即扣体力 15 / 城金 100、执行人 `busy`、入 `pendingCommands(suborn)`。前置任一不满足时 no-op。
- [ ] 招降月末四关：①智力差关 `RandInt(0,99) > (执行人有效智力 − 目标有效智力 + 50)` 失败；②性格系数取 `[1,3,4,5,2]`；③读原始忠诚 `L0`、扣 `floor(L0/10)` 并持久化（即便失败也保留），`L0 > 60` 直接失败；④终判 `RandInt(0,99) < floor(L/S)` 失败、否则成功。成功：目标 `lordId` 改为执行人君主、忠诚 `= RandInt(40,79)`；执行人月末回城。
- [ ] 处斩：城有俘虏时可下令；立即删除该武将、其道具退回本城并 `discovered=true`；不占人、不耗体力/金/RNG。前置不满足时 no-op。
- [ ] 流放：城有目标（在任武将或俘虏，**非在任君主**）时可下令；立即置 `lordId=null`、随机落到一座城、**其道具随人保留**（`holder` 仍指向该武将，不退城、不改 `discovered`）；不占人。前置不满足时 no-op。
- [ ] 处斩 / 流放经 store 按作用城归属（`city.lordId === playerLordId`）校验己方；招降按执行人归属校验。`lordId` 不作为 Action 字段。
- [ ] 月末顺序不变：招降（suborn）在第一趟（非 campaign）按入队序执行，仍先于 settle 与回城。
- [ ] 既有指令与循环（经营/掠夺/侦察/出征/收粮收税/体力恢复/跨年/重选君主）行为不受影响。
- [ ] 给定相同 RNG 种子，整段推进结果可复现（core 纯函数、确定性；招降/流放消费 RNG）。
