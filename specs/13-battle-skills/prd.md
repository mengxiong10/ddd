# battle-skills 战斗技能（计谋）PRD

> 在 `12-battle` 战棋骨架之上加回当时砍掉的**天气 + 技能（计谋）+ 人物状态 + MP**。
> 前置：依赖 `12-battle`（`BattleState`/`BattleUnit`/日循环/普攻/胜负）已实现。
> 大数据表（30 技能速查/倍率表/9×9 掩码）属 dev 播种数据，本 PRD 只定规则与决策，附录见根目录 `appendix-a-skill-ranges.md`。

## 问题
`12-battle` 的战棋只有「移动 + 普攻」，明确不做天气、技能、人物状态、MP——战斗缺少战术深度：没有计谋的伤害/控制/恢复/破粮，没有天气与地形对计谋的克制，单位也没有混乱/定身等状态博弈。这层是战斗的核心乐趣，需要补齐。

## 目标
给战斗加上一套**技能（计谋）系统**：每名单位有 MP（开战由公式派生），施放技能扣 MP、过成功率判定后结算（扣兵/恢复/破粮/施加状态/改天气/情报/围攻）；引入**天气**（晴/阴/风/雨/雹，逐日刷新）与**人物状态**（混乱/禁咒/定身/奇门/石阵/死亡）影响施法与行动；技能可用性与威力受天气、双方地形、目标兵种四重倍率（`0`=不可用）调制。本切片**沿用 `12-battle` 的"对手方静止、仅玩家进攻可达"**，只有玩家方单位可施法。

## 关键行为 / 用户故事
1. 作为玩家，开战时我的每名单位按公式获得初始 MP，战斗界面能看到 MP；休息（rest）每次回 1 点 MP（不超初始上限）。
2. 作为玩家，我选中一名己方存活、本日未行动、且未处于混乱/石阵的单位后，除「移动/普攻/休息」外还能选「施法」：从该单位**当前可用技能**里选一个，命中范围内选目标（自身/无目标技能不选格），施放结束本日行动。
3. 作为玩家，我的技能只有在 MP 足够、施法者不处于禁咒、且天气/施法者地形/目标地形/目标兵种四项均「允许」（对应倍率≠0）、目标在该技能 9×9 掩码范围内、目标阵营符合时才可施放。
4. 作为玩家，技能要过一次成功率判定；失败仍扣 MP 但无效果。成功则按固定倍率链结算扣兵/恢复/破粮，或施加状态/改天气/情报/触发围攻。
5. 作为系统，每天日循环开头先刷新天气、再对每个非死亡单位做一次状态判定（按概率恢复/消失），其中石阵单位每天先损兵 1/8；随后才是单位恢复可行动、查粮草、玩家行动、对手行动（no-op）、扣粮。
6. 作为系统，技能与天气/状态判定都消耗 `GameState.rng`——战斗自此**不再是零 RNG**（这是对 `12-battle` 确定性红线的有意修订）。

## 关键规则（与用户对齐结论）

### MP（技能点）
- 初始 `MP = floor((floor(智力×80/100) + floor(sqrt(武力)/2) + 等级) × 体力 / 100)`，取**有效智力/有效武力**（`effectiveOfficer`）。
- MP 只活在 `BattleUnit` 快照、开战派生、**战后不写回 Officer**（与 12 里"MP 仅供技能"一致）；初始值即上限。
- 休息 +1 MP（封顶初始值）。援兵/援军/陷阱只恢复兵力，**不恢复 MP**。
- **不建 HP 模型**：扣的仍是兵力（沿用 12），表中"不恢复 HP/MP"对本作等价于"只恢复兵力"。

### 天气
- 五种：晴/阴/风/雨/雹（对应倍率表索引 0..4）。
- 开战时天气=**风**；**第 1 天开头先刷新一次**（故实战第 1 天天气也是随机的）；此后每天日循环开头都刷新。刷新为均匀 `RandInt(0,4)`（dev 可调分布）。
- `天变(22)` 技能成功后用同一机制重新刷新天气。
- 冰雹（雹）下多数技能不可用（由各技能天气列 `0` 表达，不另设规则）。

### 技能获取与解锁
- 每个兵种自带一组**默认技能**（骑/步/弓/水/极/玄，见速查表）；默认技能随等级解锁：`已解锁数 = floor(本兵种默认技能数 × 等级 / 21) + 1`，按**默认技能列表顺序**取前 N 个，封顶本兵种默认技能数。
- **个人技能**：`Officer` 级 fixture 数据（可空），不受解锁公式限制、恒可用。
- **君主技能**：派生——`lordId===自身`（君主）额外获得 `谍报(30)`，不受解锁公式限制。
- 单位「当前可用技能」= 已解锁默认 ∪ 个人技能 ∪（是君主则 [谍报]）；每次施放仍要过 MP/状态/天气/地形/兵种/范围/阵营/成功率诸关。

### 技能范围（§6.4.2 / 附录 A）
- 每技能一张独立 `9×9` 掩码（非整块 9×9 可选），中心对齐施法者，掩码 `1` 的格才可能被选。
- 目标还要同时过「目标阵营 + 目标地形（倍率≠0）+ 目标兵种（倍率≠0）」校验。
- `天变/谍报` 等自身/无目标技能不走选格流程（掩码仅中心 `C`）。
- 30 张掩码原文见根目录 `appendix-a-skill-ranges.md`，dev 期录入为数据。

### 可用性判定（§6.4.2）
施放须同时满足：施法者非禁咒、MP 足够、目标在掩码范围内、目标阵营符合；且该技能在**天气 / 施法者地形 / 目标地形 / 目标兵种**四列对应倍率**均 ≠ 0**（`0`=不可用/无效；`50/60/80`=减效仍可用）。

### 成功率（§6.4.3）
- `施法能力 = 施法者有效智力 + 等级 + 5`；`目标抗性 = 目标有效智力 + 等级 + 5`。
- `R = RandInt(0, 目标抗性 + 19)`；`R ≤ floor(施法能力/2)` 成功，否则失败。
- 失败扣 MP、不产生效果。`天变/谍报` 无目标但仍判成功率（无目标技能的「目标抗性」取法见待定决策）。

### 效果结算（§6.4.4）
- 兵力效果固定倍率链（每步 `floor`）：`基础兵力 → ×天气% → ×目标兵种% → ×目标地形% → ×施法者地形% = 最终兵力值`。
  - 敌方技能：`实际扣兵 = min(最终兵力值, 目标当前兵力)`。
  - 友方恢复：`实际恢复 = min(最终兵力值, 目标最大带兵量 − 目标当前兵力)`，最大带兵量取 `effectiveOfficer` 的带兵量上限。
- 破粮（火箭11）用同一倍率链：`实际扣粮 = min(最终破粮值, 敌方当前战场粮草)`，扣对手方 `provisions`；不立即判负，下一天开头查粮草时才可能触发。
- 状态：成功且目标未击溃 → 目标状态改为该技能状态；已击溃则保持死亡；新状态为定身时移动力立刻降为 1。
- 特殊：`天变` 刷新天气；`谍报(30)` 揭示对手战场粮草（core 全可见，落地为 UI 揭示，core 仅判成功、不改状态）；`围攻(27)` 成功后，目标上下左右相邻的**施法者方**单位逐个对目标执行一次普攻（复用普攻结算，可致击溃并触发胜负检查）。

### 人物状态与日循环（§6.6.2）
- 建模状态：`正常 / 混乱 / 禁咒 / 定身 / 奇门 / 石阵 / 死亡`（死亡=既有击溃 `routed`）。`遁甲/潜踪` 因对应技能不纳入、不建模。
- 效果：混乱=不能行动；禁咒=不能施法（仍可移动/普攻/休息）；定身=移动力降为 1；奇门=可穿越敌方接敌停步区；石阵=不能行动且每天损当前兵力 `floor(/8)`；死亡=退出行动序列。
- **每天开头状态判定**：对每个非死亡单位掷 `判定成功 = RandInt(0,59) < floor(有效智力/2)`——混乱/禁咒/定身/石阵：成功则复正常；奇门/潜踪：失败则复正常；正常/遁甲：不变；死亡：跳过。石阵单位先损兵（可能致死亡，则跳过其后判定）。
- 日循环顺序（在 12 基础上**纯插入**天气与状态两步）：`天数+1 → 查30天上限 → 刷新天气 → 单位状态判定(含石阵损兵) → 单位恢复可行动 → 查双方粮草 → 玩家方行动 → 对手方行动(no-op) → 双方扣当日粮草`。

### 架构定调
- **施法是新的战斗 action**：沿用 12 的「移动 + 终结动作」结构，给 `act` 的终结动作增加 `cast`（技能 id + 可选目标格），与 attack/rest 同级、结束本日行动；混乱/石阵单位不可被操作、禁咒单位不可选 cast。
- **战斗自此消耗 `GameState.rng`**：`reduceBattle`/日循环读取并推进 `state.rng`（成功率、天气刷新、状态判定）。这是对 `12-battle`「战斗确定性、不耗对局 RNG」红线的有意修订——红线收窄为「无技能的 12-battle 子集确定性」。
- 沿用 12：仅玩家进攻可达、对手方静止（无 AI 施法）；技能只可由玩家方单位施放。`military/battle` 仍不反向 import `turn`。

## 范围 & 不做
- 做：
  - MP（派生/上限/rest 回复）、天气（开战风 + 逐日刷新 + 天变改写）。
  - 27 个技能（全 30 减 `21 遁甲 / 26 潜踪 / 28 急行`）：可用性四关、9×9 掩码、成功率、倍率链结算（扣兵/恢复/破粮/状态/改天气/情报/围攻）。
  - 技能解锁（默认按等级公式）+ 个人技能（fixture）+ 君主技能（谍报，派生）。
  - 人物状态（混乱/禁咒/定身/奇门/石阵/死亡）+ 每日状态判定 + 石阵损兵。
  - 日循环插入天气刷新与状态判定两步；`cast` 战斗 action 接线。
- 不做（本切片）：
  - `遁甲(21) / 潜踪(26) / 急行(28)`（当前无真实战斗效果，YAGNI 暂不纳入；对应状态也不建模）。
  - 任何 **AI 战斗施法 / AI 主动出征 / 玩家防守实战**（沿用 12，结构预留、不可达）。
  - 战争迷雾 / 情报隐藏（core 全可见，谍报落地为 UI）；HP 模型。
  - 装备覆盖技能范围/MP 加成；技能动画/音效（UI）。

## 验收标准
- [ ] 初始 MP 按公式（吃有效智力/武力）正确；rest +1 封顶初始值；援兵/援军/陷阱不回 MP；战后 MP 不写回 Officer。
- [ ] 天气：开战=风、第 1 天开头刷新、每日刷新；天变成功改写天气；冰雹下对应技能不可用。
- [ ] 解锁：`floor(默认数×等级/21)+1` 按列表顺序取数并封顶；个人/君主技能不受公式限制；君主含谍报。
- [ ] 可用性四关（天气/施法者地形/目标地形/目标兵种倍率均≠0）+ 范围掩码 + 阵营 + 禁咒 + MP 全部命中；非法 no-op。
- [ ] 成功率 `R=RandInt(0,目标抗性+19) ≤ floor(施法能力/2)`；失败扣 MP 无效果。
- [ ] 倍率链 5 步逐 floor：扣兵 `min(最终,目标兵力)`、恢复 `min(最终,上限−当前)`、破粮 `min(最终,对手战场粮草)`。
- [ ] 状态：成功施加（击溃保持死亡）、定身即降移动力 1；每日判定 `RandInt(0,59)<floor(智力/2)` 按状态组恢复/消失；石阵每日损兵 1/8 且可致死亡。
- [ ] 特殊：天变改天气、谍报判成功（core 不改状态）、围攻触发相邻友军逐个普攻并可致击溃。
- [ ] 日循环新顺序正确（天气/状态两步纯插入，不破坏 12 既有移动/普攻/粮草/胜负行为）。
- [ ] 战斗消耗 `state.rng` 且整局可复现（同 seed 同结果）；AGENTS 红线已同步修订。

## 待定决策
- **无目标技能的目标抗性**：`天变/谍报` 无目标，成功率公式的「目标抗性」无来源。倾向取 `0`（即 `R=RandInt(0,19)`，几乎必成）；dev 期确认是否改用施法者自身抗性。
- **石阵损兵与状态判定的先后**、**围攻相邻友军的遍历定序**（拟按 上/下/左/右 或 officerId 字典序）：细节留 dev。
- **天气刷新分布**：本 PRD 取均匀 `RandInt(0,4)`；若要带权重留 dev 调平衡。
- **个人技能 fixture 具体播种**（哪些武将带 22/23/24/25 等）、**9×9 掩码与 30 行倍率表的数据录入**：均属 dev/实现期数据，附录 `appendix-a-skill-ranges.md` 为掩码来源。
- **cast 是否允许先移动再施法**：本 PRD 沿用 `act` 的「可选移动 + 终结动作」结构默认允许；若需禁止留 dev。
