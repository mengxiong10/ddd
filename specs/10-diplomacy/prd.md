# diplomacy（外交：招揽 / 离间 / 策反 / 劝降）PRD

## 问题

- 玩家对**敌方在任武将与君主**没有任何非军事手段：只能靠出征硬打。缺少「不打仗就瓦解敌方」的外交玩法。
- `08-personality-captive` 已落地性格（普通武将表驱动招降难度，君主表仅占位）。君主性格至今**无任何下游规则**，需要外交切片赋义。

## 目标

- 新增四条**经营·外交指令**，复用既有 `canX`/`X` 两段 + 月末「多关判定」形态（与招降同构），经 `game.canApply/apply` 分派、入 `pendingCommands` 月末结算：
  - **招揽**：把敌方一名在任武将吸引过来，物理迁入执行人所在城、归己。
  - **离间**：小幅降低敌方一名在任武将忠诚（低收益高消耗）。
  - **策反**：让敌方某城**太守**（非君主）自立为君，带走该城从原势力分裂。
  - **劝降**：劝说敌方**君主**整体投降，吸收其全部城池与臣属（需城池压制）。
- 引入**太守（派生概念，零新字段）**为策反提供目标，并让君主性格表首次进入规则消费（劝降）。

## 关键行为 / 用户故事

1. 作为玩家，我能派一名在任武将对**任意敌城**（无需邻接）的一名敌方在任武将下「招揽」；月末按智力差/忠诚/性格三关判定，成功则该武将迁入我执行人所在城、归我势力。
2. 作为玩家，我能对敌方一名在任武将下「离间」；月末三关判定，成功仅令其忠诚 −4。
3. 作为玩家，我能对敌方某城**太守**（该城非君主的领头武将）下「策反」；成功则该太守自立为君、带走其城与同城同势力武将。
4. 作为玩家，当我方城池数 ≥ 敌方城池数 ×2 时，我能对敌方**君主**下「劝降」；月末按城池压制/智力差/君主性格三关判定，成功则吸收其全部城池与臣属。
5. 作为玩家，上述任一指令前置不满足时下令被拒、状态不变；目标在结算前已失效（被占/易主/俘获）时月末安全 no-op。

## 规则细节

### 太守（Governor，派生 · 零存储字段）

- 某城太守 = **派生查询**（`world/queries`，新增 `governorOf(state, cityId)`），不引入存储字段、不破坏单一真相源：
  - 若该城归属君主（`id === city.lordId` 的武将）此刻**正驻本城**（其 `cityId === 本城`）→ 太守 = **该君主**。
  - 否则 → 太守 = 本城**在任（serving，`lordId === city.lordId` 且非俘虏）武将中有效智力最高者**（平局取 id 字典序最小，同 `succession`）。
  - 城内无在任武将（空城/仅俘虏）→ 无太守（`null`）。
- 智力比较取 `effectiveOfficer` 有效智力（遵架构红线）。
- 太守仅服务于策反目标判定；不产生其它下游规则。

### 共同形态（四条指令）

- 均为**经营上下文（`economy/`）指令**：`占人 ✓`、`效果时机=月末`、入 `pendingCommands`（各自分支，非 campaign 趟、按入队序）。下令阶段（`canX`/`X`：校验、扣本城金、扣执行人体力、占人离城、入队）就近归 `economy/`，与既有招降同构；月末执行 `executeX` 由 `turn/pending` 按 `type` 分派。territorial 改写（招揽迁城、策反自立、劝降吸收）沿用招降把 `lordId` 写在 economy 执行内的先例，可读用 `world/queries`（`governorOf`/`citiesOfLord`/`isCaptive`），**不**触 `military`/`world/succession`。
- 本城 = 执行人所在城（`officer.cityId`，单一真相源），由本城出金。指令额外带真输入 `targetOfficerId`（被作用的敌方武将，类比招降的 `captiveId`）。
- **下令阶段**（立即扣减，不消费 RNG）：扣执行人体力、扣本城金、执行人离城（`busy`）、入 `pendingCommands`（记 `officerId` + `targetOfficerId`）。前置不满足为 no-op。
- **执行阶段**（月末，消费 RNG）：先**重校目标仍有效**（存在、仍为合法目标），失效则安全 no-op、不动 RNG（同招降守卫）。各关按下列**固定次序**掷骰（锁定，保可复现）。
- **执行结束**：执行人月末回出发城（`busy` 由回城步骤统一翻回）；招揽迁城作用于**目标**、不改执行人 `cityId`。
- 归属（己方）校验**不在 core**：按**执行人归属**（`执行人.lordId === playerLordId`）在 store 派发口校验，沿用既有占人指令口径。

### 招揽（Entice，普通武将性格表）

- **目标**：敌方在任**非君主**武将（`lordId !== null` 且 `lordId !== 执行人君主`、非俘虏、`lordId !== 自身`）。任意敌城、无需邻接。
- **前置（`canEntice`）**：执行人为本城在任武将（存在/未占用/非俘虏）；执行人体力 ≥ 20；城金 ≥ 50；存在合法目标。
- **下令**：体力 −20、城金 −50、占人、入队（`entice` 分支）。
- **执行三关**（次序锁定）：
  1. **智力差关**（**无 +50 安全线**，纯按差判定）：`阈值 = 执行人有效智力 − 目标有效智力`。掷 `R1 = RandInt(0,99)`，`R1 > 阈值` → 失败。
  2. **忠诚关**：掷 `R2 = RandInt(0,99)`，`R2 < 目标忠诚` → 失败（目标非君主，忠诚 raw === 派生；忠诚 100 必败）。
  3. **性格关**：`S = ENTICE_COEFF[目标性格]`；掷 `R3 = RandInt(0,99)`，`R3 < S` → 通过，否则失败。
     - `ENTICE_COEFF = [5, 20, 30, 40, 15]`（按 `personality` 0..4：忠义5 / 大志20 / 贪财30 / 怕死40 / 卤莽15）。
- **成功效果**：目标从原城移除、迁入执行人所在城（`cityId = 执行人.cityId`）；`lordId = 执行人君主`；忠诚 `= R4 = RandInt(40,79)`。目标所持道具随人带走（`holder` 不变）。

### 离间（Alienate，普通武将性格表）

- **目标 / 前置 / 下令**：同招揽（敌方在任非君主武将；体力 ≥ 20、城金 ≥ 50；下令体力 −20、城金 −50、占人、入队 `alienate` 分支）。
- **执行三关**（次序锁定）：
  1. **智力差关**：`阈值 = 执行人有效智力 − 目标有效智力 + 50`。掷 `R1`，`R1 > 阈值` → 失败。
  2. **忠诚关**：掷 `R2`，`R2 < 目标忠诚` → 失败。
  3. **性格关**：`S = ALIENATE_COEFF[目标性格]`；掷 `R3`，`R3 < S` → 通过，否则失败。
     - `ALIENATE_COEFF = [5, 30, 40, 30, 50]`（忠义5 / 大志30 / 贪财40 / 怕死30 / 卤莽50）。
- **成功效果**：目标忠诚 `−4`（下限 0）。无成功 RNG。
- 设计定位：低收益高消耗（20 体力 + 50 金换 −4 忠诚）。

### 策反（Instigate，普通武将性格表）

- **目标**：敌方某城**太守**且该太守**非君主**（即该敌城此刻未驻其君主 → 太守为最高智力在任武将）。`targetOfficerId` 须等于其所在城的 `governorOf`、且非君主、敌方、非俘虏。
- **前置（`canInstigate`）**：执行人为本城在任武将；体力 ≥ 20；城金 ≥ 50；存在合法太守目标（含「目标城未驻敌君主」——君主即太守，君主不可策反）。
- **下令**：体力 −20、城金 −50、占人、入队（`instigate` 分支）。
- **执行三关**（次序锁定）：
  1. **智力差关**：`阈值 = 执行人有效智力 − 目标有效智力 + 50`。掷 `R1`，`R1 > 阈值` → 失败。
  2. **忠诚关**：掷 `R2`，`R2 < 目标忠诚` → 失败。
  3. **性格关**：`S = INSTIGATE_COEFF[目标性格]`；掷 `R3`，`R3 < S` → 通过，否则失败。
     - `INSTIGATE_COEFF = [5, 60, 20, 10, 30]`（忠义5 / 大志60 / 贪财20 / 怕死10 / 卤莽30；大志最易"自立"）。
- **成功效果**（无成功 RNG，单城分裂；因君主不在该城 → 不触发重选君主）：
  - 目标自立为君：`目标.lordId = 目标.id`。
  - 目标所在城 `lordId = 目标.id`。
  - 该城**其余原势力武将**（`lordId === 原君主`者）`lordId` 改归目标。该城内属第三方的俘虏不变。

### 劝降（Induce，君主性格表）

- **目标**：敌方**君主**（`lordId === 自身` 且 `lordId !== 执行人君主`）。
- **前置（`canInduce`）**：执行人为本城在任武将；体力 ≥ 10；城金 ≥ 50；目标为敌方君主；**城池压制**——执行人君主城池数 ≥ 目标君主城池数 ×2。
- **下令**：体力 −10、城金 −50、占人、入队（`induce` 分支）。
- **执行判定**（次序锁定）：
  1. **玩家君主免疫**（无 RNG，游戏规则非归属校验）：目标 `id === state.playerLordId` → 直接失败（防止 AI 劝降玩家君主）。
  2. **城池压制关**（无 RNG，月末重校）：执行人君主城数 `< 目标君主城数 ×2` → 失败。
  3. **智力差关**：`阈值 = 执行人有效智力 − 目标有效智力 + 50`。掷 `R1`，`R1 > 阈值` → 失败。
  4. **君主性格关**：`S = INDUCE_COEFF[目标君主性格]`；掷 `R2`，`R2 < S` → 通过，否则失败。
     - `INDUCE_COEFF = [15, 5, 20, 1, 10]`（按君主性格 0..4：和平15 / 大义5 / 奸诈20 / 狂人1 / 冒进10；奸诈最易、狂人几乎不可能）。
- **成功效果**（无成功 RNG，整体吸收；先按原势力快照计算再写回）：
  - 令 `C` = 目标君主名下全部城；逐城 `lordId = 执行人君主`。
  - 凡 `lordId === 目标君主` 的武将：在 `C` 中某城者（含目标君主本人）→ `lordId = 执行人君主`（并入）；不在 `C` 中者（散落/被他处俘获）→ `lordId = null`（在野）。
  - 忠诚不改（spec 未规定，沿用现值）。

### 数值 & 归类（依总纲取舍）

- **扁平成本 → 平衡旋钮（config）**：招揽/离间/策反 体力 20、城金 50；劝降 体力 10、城金 50。
- **公式系数 / 阈值 / 概率 / 量纲 → 规则身份（内联领域常量）**：
  - 智力差安全线 `+50`（离间/策反/劝降；招揽**无**安全线）。
  - 四张性格系数表：招揽 `[5,20,30,40,15]`、离间 `[5,30,40,30,50]`、策反 `[5,60,20,10,30]`、劝降（君主表）`[15,5,20,1,10]`。
  - 招揽成功忠诚区间 `RandInt(40,79)`；离间忠诚扣减 `4`；劝降城池压制倍数 `2`。
  - 掷骰量纲 `RandInt(0,99)`；忠诚 `[0,100]`。
- 门槛字面量：在任武将 `≥1`。

## 范围 & 不做

- 做：
  - `world/queries` 新增派生 `governorOf(state, cityId)`（太守）。
  - 招揽 / 离间 / 策反 / 劝降 四条指令的 `canX` 校验 + 变更 + `executeX` 月末执行，经 `game.canApply/apply` 分派、入 `pendingCommands`（四个新分支）、`turn/pending` 月末分派。
  - 劝降首次消费**君主性格表**；玩家君主劝降免疫在 core 执行内用 `state.playerLordId` 编码。
  - store 派发口：四条均按执行人归属校验己方。
  - UI：四条指令下令入口（选执行人 / 敌城与敌方目标；策反仅列太守目标；劝降仅在城池压制满足时可下并选敌君主）。
- 不做（本切片）：
  - **AI 使用这四条指令**（AI 切片再接；劝降的「玩家君主免疫」规则本切片即落地，为 AI 接入预留）。
  - 多 AI 势力、外交关系/盟约/宣战等更复杂外交结构。
  - 离间/招揽失败时的持久副作用（招降式「失败也扣忠诚」不引入；本切片失败即纯消耗）。
  - 太守的其它下游规则（仅服务策反目标判定）。

## 依赖（既有切片基建）

- **性格表**：依赖 `08-personality-captive` 的 `Officer.personality`（0..4）与君主/普通武将两套解读。
- **在野（Wandering）**：劝降「散落武将转在野」依赖 `06-debut-search` 的 `lordId` 可空（`null`）。
- **派生俘虏 / 城池查询**：依赖 `world/queries` 的 `isCaptive`、`citiesOfLord`、`effectiveOfficer`、`officerLoyalty`。
- 月末固定顺序**不变**：四条新指令均在 `pendingCommands` 第一趟（非 campaign）按入队序执行，仍先于 settle 与回城。

## 验收标准

- [ ] `governorOf(state, cityId)`：君主在城则返君主；否则返本城在任武将中有效智力最高者（平局取 id 最小）；空/仅俘虏返 `null`。
- [ ] 招揽：体力 ≥ 20、城金 ≥ 50、有敌方在任非君主目标时可下令；下令即扣体力 20 / 城金 50、执行人 `busy`、入 `pendingCommands(entice)`；前置不满足 no-op。月末三关 ①`R1 > (执行人有效智力 − 目标有效智力)`（无 +50）失败 ②`R2 < 目标忠诚`失败 ③`R3 < ENTICE_COEFF[性格]` 通过。成功：目标迁入执行人城、`lordId=执行人君主`、忠诚 `=RandInt(40,79)`。
- [ ] 离间：同前置/下令（`alienate` 分支）；月末三关同上但智力差 `+50`、系数 `ALIENATE_COEFF`；成功仅 `忠诚 −4`（下限 0）、无成功 RNG。
- [ ] 策反：目标须为其所在城太守且非君主（敌城未驻其君主）、敌方、非俘虏；体力 ≥ 20 / 城金 ≥ 50 可下令（`instigate` 分支）。月末三关（智力差 `+50`、`INSTIGATE_COEFF`）。成功：目标 `lordId=自身`、其城 `lordId=自身`、该城其余原势力武将 `lordId` 改归目标；第三方俘虏不动；不触发重选君主。
- [ ] 劝降：仅当执行人君主城数 ≥ 目标君主城数 ×2 且目标为敌方君主时可下令；体力 ≥ 10 / 城金 ≥ 50（`induce` 分支）。月末 ①目标为玩家君主直接失败 ②城池压制重校失败 ③`R1 > 智力差+50` 失败 ④`R2 < INDUCE_COEFF[君主性格]` 通过。成功：目标君主全部城与城内臣属（含君主本人）归执行人君主、散落武将转在野（`lordId=null`）。
- [ ] 四条指令目标在月末结算前已失效（不存在/已易主/已俘获/已非合法目标）→ 安全 no-op、不动 RNG。
- [ ] 四条经 store 按执行人归属（`执行人.lordId === playerLordId`）校验己方；`lordId` 不作为 Action 字段。
- [ ] 月末顺序不变：四条新指令在第一趟（非 campaign）按入队序执行，先于 settle 与回城。
- [ ] 既有指令与循环（经营/掠夺/侦察/出征/招降/收粮收税/体力恢复/跨年/登场/灾害）行为不受影响。
- [ ] 给定相同 RNG 种子，整段推进结果可复现（core 纯函数、确定性；各关按锁定次序消费 RNG）。
