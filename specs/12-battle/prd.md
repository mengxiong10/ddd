# battle 战斗系统 PRD

> 覆盖战斗骨架：入口 / 派生属性 / 战场地图 / 日循环 / 战内结算 / 输入输出。
> 前置：依赖 `11-troop-types`（兵种、有效兵种、移动力）先实现完成。

## 问题

出征（`04-campaign`）目前在**月末**用「双方兵力总和比大小」一锤定音，没有任何战术过程：地图、地形、兵种、走位、逐日攻防全部缺席，也没有兵力损耗、经验与升级。战斗只是个占位速算，撑不起「回合制战棋」这个核心玩法。

## 目标

把出征的月末速算替换为一场**交互式逐日地图战**：读取目标城对应的战斗地图，按攻守模式摆放双方单位，玩家逐日操作己方单位移动/普攻，最多打 30 天，按粮草/主将/全灭/城池格判胜负；战后把**剩余兵力、经验、等级、占领/俘虏/重选君主**写回 `GameState`，再继续月末经营循环。本切片只做骨架——**不做天气、技能、人物状态、HP/MP、AI 战斗行动**。

## 关键行为 / 用户故事

1. 作为玩家，我下达出征后进入该敌城的**地图战**：双方单位按攻守方向与出生点摆好，城池格（胜负点）标出，每名单位按等级/武力/智力/体力/兵种/装备算出攻击力、防御力、移动力（HP/MP 本切片不做）。
2. 作为玩家，我每天可逐个操作己方「存活且当天未行动」的单位：选单位 → 看移动范围 → 选落点 → 开行动菜单（攻击 / 休息 / 查看）；攻击或休息结束本日行动，查看不结束；移动后退出菜单可回到移动前位置。
3. 作为玩家，我的普攻命中范围内的敌方单位会按公式扣**兵力**（不是 HP），兵力归零即被击溃；造成兵力变化时我的单位获得经验，满 100 升 1 级。
4. 作为玩家，我可随时**全军撤退**，立即判定玩家方失败。
5. 作为系统，每天按固定顺序推进（天数+1 → 查 30 天上限 → 单位恢复可行动 → 查双方粮草 → 玩家方行动 → 对手方行动 → 双方扣当天粮草），到达任一胜负条件即结束。
6. 作为系统，战斗分出胜负后：沿用 `04-campaign` 的战后处理——攻方胜则占城（`city.lordId` 改攻方、随军粮草并入、出征武将进驻）、败则出征武将就地成俘虏，并对被俘君主重选君主 / 灭亡；**额外**把每名单位的剩余兵力、经验、等级写回各自 `Officer`。

## 关键规则（与用户对齐结论）

### 战斗入口与主将

- 仅两种模式：**玩家进攻**（玩家攻敌城）/ **玩家防守**（敌攻玩家城）。有玩家参与才进地图战；电脑互打速算**不做**。
- 进攻方、防守方各最多 10 名武将；空槽位从开始即无效。开战后双方武将分属「玩家方」与「对手方」两阵营。
- 防守方选取：目标城归属方在城武将中，**太守领衔，其余按兵力降序（平局 id 升序）**，取前 10。
- **攻守双方各有主将**：攻方主将 = 出征名单首位；守方主将 = 防守方首位（即太守）。主将开战定格、不随减员漂移；**任一方主将被击溃立即触发该方失败**（对手方主将击溃→玩家胜、玩家方主将击溃→玩家败）。

### 派生属性（§6.3）

- 输入取**有效武力/有效智力**（`effectiveOfficer`）、等级、体力、有效兵种（`effectiveTroopType`）、装备。
- `基础攻击 = floor(武力 × (等级 + 10) × 兵种攻击系数)`；`基础防御 = floor(智力 × (等级 + 10) × 兵种防御系数)`（兵种系数见兵种表）。
- 移动力沿用 `11-troop-types` 派生（有效兵种基础移动力 + 装备加成），上限 8。
- **HP/MP 本切片不做**（HP 仅显示、MP 仅供技能，技能已砍）；兵种相克倍率与默认 5×5 普攻范围掩码按兵种取（弓兵散点 / 骑水玄十字 / 步极周身；装备可整体覆盖范围与移动力）。

### 战场地图（§6.5）

- 8 种地形：草地/平原/山地/森林/村庄/城池/营寨/河流。地形同时影响**移动消耗**（按兵种查表）、**攻击/防御折减档**（无/中/重/极重 = `/1 /2 /4 /8`，按兵种查表）、**防御系数**（按地形查表）。
- `攻击力 = floor(基础攻击 / 2^折减档)`；`防御中间值 = floor(基础防御 / 2^折减档)`；`防御力 = floor(防御中间值 × 地形防御系数)`。
- **城池格**既是地形（防御系数 1.5）也是胜负点：玩家进攻进入即胜、玩家防守对手进入即败。
- 棋盘 **32×32**；地图领域数据只保存行主序地形格。唯一 `city` 地形即城池格/胜负点；攻方出生点按出发城相对目标城的八方向、守方出生点按城池格，在开战时依原版阵形表动态计算。fixture 播种少量模板地图，所有城可复用/指派；不逐城独立画。

### 日循环（§6.6，最多 30 天）

顺序：天数+1 → 查 30 天上限 → 所有单位恢复可行动 → 查双方粮草 → **玩家方行动** → **对手方行动** → 双方扣当天粮草。

- 行动后扣粮到 0 不立刻判负，下一天开头检查时才生效；每单位每天最多行动一次。
- 移动范围 = 基础移动力 − 路径各格地形消耗；存活单位占格不可进入；友方只限制落点不阻路径；**敌方四邻形成接敌停步区**，进入后剩余移动力压到 1、不可继续穿越。
- 普攻用兵种默认 5×5 范围（装备可覆盖），范围内有敌即可发动。

### 战内结算（§6.7）

- 普攻：`基础伤害 = floor(攻击力 / 防御力 × floor(攻击者当前兵力 / 8))`；`最终伤害 = floor(基础伤害 × 兵种相克倍率) + 10`；`实际扣兵 = min(最终伤害, 目标当前兵力)`。兵力归零 → 击溃 → 触发胜负检查。
- 粮草：双方进入战斗各带**战场粮草**（默认：攻方=随军粮草、守方=目标城开战快照城粮）；每天行动后 `每日耗粮 = floor(sqrt(本方未击溃单位总兵力) / 3)`，不超当前战场粮草。
- 经验：`等级差 = 攻击者等级 − 目标等级`；`伤害经验 = floor(sqrt(实际兵力变化) / 4)`；攻击者较低 `基础经验 = 伤害经验 − 等级差 + 2`，否则 `max(伤害经验 − 等级差, 0) + 2`；击溃额外经验（低/平/高 = 24/16/8）。经验满 100 扣 100、等级+1，一次只升 1 级。`Officer` 新增 `experience` 存储字段（fixture 播种）。
- 胜负判定表（触发点：击溃时 / 每日开头查粮草 / 有人进城池格 / 第 30 天结束 / 全军撤退）：

| 条件                                        | 结果            |
| ------------------------------------------- | --------------- |
| 超 30 天且玩家进攻                          | 玩家方失败      |
| 超 30 天且玩家防守                          | 玩家方胜利      |
| 对手方粮草=0 / 玩家方粮草=0                 | 玩家胜 / 玩家败 |
| 对手方主将击溃 / 玩家方主将击溃             | 玩家胜 / 玩家败 |
| 某方全灭                                    | 另一方胜        |
| 玩家进攻进入城池格 / 玩家防守对手进入城池格 | 玩家胜 / 玩家败 |
| 玩家全军撤退                                | 玩家方失败      |

双方粮草同日归零按玩家方失败。

### 架构定调

- 战斗是 `apply(state, action)` 之上的**交互式子对局**：出征/被攻打时**暂停经营月末**，进入独立的 `BattleState` + 一组战斗专属 action（选单位/移动/攻击/休息/查看/撤退/对手方推进/天结束），分出胜负后把结果写回 `GameState` 再继续月末。战斗**确定性、不耗对局 RNG**（公式全 floor、本切片对手不行动）。
- 战斗替换 `04` 月末速算里「比兵力总和定 `attackerWins`」这一步；其后的占城/俘虏/重选君主/随军粮草沿用既有 `military/campaign` + `world/succession`，**额外**写回每单位剩余兵力/经验/等级。

## 范围 & 不做

- 做：
  - 战斗入口（玩家进攻/防守两模式、≤10/方、主将判定）。
  - 派生属性（攻击力/防御力/移动力/相克/普攻范围；HP/MP 除外）。
  - 战场地图（8 地形、移动消耗/折减/防御系数、城池格胜负点、32×32、少量模板地图 fixture）。
  - 日循环（≤30 天、移动+接敌停步、普攻）、玩家逐日交互。
  - 战内结算（普攻伤害、每日粮草、胜负判定、经验/升级）。
  - `Officer.experience` 字段 + fixture 播种；战后把兵力/经验/等级/占领/俘虏/重选君主写回。
- 不做（本切片）：
  - **天气、技能（计谋）、人物状态（混乱/石阵/定身等）、HP/MP**。
  - **任何 AI 战斗行动**：对手方单位本切片**静止**（日循环「对手方行动」为 no-op）；故实战中仅**玩家进攻**会发生，防守模式引擎结构上支持但需 AI 主动出征 + AI 战斗行动落地后才可达。
  - **AI 主动出征**（沿用 `04`，留 AI 切片）→ 玩家防守本切片不会被触发。
  - 电脑互打速算；战后粮草的独立 `08-aftermath` 处理（沿用 `04`：胜并入/败损失）；多跳路径/拦截；伤兵/补员/兵种成长/转职。

## 验收标准

- [ ] 出征触发地图战：读模板地图，从 `tiles` 判断城池格，按运行时攻击方向动态计算原版攻守出生点并摆 ≤10/方单位，算每单位攻击/防御/移动力（无 HP/MP）。
- [ ] 派生公式正确：基础攻击/防御按兵种系数、地形折减档（/1 /2 /4 /8）、地形防御系数；移动力上限 8、接敌停步区压到 1。
- [ ] 普攻伤害 = `floor(攻击力/防御力 × floor(兵力/8))`，×相克 +10，扣兵 ≤ 目标当前兵力；归零即击溃并触发胜负检查。
- [ ] 日循环按固定顺序推进、≤30 天；每单位每天最多行动一次；攻击/休息结束本日、查看不结束、退出菜单回原位。
- [ ] 每日耗粮 = `floor(sqrt(总兵力)/3)`、不超战场粮草；下一天开头才据粮草判负。
- [ ] 胜负判定表全部命中（含 30 天上限按攻守、主将击溃、全灭、城池格、撤退、同日粮草归零按玩家败）。
- [ ] 经验/升级公式正确、满 100 升 1 级（一次一级）；`Officer.experience` 持久化。
- [ ] 战后写回：剩余兵力/经验/等级落到各 `Officer`；占城/俘虏/重选君主与 `04` 一致。
- [ ] 对手方单位本切片不行动；战斗确定性、不耗对局 RNG；既有经营循环行为不受影响。
