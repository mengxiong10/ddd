# campaign（出征）PRD

## 问题
现有指令都在本城内作用，武将从不跨城移动，城与城之间也没有任何地图关系。但游戏的核心目标是"占领州郡、统一全国"——必须有一条**派兵进攻敌城并占领**的指令。出征是项目第一条引入以下新概念的功能：

- **城邻接**（地图拓扑，此前完全没有）；
- **武将跨城移动**（此前武将永远留在本城）；
- **战斗与占领**（城易主）；
- **俘虏**（败方武将的去向）。

## 目标
在月度循环上新增一条出征指令：玩家从本城派最多 10 名在任武将、携随军粮草进攻一座**相邻敌城**；下令时即扣随军粮草并占用武将，**月末**结算一场（本切片从简的）战斗——按双方兵力总和定胜负，胜方占城、败方武将就地成俘虏；若某势力君主被俘且仍有城，则重选君主。战斗结算先从简，后续切片再深化为战棋。

## 关键行为 / 用户故事
1. 作为玩家，我可以从本城选最多 **10 名在任武将**、输入**随军粮草**（1 ~ 本城城粮）、指定一座**相邻敌城**下达出征：立即扣本城城粮（随军粮草）、占用这些武将（busy），出征命令入月末待执行队列。
2. 作为玩家，出征前置任一不满足（本城无在任武将 / 城粮 ≤ 0 / 目标城是本城或己方城 / 目标城与本城不相邻 / 随军粮草不在 1~本城城粮 / 选了 0 或 >10 名武将 / 选中武将不全在本城或不在任）时，下令被拒绝、状态不变。
3. 作为系统，战斗在**月末**结算（先于收粮/收税），而非下令当下：按**双方兵力总和**比大小——攻方=出征武将兵之和；守方=目标城"该城归属方"武将兵之和 + 城后备兵（**排除城内已有俘虏**）。攻方兵力 **严格大于** 守方兵力才算攻方胜，否则守方胜（含平局守方胜）。
4. 作为系统，攻方胜：目标城归属改为攻方君主（连同其农业/商业/资源/后备兵），随军粮草并入该城城粮，出征武将进驻该城（cityId 改为目标城），原守军武将就地成为俘虏（派生：其 lordId 已 ≠ 所在城 lordId）。
5. 作为系统，攻方败：目标城归属不变，随军粮草损失，出征武将停留在目标城（cityId 改为目标城），因目标城归属方非攻方，这些出征武将就地成为俘虏。
6. 作为系统，结算后若某势力**君主被俘**（君主所在城归属 ≠ 君主自身 lordId）：
   - 该势力仍有其他城 → **重选君主**：从该势力剩余未被俘武将中取智力最高者为新君主，把该势力其余城与未被俘武将的归属改归新君主；
   - 该势力已无任何城 → 该势力**灭亡**（本切片不做统一全国的全局胜负判定）。
7. 作为玩家，本切片俘虏（`officer.lordId !== 所在城.lordId`）**不可被任何指令指派**，"在任武将"定义排除俘虏。

## 规则细节

### 出征（占人 ✓ · 一次最多 10 将 · 效果时机=月末）
- 从单一本城发起：所选武将必须全部位于同一座城（本城）且全为在任武将；随军粮草取自该城城粮。
- 前置条件（下令时校验，全部满足才可下令）：
  - 选中武将数 1~10，且全部在本城、全部在任（未占用、非俘虏）；
  - 本城城粮 ≥ 1（城粮 > 0）；
  - 随军粮草为整数且 1 ≤ 随军粮草 ≤ 本城城粮；
  - 目标城存在，且**不是本城**、**不是己方城**（目标城 lordId ≠ 出征方 lordId）；
  - 目标城与本城**相邻**（路径可达 = 相邻；不做多跳路径搜索）。
- 下令阶段（立即发生）：
  - 本城城粮 -= 随军粮草；
  - 选中武将全部 busy（占人）；
  - 把"出征命令（武将集合 + 目标城 + 随军粮草）"登记进**月末待执行队列**；出征命令在队列中排在所有普通待执行指令之后执行。
- 执行阶段（月末，先于收粮/收税）：
  - 计算攻守兵力总和并判胜负（见上）；
  - 胜：目标城 lordId 改为攻方、城粮 += 随军粮草；出征武将 cityId 改为目标城；
  - 败：目标城不变、随军粮草损失；出征武将 cityId 改为目标城（就地成俘虏）；
  - 结算后处理被俘君主（重选 / 灭亡，见关键行为 6）。
- 月末待执行队列在执行后清空（不跨月残留）。出征武将月末仍走统一的 `busy:=false` + 体力恢复，但**不回出发城**（cityId 已由战斗结算决定）。

### 战斗（本切片从简）
- **确定性**：纯按兵力总和比较，不消耗 RNG，无随机。
- **无兵力损耗**：胜负只比总和，战后双方武将兵不变（败方武将成俘虏后其兵随俘虏冻结、不可用）。
- 同月多支出征打同一座城：月末按入队顺序逐条结算，每条基于当时状态，连锁结果不另作处理（留边界）。

### 俘虏（派生状态，非存储字段）
- 定义：`officer.lordId !== 所在城.lordId` 即为俘虏；不新增 `captured` 字段。
- 俘虏在本切片"什么都不变"——其 lordId / cityId / 兵 / 体力均不因被俘而改动，仅因所在城易主而在派生意义上成为俘虏。
- 俘虏不可被任何指令指派；守方兵力统计排除城内俘虏；重选君主的候选池排除俘虏。

### 城邻接（新增地图拓扑）
- 新增城与城之间的**无向邻接关系**，保证至少存在一对敌城相邻（玩家有可攻目标）。
- "可达"= 目标城与出发城相邻。

### 数值 & 归类（依总纲取舍）
- **规则身份（内联领域常量）**：单次出征武将上限 10（量纲上限）；胜负判据（攻 > 守 才胜，平局守方胜）。
- **领域数据**：城邻接关系（地图拓扑，随对局/地图而定，放世界/地图模块，不进 config、不做全局常量）。
- 门槛字面量：武将数 1~10、随军粮草 1~城粮、城粮 ≥ 1。
- 本切片不新增 config 平衡旋钮（出征无固定金/体力扁平成本，仅消耗随军粮草=城粮）。

### 与既有循环/状态的衔接
- 出征沿用"占人 = `Officer.busy`、月末统一翻回 false + 体力恢复"机制；其特殊性（不回出发城、可能成俘虏/进驻新城）由战斗结算改 cityId/城 lordId 表达，turn 层无需为出征开特例。
- 出征是第二类"效果延到月末执行"的指令，进 `pendingCommands`（判别式并集追加 `campaign` 分支），由 turn 层按 type 分派；排在掠夺等普通待执行指令之后。
- 引入"武将跨城移动"（出征武将 cityId 改变）与"城易主"（city.lordId 改变），均为本切片首次。
- AI 本切片不主动出征（AI 城仍静止，但可作为玩家出征目标，也可被攻陷/重选君主/灭亡）。
- 本切片不改动收粮/收税公式、体力恢复、跨年逻辑（仅在月末待执行队列里追加出征结算，仍先于收粮/收税）。

## 范围 & 不做
- 做：出征指令（多将选择最多 10、随军粮草、相邻敌城、可达校验、下令扣粮占人入队、月末战斗结算）；新增城邻接拓扑；武将跨城移动；城易主与占领；从简战斗（按兵力总和判胜负、无损耗、确定性）；俘虏（派生）；被俘君主的重选 / 势力灭亡。
- 不做（本切片）：战棋棋盘/逐回合战斗/兵力损耗/伤亡（留军事切片深化）；多跳路径搜索与中途拦截；统一全国的全局胜负判定与游戏结束流程；俘虏的处置（释放/录用/斩杀/赎回）；AI 主动出征；出征过程中的行军时间/距离耗时（本切片下令即占人、月末即到达结算）；出征 UI 的具体排版（仅约定多选最多 10 将 + 粮草输入 + 目标城选择）。

## 验收标准
- [ ] 出征下令：本城有在任武将、城粮 ≥ 1、随军粮草 ∈ [1,城粮]、目标城为相邻敌城、选中 1~10 名全在本城的在任武将时可下令；立即扣本城城粮（随军粮草）、选中武将全部 busy、出征命令入队（排在普通待执行指令之后）。
- [ ] 出征前置任一不满足（无在任武将 / 城粮 ≤ 0 / 随军粮草越界 / 目标城是本城或己方城 / 目标城不相邻 / 武将数 0 或 >10 / 选中武将不全在本城或非在任）时拒绝，状态不变。
- [ ] 月末战斗先于收粮/收税结算；攻方兵力（出征武将兵之和）> 守方兵力（目标城归属方武将兵之和 + 后备兵，排除俘虏）才攻方胜，否则守方胜（含平局）。
- [ ] 攻方胜：目标城 lordId 改为攻方、城粮 += 随军粮草、后备兵随城易主；出征武将 cityId=目标城；原守军武将就地成俘虏（lordId ≠ 所在城 lordId）。
- [ ] 攻方败：目标城不变、随军粮草损失；出征武将 cityId=目标城并就地成俘虏。
- [ ] 君主被俘且该势力仍有其他城：重选——从剩余未被俘武将取智力最高者为新君主，其余城与未被俘武将归属改归新君主；被俘君主与俘虏不改归属。
- [ ] 君主被俘且该势力已无城：该势力灭亡（无新君主，剩余武将均为俘虏）。
- [ ] 俘虏不可被任何指令指派；既有"在任武将"查询排除俘虏。
- [ ] 出征武将月末 `busy` 翻回 false 且体力恢复，但 cityId 为战斗结算后的城（不回出发城）。
- [ ] 既有经营循环（开垦/招商/征兵/分配/掠夺/侦察、收粮/收税、体力恢复、跨年）行为不受影响。
- [ ] 给定相同 RNG 种子，整段推进结果可复现（战斗确定性、不耗 RNG）。

## 待定决策
- 战斗深化（战棋棋盘、兵力损耗/伤亡、士气/地形等）留军事切片。
- 俘虏处置（释放/录用/赎回/斩杀）留后续切片；本切片俘虏只是被冻结的派生状态。
- 统一全国的全局胜负判定与游戏结束流程留后续切片。
- 具体城邻接拓扑（哪几对城相邻）在 dev/实现阶段定，PRD 仅要求"至少一对敌城相邻"。
- AI 出征决策留 AI 切片。
