# plunder-scout（掠夺 / 侦察）PRD

## 问题
现有指令（开垦/招商/征兵）效果都在**下令当下立即生效**，"占人"只靠 `Officer.busy` 标志、月末统一回城，没有"待月末执行"的概念。但游戏需要两类新行为：

- **掠夺**：玩家在困境下牺牲本城长期发展，换取**当下的粮草**——这是一种"竭泽而渔"的应急手段，其破坏与收益都应在**月末**结算。
- **侦察**：玩家在与他城交手前，需要花钱派人去**查看一座非己方城的详细信息**——真正的价值是下令瞬间看到对方城情报。

掠夺是本项目**第一条"效果延到月末执行"的指令**，需要引入"本月待月末执行的掠夺"这一新状态。

## 目标
在现有月度循环上新增两条占人指令：

- **掠夺**（占人 ✓，效果时机=月末）：下令时扣体力、入队、离城；月末统一在收粮/收税**之前**执行——破坏本城（农业/商业/民忠减半），并把执行人能力转化为本城粮、金收益。
- **侦察**（占人 ✓，效果时机=下令瞬间）：下令时扣体力、扣本城金，并**立即向玩家展示目标（非己方）城详情**；执行人离城，月末回城。core 侧侦察月末无实质效果。

并明确月末新顺序：**先执行待月末执行的指令（掠夺）→ 再收粮/收税 → 再回城/体力恢复**。

## 关键行为 / 用户故事
1. 作为玩家，我可以对本城**在任**武将下「掠夺」指令：立即扣其体力 12、该武将离城（占人），本城被登记为"本月待掠夺"。
2. 作为玩家，掠夺的破坏与收益都在**月末**执行，而非下令当下——下令当下本城属性/资源不变。
3. 作为系统，月末**先于收粮/收税**执行所有待掠夺：每条掠夺把目标城**农业、商业、民忠各减半**（`floor`），并给该城 `粮 += (执行人智力+武力)×5`、`金 += (执行人智力+武力)×2`。
4. 作为系统，同一座城本月被多条掠夺登记时，月末按登记顺序**依次连续减半**（后一条基于前一条减半后的值），收益各自独立累加。
5. 作为玩家，当某月恰为收粮月（6/10）或收税月（3/6/9/12）时，因掠夺先执行，该城**当月**的收粮/收税即按减半后的农业/商业结算（掠夺会削当月产出）。
6. 作为玩家，掠夺前置任一不满足（本城无在任武将 / 执行人体力 < 12）时，下令被拒绝、状态不变。
7. 作为玩家，我可以对本城在任武将下「侦察」指令并指定一座**非己方、非本城**的目标城：立即扣执行人体力 10、扣**本城**金 20，系统**立即弹出该目标城详情面板**，执行人离城（占人）。
8. 作为玩家，侦察前置任一不满足（本城金 < 20 / 本城无在任武将 / 执行人体力 < 10 / 目标城是本城或己方城）时，下令被拒绝、状态不变。
9. 作为玩家，侦察的执行人在月末回到出发城；侦察在 core 月末无其它效果（情报展示已在下令瞬间完成）。

## 规则细节

### 掠夺（占人 ✓ · 效果时机=月末）
- 目标 = 执行人所在城（本城），不接受外部目标城参数。
- 前置条件（下令时校验，全部满足才可下令）：
  - 本城内至少 1 名在任武将（作为执行人）
  - 执行人体力 ≥ 12
- 下令阶段（立即发生）：
  - 执行人体力 -= 12
  - 把"本城 + 执行人"登记进**本月待月末执行的掠夺队列**
  - 执行人离城（占人）
- 执行阶段（月末，先于收粮/收税）：
  - 破坏本城：`农业 = floor(农业/2)`、`商业 = floor(商业/2)`、`民忠 = floor(民忠/2)`
  - 本城收益：`粮 += (执行人智力 + 武力) × 5`、`金 += (执行人智力 + 武力) × 2`
  - 多条掠夺同城时按登记顺序依次执行（连续减半）
- 月末队列在执行后清空（不跨月残留）。

### 侦察（占人 ✓ · 效果时机=下令瞬间）
- 需指定目标城 `targetCityId`。
- 前置条件（下令时校验，全部满足才可下令）：
  - 本城金 ≥ 20
  - 本城内至少 1 名在任武将（作为执行人）
  - 目标城存在，且**不是本城**、**不是己方城**（目标城 `lordId` ≠ 执行人 `lordId`）
  - 执行人体力 ≥ 10
- 下令阶段（立即发生）：
  - 执行人体力 -= 10、本城金 -= 20
  - **立即弹出目标城详情面板**（侦察的真正效果，UI 行为：读取并展示目标城当前全部信息）
  - 执行人离城（占人）
- 执行阶段（月末）：执行人回到出发城；core 无其它状态变更。

### 月末顺序（更新既有循环）
原顺序为：AI 下令 → 收粮/收税 → 回城+体力恢复 → 月份+1。本切片在其中插入"待月末执行指令"，新顺序：
1. AI 下令（本切片仍为空步）
2. **执行本月待月末执行的指令（掠夺）**，执行后清空队列
3. 收粮/收税结算（按掠夺后的农业/商业）
4. 占用武将回城 + 体力恢复（封顶 100）
5. 月份 +1（跨年）

### 数值 & 归类（依总纲取舍）
- **扁平成本 → 平衡旋钮**：掠夺体力消耗 12、侦察体力消耗 10、侦察本城金消耗 20。
- **公式系数 / 除数 → 规则身份（内联领域常量）**：掠夺收益系数粮 ×5、金 ×2；破坏减半除数（÷2 取 `floor`）。
- 门槛字面量：在任武将 `≥1`、体力/城金门槛同上消耗值。

### 与既有循环/状态的衔接
- 掠夺、侦察均沿用既有"占人 = `Officer.busy`，月末翻回"机制。
- 新增"本月待月末执行的掠夺"为新状态（队列），仅掠夺使用；侦察不入此队列。
- 掠夺使民忠**首次变为动态值**（被减半）：民忠原为静态，本切片起可被掠夺降低（仍夹在 `[0,100]`，本切片无回升机制）。
- 本切片不改动收粮/收税公式、体力恢复速率、跨年逻辑（仅调整月末执行顺序，插入掠夺步骤）。
- AI 本切片不使用掠夺/侦察（AI 城仍静止，但其城可作为玩家侦察目标）。

## 范围 & 不做
- 做：掠夺（月末执行的破坏+收益）、侦察（下令瞬间展示目标城情报）两条占人指令及其前置/扣减/执行；引入"本月待月末执行的掠夺队列"状态与月末新执行顺序；民忠由静态改为可被掠夺减半。
- 不做（本切片）：民忠的自然涨跌/回升机制；AI 使用掠夺/侦察；地图/邻接/距离对侦察目标的限制（任意非己方城均可侦察）；侦察面板的具体 UI 排版（仅约定"展示目标城当前全部信息"）；掠夺对武将兵/后备兵的影响；战斗/占领。

## 验收标准
- [ ] 掠夺下令：本城有在任武将且执行人体力 ≥ 12 时可下令，立即扣体力 12、执行人 busy、本城入待掠夺队列；下令当下本城农业/商业/民忠/粮/金**不变**。
- [ ] 掠夺前置任一不满足（无在任武将 / 体力 < 12）时拒绝，状态不变。
- [ ] 月末执行掠夺：农业/商业/民忠各 `floor(/2)`；粮 `+=(智+武)×5`、金 `+=(智+武)×2`；执行后队列清空、不跨月残留。
- [ ] 月末顺序正确：掠夺**先于**收粮/收税——在收粮/收税月掠夺，本城当月收粮/收税按减半后农业/商业结算。
- [ ] 同城多条掠夺：月末按登记顺序连续减半，收益各自独立累加。
- [ ] 侦察下令：本城金 ≥ 20、有在任武将、执行人体力 ≥ 10、目标城非本城且非己方城时可下令，立即扣体力 10、扣本城金 20、执行人 busy；UI 立即展示目标城详情。
- [ ] 侦察前置任一不满足（本城金 < 20 / 无在任武将 / 体力 < 10 / 目标城是本城或己方城 / 目标城不存在）时拒绝，状态不变。
- [ ] 掠夺/侦察的执行人月末回到出发城（busy 翻回 false）。
- [ ] 既有经营循环（收粮/收税公式、体力恢复、跨年、征兵/分配）行为不受影响。
- [ ] 给定相同 RNG 种子，整段推进结果可复现（core 纯函数、确定性）。
