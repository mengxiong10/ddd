# debut-search 登场与搜寻 PRD

## 问题

目前所有武将与道具在开局就全部在场、各有归属，地图是「静态阵容」：玩家没有随时间扩充班底的途径，道具也只能靠初始播种，缺少「访贤纳士、挖掘宝物」这一经典策略循环。也没有把「一名武将花一个月在本城探查、博取意外收获」的低风险经营动作。

## 目标

引入「登场」与「搜寻」：部分武将/道具开局处于未登场状态，到达各自登场年后出现在某城但**不归属任何君主/城**（在野/未发现，对玩家隐藏）；占据该城的势力可派在任武将执行「搜寻」，月末有概率获得外快/粮食，或发现并招募在野武将、发现隐藏道具，从而随时间动态扩充阵容。

## 关键行为 / 用户故事

1. 作为玩家，我希望部分武将/道具不在开局出现，而是到达「登场年」后才现身于某座城（指定城；未指定则随机一城），且不归属任何势力。
2. 作为玩家，我要对本城一名在任武将下达「搜寻」，让他花一个月在本城探查，月末博取意外收获。
3. 作为玩家，我希望搜寻可能找到隐匿的在野武将并把他招募到本势力，或发现本城的隐藏道具使其可被赏赐。
4. 作为玩家，我希望搜寻即便没找到人/物，也可能带来一笔外快或粮食，或一无所获。
5. 作为玩家，我希望某武将是否容易被招募，取决于他的「伯乐条件」与执行人的智力（伯乐本人必成、无指定看智力、他人无效）。

## 关键规则（与用户对齐结论）

### 登场（Debut）

- 部分武将/道具开局处于**未登场**：暂不在 `state.officers`/`state.items` 中，存于 `GameState` 顶层的**待登场池**，每条记录带：完整的武将/道具数据、`登场年`、`目标城`（可为空=随机）。
- **登场为运行时事件**：月末在固定日历的**「月份+1」之后**执行登场步骤——凡待登场池中 `当前年 ≥ 登场年` 者登场：
  - 决定落城：有指定城用指定城；未指定则在**全部城中随机**一座（消耗 RNG，确定性可复现）。
  - 武将登场：插入 `state.officers`，`lordId = null`（无主→在野）、`cityId = 落城`、`troops = 0`；该条移出待登场池（一次性）。
  - 道具登场：插入 `state.items`，`holder = { kind:'city', cityId: 落城 }`、`discovered = false`（未发现）；移出待登场池。
- 月末固定顺序因此扩展为：`pendingCommands → settle（收粮/收税）→ 回城+体力恢复 → 月份+1 → 登场`。

### 在野 / 无主（Wandering）

- `Officer.lordId` 改为**可空**：`null` 表示无主，统一覆盖「未登场（在池中）」与「在野（已登场未被招募）」。归属武将 `lordId` 非空。
- 在野武将对玩家**隐藏**：不计入城池在任列表、不参与守城兵力、不被任何指令指派；唯有「搜寻」可发现并招募之。
- 既有派生需对 `null` 加守卫：俘虏判定（`isCaptive`）对无主武将返回 false；`officersInCity` 的在任筛选排除无主武将；君主忠诚派生（`lordId===id`）对 `null` 自然不成立。

### 道具的「未发现 / 已发现」

- `Item` 新增 `discovered: boolean`：开局既有（fixture 播种）道具 `discovered = true`；登场道具落城时为 `false`。
- 未发现道具对玩家隐藏、**不可被赏赐**；经「搜寻」发现后置 `true`，方可进入赏赐候选。

### 伯乐（Recruiter）

- 武将与道具各新增 `recruiterId: OfficerId | null`（伯乐）：`null` = 无指定（用户口径的「0」），否则指向特定武将 id。
- 招募/发现的成败按伯乐条件判定（见下）。

### 搜寻（Search）指令

- **占人 ✓**；**效果时机 = 月末**（沿用掠夺形态：下令入队、月末执行）。
- **前置条件**：城里至少 1 名在任武将（即执行人本身可用）、执行人体力 ≥ 8；不需要金钱。
- **下令阶段**：扣执行人体力 8、置 busy（离城）、入 `pendingCommands`；不动 RNG、不改城。
- 体力成本 8 为**扁平成本** → 进 `GameConfig`（`searchStaminaCost`），与开垦/掠夺等一致。

### 搜寻执行阶段（月末，四种结果各 1/4 概率）

所有判定中的「智力」一律取**有效智力**（基础 + 道具加成）。城金/城粮各有量纲上限 `30000`（内联常量）。

- **无事发生**。
- **尝试发现武将或道具**（见下「发现判定」）。
- **获得金钱**：`上限 = max(10, 有效智力×2)`；`获得 = RandInt(10, 上限)`；`城金 = min(城金 + 获得, 30000)`。
- **获得粮食**：`上限 = max(10, 有效智力×2)`；`获得 = RandInt(10, 上限)`；`城粮 = min(城粮 + 获得, 30000)`。

### 发现判定

1. **筛子**：`RandInt(0, 149) < 有效智力` 才继续，否则视为无事发生（智力 150 必中，智力 50 约 1/3）。
2. 通过后掷 1/2 决定本次找**武将**还是**道具**。
3. **搜到武将**：
   - 候选 = 本城所有在野武将（`lordId === null`）；为空则无事发生（不改找道具）。
   - 随机选中一名，读其 `recruiterId`：
     - 伯乐 = 执行人本人 → 必定成功；
     - 伯乐 = `null`（无指定）→ `RandInt(0, 109) < 有效智力` 则成功；
     - 伯乐 = 他人 → 失败。
   - 成功：该武将 `lordId = 执行人.lordId`（归本势力君主）、`loyalty = RandInt(70, 99)`、`cityId` 保持本城、`troops` 保持 0。
4. **搜到道具**：
   - 候选 = 本城所有未发现道具（`holder` 为本城且 `discovered === false`）；为空则无事发生（不改找武将）。
   - 随机选中一件，读其 `recruiterId`：伯乐 = `null` 或 = 执行人本人 → 成功；否则失败。
   - 成功：该道具 `discovered = true`（可被赏赐）。

### 同城多搜寻 & 行动方

- 同城同月可多名武将各自搜寻；月末按 `pendingCommands` **入队序逐条结算**，在野武将/未发现道具候选池随结算实时更新（保证确定性、不重复招募同一目标）。
- 占据该城的任一势力（含 AI）都可搜寻其中的在野武将/未发现道具。搜寻为普通占人：执行人月末回出发城（非出征例外）。

## 范围 & 不做

- 做（仅 `src/core/`，含单测）：
  - 待登场池（`GameState` 顶层）+ 月末登场步骤（运行时落城、随机选城消耗 RNG）。
  - `Officer.lordId` 可空（无主/在野）；既有派生（`isCaptive`/`officersInCity`/君主忠诚）加 `null` 守卫。
  - `Item.discovered` 字段；`recruiterId`（伯乐）字段（武将 + 道具）。
  - 「搜寻」指令：`canSearch` 校验 + `search` 下令（占人、入队）+ `executeSearch` 月末执行（四分支 + 发现判定），经 `game.apply/canApply`、`turn` 月末分派接线。
  - `searchStaminaCost` 进 `GameConfig`；fixture 播种若干未登场武将/道具与既有道具的 `discovered=true`。
- 不做（本切片）：
  - 任何 UI（搜寻下令入口、在野/隐藏的展示、搜寻结果反馈/日志）——留后续切片。
  - 在野武将的自然消失/流动、登场后被他城争夺的特殊机制、伯乐之外的招募手段（策反/在野主动投靠）。
  - 道具的打造/购买/掉落；搜寻外的其他随机事件。
  - 登场年的「月」粒度（只到年）。

## 验收标准

- [ ] 未登场武将/道具不在 `officers`/`items` 中、不影响任何现有查询；到达登场年后于月末登场到指定城（未指定则随机一城，确定性可复现），武将 `lordId=null`、道具 `discovered=false`。
- [ ] 在野武将（`lordId=null`）不被判为俘虏、不进在任列表、不参与守城兵力，仅可经搜寻招募。
- [ ] 搜寻下令：体力 ≥ 8 才可下令，扣体力 8、置 busy、入 `pendingCommands`；不耗 RNG、不改城；非法即 no-op。
- [ ] 搜寻月末四分支各 1/4：金钱/粮食按 `RandInt(10, max(10,有效智力×2))` 增、各封顶 30000；无事分支不改状态。
- [ ] 发现分支：`RandInt(0,149) ≥ 有效智力` 即无事；通过后 1/2 找武将/道具；候选为空当作无事（不改找另一类）。
- [ ] 招募判定：伯乐=执行人必成、伯乐=null 看 `RandInt(0,109)<有效智力`、伯乐=他人必败；成功后该武将归执行人君主、忠诚 `RandInt(70,99)`。
- [ ] 道具发现判定：伯乐=null 或执行人成功、否则失败；成功后 `discovered=true`，方可被赏赐；未发现道具不在赏赐候选内。
- [ ] 同城多搜寻按入队序结算，候选池实时更新，同一在野武将/道具不被两条搜寻重复获得。
- [ ] 月末固定顺序为 `pendingCommands → settle → 回城+体力恢复 → 月份+1 → 登场`。
